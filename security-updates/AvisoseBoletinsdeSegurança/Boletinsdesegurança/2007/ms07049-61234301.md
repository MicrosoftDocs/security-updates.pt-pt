Security Bulletin

Microsoft Security Bulletin MS07-049 - Importante
=================================================

Vulnerabilidade no Virtual PC e Virtual Server poderia permitir Elevação de Privilégios (937986)
------------------------------------------------------------------------------------------------

Data de publicação: 14 de agosto de 2007 | Updated: 13 de novembro de 2007

**actualizada:** 2.0

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança importante resolve uma vulnerabilidade comunicada de forma privada. Esta é uma vulnerabilidade de elevação de privilégios. A vulnerabilidade no Microsoft Virtual PC e Microsoft Virtual Server poderia permitir que um utilizador convidado do sistema operativo pudesse executar código no sistema operativo anfitrião ou noutro sistema operativo convidado. Apenas os utilizadores convidados de um sistema operativo com permissões de administrador conseguiriam explorar esta vulnerabilidade. Os utilizadores convidados de um sistema operativo que não têm permissões de administrador não conseguem explorar esta vulnerabilidade.

Esta é uma actualização de segurança importante para versões suportadas do Microsoft Virtual PC 2004, Microsoft Virtual Server 2005, Microsoft Virtual Server 2005 R2, Microsoft Virtual PC para Mac versão 6.1 e Microsoft Virtual PC para Mac versão 7. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

Esta actualização de segurança resolve a vulnerabilidade ao melhorar a interacção e inicialização de componentes que comunicam com o sistema operativo anfitrião para máquinas virtuais no Microsoft Virtual PC e Microsoft Virtual Server. Para mais informações sobre as vulnerabilidades, consulte a subsecção de Perguntas Mais Frequentes quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade.

**Questões conhecidas.** O [Artigo 937986 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/937986) documenta as questões actualmente conhecidas que os clientes podem experimentar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Para determinar o ciclo de vida de suporte da versão ou edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Perguntas mais frequentes relativas a esta actualização de segurança
--------------------------------------------------------------------

<span></span>
**Por que razão decidiu a Microsoft voltar a publicar este boletim em 13 de Novembro de 2007?**  
A actualização de segurança para Microsoft Virtual PC 2004, Microsoft Virtual PC 2004 Service Pack 1, Microsoft Virtual Server 2005 Standard Edition e Microsoft Virtual Server 2005 Enterprise Edition não foi instalada correctamente no Windows 2000 Service Pack 4 devido a uma questão relacionada com a chave de registo. Está agora disponível uma actualização de segurança revista para se efectuar a instalação no Windows 2000 Service Pack 4. A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade. Não é necessária qualquer acção em sistemas em que a actualização de segurança tenha sido instalada com sucesso.

A actualização de segurança para Microsoft Virtual PC 2004, Microsoft Virtual PC 2004 Service Pack 1, Microsoft Virtual Server 2005 Standard Edition, Microsoft Virtual Server 2005 Enterprise Edition, Microsoft Virtual Server 2005 R2 Standard Edition e Microsoft Virtual Server 2005 R2 Enterprise Edition extrai os ficheiros para a pasta %TEMP%. Está agora disponível para instalação uma actualização de segurança revista que extrai os ficheiros para a pasta %ProgramFiles%. A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade. Não é necessária qualquer acção em sistemas em que a actualização de segurança tenha sido instalada com sucesso.

**Quais são as questões conhecidas que os clientes poderão experimentar quando instalam esta actualização de segurança?**  
O [Artigo 937986 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/937986) documenta as questões actualmente conhecidas que os clientes podem experimentar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

**Estou a usar uma edição anterior do software discutido neste boletim de segurança. O que devo fazer?**  
O software afectado incluído neste boletim foi testado para determinar quais as versões afectadas. As outras edições ultrapassaram o respectivo ciclo de vida de suporte. Para determinar o ciclo de vida de suporte da edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Os utilizadores que tenham edições anteriores do software deviam considerar a migração para edições suportadas como uma prioridade, de modo a poderem proteger os seus sistemas contra a potencial exposição a vulnerabilidades. Para mais informações sobre o Ciclo de Vida dos Produtos Windows, visite o seguinte Web site do [Ciclo de Vida de Suporte Microsoft.](http://support.microsoft.com/lifecycle/) Para mais informações sobre o período de suporte alargado a actualizações de segurança para estas edições de software, visite o [Web site de Suporte Técnico da Microsoft](http://go.microsoft.com/fwlink/?linkid=33328).

Para obter informações de contacto, visite o [Web site Microsoft Worldwide Information](http://go.microsoft.com/fwlink/?linkid=33329), seleccione o país e, em seguida, clique em **Go** para ver uma lista de números de telefone. Vulnerabilidade de Sobrecarga de Área dinâmica para dados no Virtual PC e Virtual Server - CVE-2007-0948  
--------------------------------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de elevação de privilégios no Microsoft Virtual PC e Microsoft Virtual Server que poderia permitir que um utilizador com permissões de administrador num sistema operativo convidado executasse código no sistema operativo anfitrião ou noutros sistemas operativos convidados. Um intruso com permissões de administrador no sistema operativo anfitrião poderia explorar ao vulnerabilidade ao executar código especialmente concebido para o efeito no sistema operativo convidado. Isto poderia resultar numa sobrecarga da área dinâmica para dados (heap) no sistema operativo anfitrião ou em sistemas operativos convidados. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2007-0948](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-0948).
  
#### Factores atenuantes para a Vulnerabilidade de Sobrecarga de Área dinâmica para dados no Virtual PC e Virtual Server - CVE-2007-0948
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso tem de ter permissões de administrador no sistema operativo convidado para explorar esta vulnerabilidade. Os inícios de sessão como convidado num sistema operativo sem permissões de administrador não conseguem explorar esta vulnerabilidade.  
-   O Microsoft Virtual PC 2007 não é vulnerável a esta questão. Pode transferir a versão completa do Microsoft Virtual PC 2007 no [Centro de Transferências da Microsoft](http://www.microsoft.com/downloads/details.aspx?familyid=04d26402-3199-48a3-afa2-2dc0b40a73b6).  
-   Microsoft Virtual Server 2005 R2 SP1 não é vulnerável a esta questão. Pode transferir a versão completa do Microsoft Virtual Server 2005 R2 SP1 no [Centro de Transferências da Microsoft](http://www.microsoft.com/technet/prodtechnol/eval/virtualserver/default.mspx).
  
#### Soluções alternativas para a Vulnerabilidade de Sobrecarga de Área dinâmica para dados no Virtual PC e Virtual Server - CVE-2007-0948
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas mais frequentes sobre a Vulnerabilidade de Sobrecarga de Área dinâmica para dados no Virtual PC e Virtual Server - CVE-2007-0948
  
**Qual é a abrangência desta vulnerabilidade? **  
Existe uma vulnerabilidade de elevação de privilégios no Microsoft Virtual PC e Microsoft Virtual Server que poderia permitir que um utilizador com permissões de administrador num sistema operativo convidado executasse código no sistema operativo anfitrião ou noutros sistemas operativos convidados. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Em seguida, poderia instalar programas ou visualizar, alterar ou apagar dados.
  
**O que provoca a vulnerabilidade? **  
A interacção e inicialização de componentes que comunicam com o sistema operativo anfitrião não são executadas correctamente no Microsoft Virtual PC e Microsoft Virtual Server.
  
**Como poderia um intruso utilizar a vulnerabilidade? **  
Um intruso com permissões de administrador no sistema operativo convidado que explorasse com sucesso esta vulnerabilidade poderia assumir o controlo total do sistema afectado. O intruso poderia então ganhar acesso ao sistema operativo anfitrião ou a sistemas operativos convidados, o que poderia resultar numa sobrecarga da área dinâmica para dados (heap) no sistema operativo anfitrião ou nos sistemas convidados.
  
**Como poderia um intruso explorar a vulnerabilidade? **  
Um intruso com permissões de administrador no sistema operativo convidado poderia executar código especialmente concebido para explorar esta vulnerabilidade. Um intruso tem de ter permissões de administrador no sistema operativo convidado para explorar esta vulnerabilidade.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade? **  
Quaisquer sistemas que executem o Microsoft Virtual PC ou o Microsoft Virtual Server, como estações de trabalho ou servidores de terminal correm risco com esta vulnerabilidade. Esta vulnerabilidade requer que um intruso tenha permissões de administrador no sistema operativo convidado para explorar esta vulnerabilidade. Os inícios de sessão como convidado num sistema operativo sem permissões de administrador não conseguem explorar esta vulnerabilidade.
  
**O que faz a actualização? **  
A actualização remove a vulnerabilidade no Microsoft Virtual PC e Microsoft Virtual Server ao modificar a interacção e inicialização de componentes que comunicam com o sistema operativo anfitrião.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente? **  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.

**Verificar se a actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "Orientações e ferramentas de detecção e implementação" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

-   **Verificação da Chave de Registo**

    Poderá também verificar os ficheiros instalados por esta actualização de segurança consultando as chaves de registo listadas na **Tabela de referência**, nesta secção.

    Estas chaves de registo poderão não conter uma lista completa dos ficheiros instalados. Além disso, estas chaves de registo poderão não ser criadas correctamente se um administrador ou um OEM integrar ou associar esta actualização de segurança aos ficheiros de origem de instalação do Windows.

**Verificar se a actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "Orientações e ferramentas de detecção e implementação" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

-   **Verificação da Chave de Registo**

    Poderá também verificar os ficheiros instalados por esta actualização de segurança consultando as chaves de registo listadas na **Tabela de referência**, nesta secção.

    Estas chaves de registo poderão não conter uma lista completa dos ficheiros instalados. Além disso, estas chaves de registo poderão não ser criadas correctamente se um administrador ou um OEM integrar ou associar esta actualização de segurança aos ficheiros de origem de instalação do Windows.
