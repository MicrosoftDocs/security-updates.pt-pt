Security Bulletin

Microsoft Security Bulletin MS08-036 - Importante
=================================================

Vulnerabilidades no Pragmatic General Multicast (PGM) poderiam permitir Negação de Serviço (950762)
---------------------------------------------------------------------------------------------------

Data de publicação: 10 de junho de 2008 | Updated: 17 de fevereiro de 2010

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve duas vulnerabilidades no protocolo Pragmatic General Multicast (PGM), comunicadas de forma privada, que poderiam permitir uma negação de serviço se um sistema afectado recebesse pacotes PGM mal formados. Um intruso que explorasse esta vulnerabilidade com sucesso poderia fazer com que o sistema de um utilizador deixasse de responder e tivesse de ser reiniciado para restaurar a funcionalidade. Tenha em atenção que a vulnerabilidade na negação de serviço não permitiria que um intruso executasse o código ou elevasse os seus direitos de utilizador, mas poderia fazer com que o sistema afectado deixasse de aceitar pedidos.

Esta actualização de segurança está classificada como Importante para todas as edições suportadas do Windows XP e Windows Server 2003 e como Moderada para todas as edições suportadas do Windows Vista e Windows Server 2008. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve estas vulnerabilidades modificando a forma como o PGM analisa pacotes mal formados. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.**  A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade.

**Questões conhecidas.** O [Artigo 950762 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/950762) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões. Quando as questões actualmente conhecidas e soluções recomendadas software dizem respeito apenas às edições específicas deste software, este artigo não fornece ligações a outros artigos.

#### Software Afectado e Software Não Afectado

O software que se segue foi testado para determinar quais as versões ou edições afectadas. As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Para determinar o ciclo de vida de suporte da versão ou edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Vulnerabilidade de Comprimento Inválido no PGM - CVE-2008-1440  
--------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de negação de serviço em implementações do protocolo Pragmatic General Multicast (PGM) no Microsoft Windows XP e no Windows Server 2003. A vulnerabilidade é causada pela validação incorrecta de pacotes PGM especialmente concebidos para o efeito. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia fazer com que o computador deixasse de responder e tivesse de ser reiniciado para restaurar a funcionalidade.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-1440](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1440).
  
#### Factores Atenuantes para a Vulnerabilidade de Comprimento Inválido no PGM - CVE-2008-1440
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Em sistemas com o Windows XP e o Windows Server 2003 instalado, o Pragmatic General Multicast (PGM) só está activado quando o Microsoft Message Queuing (MSMQ) 3.0 está instalado. O serviço MSMQ não é instalado por predefinição.
  
#### Soluções Alternativas para a Vulnerabilidade de Comprimento Inválido no PGM - CVE-2008-1440
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Comprimento Inválido no PGM - CVE-2008-1440
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de "negação de serviço". Um intruso anónimo e remoto que explorasse esta vulnerabilidade poderia fazer com que o sistema afectado deixasse de responder até que fosse reiniciado. Tenha em atenção que a vulnerabilidade na negação de serviço não permitiria que um intruso executasse o código ou elevasse os seus direitos de utilizador, mas poderia fazer com que o sistema afectado deixasse de aceitar pedidos.
  
**O que provoca a vulnerabilidade?**    
O código de análise do protocolo não valida correctamente o campo de comprimento da opção de um pacote PGM. Se um sistema afectado receber um pacote PGM especialmente concebido para o efeito com um campo de comprimento de opção inválido, o analisador entra num ciclo infinito e acabará por fazer com que o sistema deixe de responder.
  
**O que é o Pragmatic General Multicast (PGM)? **  
O PGM é um protocolo multicast fiável e escalável que permite que os receptores detectem perdas, solicitem a retransmissão de dados perdidos ou notifiquem uma aplicação da existência de perdas não recuperáveis. O PGM é um protocolo cuja fiabilidade depende do receptor, o que significa que o receptor é responsável por garantir que todos os dados foram recebidos, libertando o remetente da responsabilidade pela fiabilidade das comunicações. O PGM é indicado para aplicações que requerem entrega de dados multicast sem duplicação, a partir de várias fontes para vários receptores. O PGM não suporta entregas reconhecidas nem garante o envio de pacotes de vários remetentes. Para mais informações sobre o PGM, consulte o seguinte [artigo MSDN](http://msdn.microsoft.com/en-us/library/ms740125(vs.85).aspx).
  
**O que é o Microsoft Message Queuing?**  
O Microsoft Message Queuing (MSMQ) é um componente do sistema operativo Windows que permite que aplicações em cooperação enviem e recebam mensagens entre si, mesmo se a aplicação destinatária pretendida não estiver em execução ou se o computador onde a aplicação destinatária ou remetente está a ser executada estiver desligado da rede. As mensagens são guardadas e reencaminhadas pelo MSMQ até chegarem à fila de destino. Posteriormente, quando uma aplicação destinatária for executada, poderá recolher as mensagens da fila. O MSMQ separa as aplicações destinatárias e remetentes para que não tenham de ser executadas ao mesmo tempo. O Message Queuing fornece segurança avançada incorporada, suporte de transacções e outras funcionalidades. Para mais informações sobre o MSMQ, consultar [MSMQ: Perguntas Mais Frequentes](http://www.microsoft.com/windowsserver2003/techinfo/overview/msmqfaq.mspx).
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade com sucesso poderia fazer com que o sistema de um utilizador deixasse de responder e tivesse de ser reiniciado para restaurar a funcionalidade.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia tentar explorar a vulnerabilidade enviando um pacote PGM especialmente concebido para o efeito para um sistema afectado. O pacote poderia fazer com que o sistema afectado deixasse de responder até ser reiniciado.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
As estações de trabalho e servidores que tenham o MSMQ instalado ou estejam a executar uma aplicação suportada por PGM (que detecta pacotes PGM) estão mais susceptíveis.
  
**O que faz a actualização?**    
A actualização remove a vulnerabilidade validando correctamente o campo de comprimento de opção de pacotes PGM.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.
  
Vulnerabilidade de Fragmento Mal Formado no PGM - CVE-2008-1441  
---------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de negação de serviço em implementações do protocolo Pragmatic General Multicast (PGM) no Microsoft Windows XP, Windows Server 2003, Windows Vista e Windows Server 2008. O código de análise do protocolo não valida correctamente fragmentos PGM especialmente concebidos para o efeito e fará com que o sistema afectado deixe de responder até o ataque ter terminado.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2008-1441](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1441).
  
#### Factores Atenuantes para a Vulnerabilidade de Fragmento Mal Formado no PGM - CVE-2008-1441
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Em sistemas com o Windows XP ou o Windows Server 2003 instalado, o Pragmatic General Multicast (PGM) só está activado quando o Microsoft Message Queuing (MSMQ) 3.0 está instalado. O serviço MSMQ não é instalado por predefinição.  
-   Em sistemas que estão a executar o Windows Vista ou o Windows Server 2008, o Pragmatic General Multicast (PGM) só está activado quando o Microsoft Message Queuing (MSMQ) 4.0 está instalado e o PGM especificamente activado. O serviço MSMQ não é instalado por predefinição. Quando o serviço MSMQ está instalado, o protocolo PGM está disponível mas não está activado por predefinição.
  
#### Soluções Alternativas para a Vulnerabilidade de Fragmento Mal Formado no PGM - CVE-2008-1441
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.

O sistema retomará o funcionamento normal quando todos os pacotes especialmente concebidos para o efeito tenham sido processados ou o sistema tenha sido reiniciado. Após ter sido reiniciado, o sistema voltará a deixar de responder se o intruso continuar a enviar pacotes PGM especialmente concebidos para o efeito.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
As estações de trabalho e servidores que tenham o MSMQ instalado ou estejam a executar uma aplicação suportada por PGM (que detecta pacotes PGM) estão mais susceptíveis.
  
**O que faz a actualização?**    
A actualização remove a vulnerabilidade validando correctamente as opções de fragmento de pacotes PGM.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.

    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.

#### Informações de Implementação
  
**Instalação da actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.
  
Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro         | Descrição                                                                                                                                            |  
|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**        | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte o [Artigo 934307 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/934307).
  
**Verificar se a actualização foi aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização em **Iniciar pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
