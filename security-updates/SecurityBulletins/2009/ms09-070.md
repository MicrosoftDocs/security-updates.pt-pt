Security Bulletin

Microsoft Security Bulletin MS09-070 - Importante
=================================================

Vulnerabilidades nos Serviços de Federação do Active Directory poderiam permitir Execução Remota de Código (971726)
-------------------------------------------------------------------------------------------------------------------

Data de publicação: 8 de dezembro de 2009 | Updated: 9 de dezembro de 2009

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve duas vulnerabilidades no Microsoft Windows, comunicadas de forma privada. A mais grave destas vulnerabilidades poderia permitir a execução remota de código se um intruso enviasse um pedido HTTP especialmente concebido para o efeito a um servidor Web preparado para ADFS. Um intruso necessitaria de ser um utilizador autenticado para explorar qualquer uma destas vulnerabilidades.

Esta actualização de segurança está classificada como Importante para os lançamentos afectados do Windows Server 2003, Windows Server 2003 x64 Edition, Windows Server 2008 e Windows Server 2008 x64 Edition. Para mais informações consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve as vulnerabilidades corrigindo a validação que os servidores Web preparados para ADFS aplicam aos cabeçalhos de pedidos enviados por um cliente Web. Para mais informações sobre as vulnerabilidades, consulte a subsecção de Perguntas Mais Frequentes quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.**  A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais, ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a actualização na primeira oportunidade, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões conhecidas.** O [Artigo 971726 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/971726) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

**O que são os Serviços de Federação do Active Directory?**  
Os Serviços de Federação do Active Directory (ADFS) são baseados na emergente Arquitectura de Serviços Web suportada pela indústria e que se define nas especificações WS-\*. Os ADFS ajudam-no a utilizar um início de sessão único (SSO) para permitir que os utilizadores se autentiquem em múltiplas aplicações Web relacionadas enquanto durar uma sessão online. Tal pode ser feito pelos ADFS partilhando de forma segura os direitos de identidade digital e autorizações através de limites empresariais e de segurança.

**Como posso verificar se os ADFS estão instalados no meu servidor? **
Para verificar se os ADFS estão instalados no seu servidor, clique em **Iniciar**, clique em **Painel de controlo**, clique em **Adicionar ou remover programas**, faça duplo clique em **Componentes do Windows** e, em seguida, procure **Serviços de Federação** como um componente instalado.

**O que é o Windows Server 2003 R2?**  
O Windows Server 2003 R2 expande o sistema operativo Windows Server 2003, proporcionando uma forma mais eficiente de gerir e controlar o acesso a recursos locais e remotos enquanto se integra facilmente no seu ambiente Windows Server 2003 existente. O Windows Server 2003 R2 fornece uma plataforma da Web escalável e com funcionalidades de segurança melhoradas, interoperabilidade sem problemas com sistemas baseados em UNIX e permite novos cenários que incluem a gestão simplificada de servidores de ramal, gestão de acesso e identidade melhorada e gestão de armazenamento mais eficiente. O Windows Server 2003 R2 Datacenter Edition e o Windows Server 2003 R2 Enterprise Edition também proporcionam um novo licenciamento dinâmico que permite que os clientes obtenham ainda mais valor da virtualização do servidor.

Para obter mais informações sobre o Windows Server 2003 R2, consulte o site [Windows Server 2003 R2](http://www.microsoft.com/windowsserver2003/default.mspx).

**O que são as especificações WS-\* da Web?**

As especificações para uma Arquitectura de Serviços Web com base em normas da indústria, tais como o protocolo SOAP (protocolo simples de acesso a objectos), XML, Web Service Description Language e UDDI (Universal Description, Discovery, and Integration). A WS-\* proporciona uma base para fornecer soluções de negócio completas e interoperáveis a toda a empresa, incluindo a capacidade de gerir segurança e identidades federadas.

O modelo de serviços Web baseia-se na ideia de que os sistemas empresariais são escritos em idiomas diferentes, com diferentes modelos de programação, sendo a respectiva execução e o respectivo acesso efectuados a partir de muitos tipos de dispositivos diferentes. Os serviços Web são um meio de construir sistemas distribuídos que podem ligar-se e interagir entre si de forma fácil e eficiente através da Internet, independentemente do idioma em que estão escritos ou da plataforma em que são executados.

Vulnerabilidade de Spoofing de Início de Sessão Único em ADFS - CVE-2009-2508  
-----------------------------------------------------------------------------
  
<span></span>
Uma vulnerabilidade de spoofing nos Serviços de Federação do Active Directory poderia permitir que um intruso se fizesse passar por um utilizador autenticado se o intruso tivesse acesso a uma estação de trabalho e a um browser da Web recentemente utilizado pelo utilizador afectado para aceder a um Web site com possibilidade de início de sessão único.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2009-2508](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2508).
  
#### Factores Atenuantes para a Vulnerabilidade de Spoofing de Início de Sessão Único em ADFS - CVE-2009-2508
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso não pode seleccionar o utilizador pelo qual quer fazer-se passar. Um intruso apenas pode fazer-se passar por alguém cujo token de autenticação tenha conseguido obter (por exemplo, num local de acesso público à Internet).  
-   Um ataque só é possível antes de a sessão terminar no servidor. Por predefinição, as sessões permanecem abertas durante 600 minutos no servidor. Se um intruso tentasse reutilizar o token de autenticação de uma vítima depois de o período de tempo ter expirado, o ataque falharia.  
-   Um intruso deve ter acesso ao token de autenticação utilizado pelo utilizador anterior nesse terminal. No caso de um servidor preparado para HTTPS, o token de autenticação terá sido protegido em trânsito e o intruso deve ter acesso ao computador da vítima (por exemplo, num local de acesso público à Internet) para explorar a vulnerabilidade. Num local de acesso público à Internet, o utilizador pode iniciar sessão numa aplicação da Web com Início de Sessão Único (SSO) em ADFS e, depois, terminar a sessão da aplicação. Um intruso poderia utilizar esse computador para aceder à aplicação Web como sendo o utilizador anterior, mesmo que o utilizador tivesse terminado a sessão.
  
#### Soluções Alternativas para a Vulnerabilidade de Spoofing de Início de Sessão Único em ADFS - CVE-2009-2508
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Spoofing de Início de Sessão Único em ADFS - CVE-2009-2508
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de ocultação (spoofing). Um intruso que explorasse esta vulnerabilidade com sucesso poderia fazer-se passar por outro utilizador, permitindo que o intruso obtivesse acesso a um Web site, como o utilizador visado, que utilize a funcionalidade de início de sessão único nos Serviços de Federação do Active Directory (ADFS).
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade existe porque o ADFS não tem uma validação de gestão de sessão suficiente.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Em sistemas com acesso partilhado, como locais de acesso público à Internet, um intruso que explorasse esta vulnerabilidade com sucesso poderia aceder a recursos pertencentes a outro utilizador previamente autenticado. Este ataque apenas poderia ser executado dentro do limite de tempo de sessão configurado pelo administrador de ADFS que, por predefinição, é de 600 minutos.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia explorar esta vulnerabilidade acedendo a dados específicos da cache do browser e reutilizando-os para se autenticar na aplicação Web utilizando o início de sessão único (SSO) em ADFS. Depois de a sessão terminar no servidor, o ADFS desactivará a sessão autenticada no lado do servidor e já não permitirá que o intruso reutilize estas credenciais.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Os servidores Web que utilizam ADFS para fins de início de sessão único são afectados por esta vulnerabilidade. Esta vulnerabilidade não afecta directamente os sistemas cliente.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade eliminando correctamente as credenciais de autenticação quando o utilizador termina a sessão na rede.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.
  
Vulnerabilidade de Execução Remota de Código em ADFS - CVE-2009-2509  
--------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código em implementações dos Serviços de Federação do Active Directory (ADFS). A vulnerabilidade deve-se à validação incorrecta de cabeçalhos de pedidos quando um utilizador autenticado se liga a um servidor Web preparado para ADFS. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2009-2509](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2509).
  
#### Factores Atenuantes para a Vulnerabilidade de Execução Remota de Código em ADFS - CVE-2009-2509
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   O intruso deve ter credenciais de início de sessão válidas no servidor vulnerável. Um ataque apenas pode ser executado depois de ser efectuada uma autenticação com êxito no servidor que suporta ADFS.
  
#### Soluções Alternativas para a Vulnerabilidade de Execução Remota de Código em ADFS - CVE-2009-2509
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Execução Remota de Código em ADFS - CVE-2009-2509
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia executar acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity), que, por predefinição, está configurado com privilégios de conta do Serviço de Rede.
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade existe porque o ADFS não valida correctamente cabeçalhos de pedidos enviados por clientes autenticados.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse tirar partido desta vulnerabilidade poderia executar acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity). Por predefinição, o WPI está configurado com os privilégios da conta do Serviço de Rede. Os servidores IIS com páginas ASP cujas zonas da aplicação estão configuradas com um WPI que utilize uma conta com privilégios de administração poderiam ser afectados com mais gravidade do que os servidores IIS cuja zona da aplicação está configurada com as definições WPI predefinidas.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia tentar explorar a vulnerabilidade enviando um pedido HTTP especialmente concebido para o efeito para um servidor Web preparado para ADFS. Um intruso necessitaria de ser um utilizador autenticado para poder explorar esta vulnerabilidade.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Os servidores Web preparados para ADFS estão susceptíveis. Os sistemas cliente não são afectados por esta vulnerabilidade.
  
**O que faz a actualização?**    
Esta actualização corrige a validação que os servidores Web preparados para ADFS aplicam aos cabeçalhos de pedidos enviados por um cliente Web.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.

**Verificar se a Actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

-   **Verificação da Chave de Registo**

    Poderá também verificar os ficheiros instalados por esta actualização de segurança consultando as chaves de registo listadas na **Tabela de referência**, nesta secção.

    Estas chaves de registo poderão não conter uma lista completa dos ficheiros instalados. Além disso, estas chaves de registo poderão não ser criadas correctamente se um administrador ou um OEM integrar ou associar esta actualização de segurança aos ficheiros de origem de instalação do Windows.

#### Informações de Implementação
  
**Instalação da actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.
  
Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro         | Descrição                                                                                                                                            |  
|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**        | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte o [Artigo 934307 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/934307).
  
**Verificar se a actualização foi aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização em **Iniciar pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
