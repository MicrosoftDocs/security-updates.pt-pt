Security Bulletin

Microsoft Security Bulletin MS09-015 - Moderada
===============================================

Vulnerabilidade de Ameaça Mista no SearchPath poderia permitir Elevação de Privilégios (959426)
-----------------------------------------------------------------------------------------------

Data de publicação: 14 de abril de 2009 | Updated: 15 de abril de 2009

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança corrige uma vulnerabilidade divulgada publicamente na função SearchPath do Windows que poderia permitir elevação de privilégios se um utilizador transferisse um ficheiro especialmente concebido para o efeito para uma localização específica, e se depois abrisse uma aplicação que pudesse carregar o ficheiro sob determinadas circunstâncias.

Esta actualização de segurança está classificada como Moderada para todas as edições suportadas do Windows XP, Windows Server 2003, Windows Vista e Windows Server 2008. No Microsoft Windows 2000, esta actualização está classificada apenas como uma alteração defensiva profunda. Os detalhes encontram-se disponíveis na secção **Perguntas Mais Frequentes relativas a esta Actualização de Segurança**. Para mais informações consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade modificando a forma como o Windows carrega ficheiros a partir do ambiente de trabalho. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

Esta actualização de segurança também resolve a vulnerabilidade descrita previamente no [Aviso de Segurança da Microsoft 953818](http://technet.microsoft.com/security/advisory/953818).

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes ponderem a aplicação da actualização de segurança, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

**Questões conhecidas.** Nenhum

Perguntas mais frequentes relativas a esta actualização de segurança
--------------------------------------------------------------------

<span></span>
**Onde estão os pormenores sobre as informações do ficheiro?**  
Os pormenores sobre as informações do ficheiro podem ser encontrados no [Artigo 959426 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/959426).

**Como é que esta actualização de segurança está relacionada com o boletim MS09-014?**  
A Vulnerabilidade de Elevação de Privilégios de Ameaça Mista, [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540), descrita neste boletim de segurança, também afecta o Internet Explorer. A Microsoft recomenda que os utilizadores instalem também a actualização de segurança fornecida pelo boletim [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), Actualização de Segurança Cumulativa para o Internet Explorer (963027).

**Esta actualização contém outras alterações de segurança em termos de funcionalidade?**  
Sim. Para além das alterações indicadas na secção **Informações sobre as vulnerabilidades** deste boletim, esta actualização inclui uma alteração defensiva profunda que introduz uma nova API, chamada SetSearchPathMode.

**O que é que a alteração defensiva profunda do SetSearchPathMode permite?**  
Esta nova API permite a execução de um modo por-processo durante a utilização da função SearchPath para localizar ficheiros. Isto permite que as aplicações forcem a pesquisa no directório actual depois da pesquisa nas localizações da aplicação e do sistema.

**Se aplicar apenas esta actualização de segurança, estarei completamente protegido contra a ameaça mista descrita em CVE-2008-2540?**  
Não. Terá também de activar a protecção defensiva profunda fornecida pela Actualização de Segurança Cumulativa para o Internet Explorer, [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). No entanto, deve verificar se a activação desta protecção defensiva profunda contra a ameaça mista é apropriada para seu ambiente.

Para activar a protecção defensiva profunda no Internet Explorer, active a chave de controlo de funcionalidade SearchPath descrita na secção Perguntas Mais Frequentes relativas a esta Actualização de Segurança, no boletim [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). Para mais informações sobre a ameaça mista, consulte o número de Vulnerabilidade e Exposição Comum [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).

**Por que razão está o Microsoft Windows 2000 listado na tabela de Software Afectado sem ter um impacto de segurança e uma classificação de gravidade?**
O Microsoft Windows 2000 está listado sem ter um impacto de segurança e uma classificação de gravidade porque esta actualização não apresenta a mesma correcção para o Microsoft Windows 2000 que apresenta para outros sistemas operativos Windows suportados. Em vez disso, esta actualização tem apenas uma alteração defensiva profunda específica do Microsoft Windows 2000.

**Por que razão a correcção para a ameaça mista no Microsoft Windows 2000 é diferente da correcção apresentada para outras plataformas? **
A arquitectura para implementar a função SetSearchPathMode não existe no sistema Microsoft Windows 2000, impossibilitando a eliminação da vulnerabilidade no Microsoft Windows 2000. Para o fazer, seria necessário reconstruir a arquitectura de uma porção significativa do sistema operativo Microsoft Windows 2000 e não só do componente afectado. O resultado final desses esforços seria suficientemente incompatível com o Microsoft Windows 2000, de tal forma que não haveria nenhuma garantia de que as aplicações concebidas para executar o Microsoft Windows 2000 continuassem a funcionar no sistema com arquitectura reconstruída. No entanto, a Microsoft não tem conhecimento de quaisquer vectores de ataque válidos dirigidos a vulnerabilidade no Microsoft Windows 2000.

Devido a estas alterações ao nível da arquitectura, para o Microsoft Windows 2000, deve activar a protecção defensiva profunda específica do Microsoft Windows 2000 utilizando a chave de registo HKLM\\System\\CCS\\control\\Session Manager\\SafeDLLSearchMode. Para obter instruções, consulte o [Artigo 959426 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/959426). Para mais informações, consulte o artigo MSDN sobre o [Pedido de Pesquisa Dynamic Link Library (DLL)](http://msdn.microsoft.com/en-us/library/ms682586.aspx).

**O que é a função SearchPath?**  
A função SearchPath é utilizada para procurar um ficheiro específico num caminho especificado. Para mais informações, consulte o artigo MSDN sobre a [Função SearchPath](http://msdn.microsoft.com/en-us/library/aa365527.aspx).

**O que é a função LoadLibrary?**  
A função LoadLibrary carrega o módulo especificado no espaço de endereço do processo de chamada. O módulo especificado pode fazer com que sejam carregados outros módulos. Para mais informações, consulte o artigo MSDN sobre a [Função LoadLibrary](http://msdn.microsoft.com/en-us/library/ms684175(vs.85).aspx).

Vulnerabilidade de Elevação de Privilégios de Ameaça Mista - CVE-2008-2540  
--------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de elevação de privilégios de ameaça mista na forma como a função SearchPath no Windows localiza e abre ficheiros no sistema. Um intruso poderia explorar a vulnerabilidade convencendo um utilizador a transferir um ficheiro especialmente concebido para o efeito para uma localização específica, e então abrir uma aplicação que pudesse carregar o ficheiro sob determinadas circunstâncias.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540).
  
#### Factores Atenuantes para a Vulnerabilidade de Elevação de Privilégios de Ameaça Mista - CVE-2008-2540
  
A Microsoft não identificou quaisquer factores atenuantes para esta vulnerabilidade.
  
#### Soluções Alternativas para a Vulnerabilidade de Elevação de Privilégios de Ameaça Mista - CVE-2008-2540
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Elevação de Privilégios de Ameaça Mista - CVE-2008-2540
  
**Qual é a abrangência desta vulnerabilidade?**    
Trata-se de uma vulnerabilidade de elevação de privilégios de ameaça mista. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado remotamente. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios.
  
**O que provoca a vulnerabilidade?**    
O Internet Explorer poderia abrir um ficheiro especialmente concebido para o efeito a partir da área de trabalho devido à forma como as bibliotecas de aplicações são localizadas no sistema Windows. Uma ameaça mista em que podem ser transferidos ficheiros para um sistema do utilizador sem aviso poderia fazer com que o Internet Explorer carregasse involuntariamente esse ficheiro transferido a partir da área de trabalho em vez do sistema Windows.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade poderia executar código arbitrário. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia criar um ficheiro especialmente concebido para o efeito e então convencer um utilizador a transferir o ficheiro para o ambiente de trabalho. O utilizador teria de iniciar uma aplicação que abrisse esse ficheiro especialmente concebido para o efeito. No entanto, um intruso não teria forma de obrigar os utilizadores a transferir tais ficheiros e a colocar os ficheiros no ambiente de trabalho.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Todos os sistemas em que os utilizadores transferem ficheiros e iniciam aplicações, tal como estações de trabalho e servidores de terminais, estão susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização modifica a ordem de procura para bibliotecas de aplicações, de tal forma que, em primeiro lugar, é feita uma procura no directório do sistema antes de ser feita nos directórios de trabalho actuais. A actualização também fornece uma alteração defensiva profunda, uma nova API chamada SetSearchPathMode, para ajudar utilizadores avançados a definir a ordem de procura desejada para a função SearchPath. Para mais informações sobre esta alteração defensiva profunda, consulte também a secção **Perguntas Mais Frequentes relativas a esta Actualização de Segurança**.
  
**Por que razão é atribuído a esta vulnerabilidade e à Vulnerabilidade de Execução Remota de Código de Ameaça Mista o mesmo número de CVE (CVE-2008-2540) no boletim MS09-014, Actualização de Segurança Cumulativa para o Internet Explorer (963027)?**  
Por um lado, o [CVE-2008-2540](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2540) afecta tanto o Internet Explorer 7 para o Windows XP Service Pack 2 e Windows XP Service Pack 3 como o Internet Explorer 7 para o Windows XP Professional x64 Edition e Windows XP Professional x64 Edition Service Pack 2 e, por outro lado, afecta a função SearchPath do Windows. A Microsoft recomenda que os utilizadores instalem também a actualização de segurança fornecida pelo boletim [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659), Actualização de Segurança Cumulativa para o Internet Explorer (963027). A alteração defensiva profunda para esta vulnerabilidade nesta actualização, a API chamada SetSearchPathMode, também corresponde à correcção no boletim [MS09-014](http://go.microsoft.com/fwlink/?linkid=146659). Consulte também nas Perguntas Mais Frequentes "Esta actualização contém outras alterações de segurança em termos de funcionalidade?", na secção de **Perguntas Mais Frequentes relativas a esta Actualização de Segurança.**
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**

**Instalação da actualização**

Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.

Caso tenha instalado anteriormente um hotfix para actualizar um destes ficheiros, o programa de instalação copia os ficheiros RTMQFE, SP1QFE ou SP2QFE para o sistema. Caso contrário, o programa de instalação copia os ficheiros RTMGDR, SP1GDR ou SP2GDR para o sistema. As actualizações de segurança podem não conter todas as variantes destes ficheiros. Para mais informações sobre este comportamento, consulte o [Artigo 824994 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824994).

Para mais informações sobre o programa de instalação, visite o [Web site Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

**Verificar se a actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação.
