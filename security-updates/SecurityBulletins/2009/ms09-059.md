Security Bulletin

Microsoft Security Bulletin MS09-059 - Importante
=================================================

Vulnerabilidade no Local Security Authority Subsystem Service poderia permitir Negação de Serviço (975467)
----------------------------------------------------------------------------------------------------------

Data de publicação: 13 de outubro de 2009 | Updated: 14 de outubro de 2009

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade no Microsoft Windows, comunicada de forma privada. A vulnerabilidade poderia permitir negação de serviço se um intruso enviasse um pacote malicioso concebido para o efeito durante o processo de autenticação NTLM.

Esta actualização de segurança está classificada como Importante para todas as edições suportadas do Windows XP, Windows Server 2003, Windows Vista, Windows Server 2008 e Windows 7. As edições suportadas do Windows XP e do Windows Server 2003 apenas são afectadas se tiver sido previamente instalada a actualização não relacionada com segurança descrita no boletim KB968389. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade implementando validação adicional de conjuntos de valores específicos utilizados no processo de autenticação. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.**  A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais, ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a actualização na primeira oportunidade, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões conhecidas.** O [Artigo 975467 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/975467) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

Perguntas mais frequentes relativas a esta actualização de segurança
--------------------------------------------------------------------

<span></span>
**Que sistemas são afectados por esta vulnerabilidade? **
Esta vulnerabilidade afecta os sistemas Windows Vista, Windows Server 2008, Windows 7 e Windows Server 2008 R2. Além do mais, também afecta os sistemas Windows XP e Windows Server 2003 que instalaram a actualização KB968389, uma actualização não relacionada com segurança que implementa a Protecção Alargada.

**O meu computador fica vulnerável se não instalar a actualização KB968389?**  
As edições suportadas do Windows XP e Windows Server 2003 que não têm a KB968389 instalada não são vulneráveis. As edições suportadas do Windows Vista, Windows Server 2008, Windows 7, e Windows Server 2008 R2 que não têm a KB968389 instalada são vulneráveis.

**Por que razão me está a ser oferecida a actualização KB968389 apenas através do Windows Update ou Microsoft Update?**  
Se obtiver a actualização KB968389 através do Windows Update ou Microsoft Update depois da publicação desta actualização de segurança, a actualização de segurança KB975467 será automaticamente instalada. Após a instalação com sucesso da actualização KB968389, as actualizações KB968389 e a actualização KB975467 no boletim MS09-059, serão listadas como software instalado.

Se instalou a actualização KB968389 antes da publicação desta actualização de segurança, o Windows Update e o Microsoft Update oferecerão a actualização de segurança KB975467 para resolver a vulnerabilidade.

**O MBSA suporta detecção para esta actualização de segurança no Windows 7 e Windows Server 2008 R2?**  
O MBSA suporta detecção para esta actualização de segurança no Windows 7 e Windows Server 2008 R2 como parte da versão 2.1.1 do MBSA, disponível a partir da data de Disponibilidade Geral destes sistemas operativos. Para obter mais informações, visite [MBSA](http://www.microsoft.com/mbsa).

Vulnerabilidade de Excesso de Número Inteiro no Local Security Authority Subsystem Service - CVE-2009-2524  
----------------------------------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de negação de serviço no Microsoft Windows Local Security Authority Subsystem Service (LSASS), devido ao processamento inadequado de pacotes mal formados durante a autenticação NTLM. Um intruso poderia criar pedidos anónimos de autenticação NTLM especialmente concebidos para o efeito o que faria com que o serviço LSASS deixasse de responder e reiniciaria o computador.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2009-2524](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2009-2524).
  
#### Factores Atenuantes para a Vulnerabilidade de Excesso de Número Inteiro no Local Security Authority Subsystem Service - CVE-2009-2524
  
A Microsoft não identificou quaisquer factores atenuantes para esta vulnerabilidade.
  
#### Soluções Alternativas para a Vulnerabilidade de Excesso de Número Inteiro no Local Security Authority Subsystem Service - CVE-2009-2524
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Desinstale a actualização KB968389 de computadores com o Windows XP ou Windows Server 2003**
  
    1.  Clique em **Iniciar** e, em seguida, clique em **Painel de Controlo** e abra **Adicionar ou Remover Programas**.  
    2.  Seleccione **KB968389** e clique em **Desinstalar**.
  
    **Impacto da solução alternativa.** Os computadores com o Windows XP e Windows Server 2003 não beneficiarão da Protecção de Autenticação Alargada enquanto esta KB estiver desactivada.
  
    **Como anular a solução alternativa.** Instale a KB968389 transferindo o pacote de instalação adequado para o seu sistema operativo do [Artigo 936389 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/default.aspx/kb/968389?wa=wsignin1.0).
  
-   **Para ajudar a proteger o seu sistema contra tentativas de exploração dessa vulnerabilidade com base na rede, active a filtragem avançada TCP/IP em sistemas que suportem essa funcionalidade**
  
    É possível activar a filtragem TCP/IP avançada para bloquear todo o tráfego de entrada não solicitado. Para obter informações adicionais sobre como configurar a filtragem TCP/IP, consulte o [artigo 309798 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/309798).
  
-   **Para ajudar a proteger contra tentativas baseadas em rede de explorar essa vulnerabilidade, utilize uma firewall pessoal, tal como a Firewall de Ligação à Internet**
  
    Todas as edições suportadas do Windows Vista incluem a Firewall do Windows, uma firewall de duas vias que está activada por defeito.
  
    Para todas as edições suportadas do Windows Server 2003, use a Firewall de Ligação à Internet para ajudar a proteger a sua ligação à Internet, bloqueando tráfego de entrada não solicitado. A Microsoft recomenda que bloqueie todas as comunicações de entrada não solicitadas da Internet.
  
    Para activar a funcionalidade Firewall do Windows utilizando o Assistente de Configuração de Rede, siga os seguintes passos:
  
    1.  Clique em **Iniciar** e, em seguida, clique em **Painel de Controlo**.  
    2.  Faça duplo clique em **Ligações de Rede** e clique em **Modificar definições da Firewall do Windows**.  
    3.  No separador **Geral**, certifique-se de que o valor **Ligado (recomendado)** está seleccionado. Isto irá activar a Firewall do Windows.  
    4.  Quando a Firewall do Windows estiver activada, seleccione **Não permitir excepções** para proibir toda a recepção de tráfego.
  
    Para configurar a Firewall de Ligação à Internet manualmente para uma ligação nos sistemas Windows Server 2003, siga os seguintes passos:
  
    1.  Clique em **Iniciar** e, em seguida, clique em **Painel de Controlo**.  
    2.  Na vista por categoria predefinida, clique em **Ligações de Rede e de Internet** e, em seguida, clique em **Ligações de Rede**.  
    3.  Clique com o botão direito do rato na ligação na qual pretende activar a Firewall de Ligação à Internet e, em seguida, clique em **Propriedades**.  
    4.  Clique no separador **Avançadas**.  
    5.  Clique para seleccionar a caixa de verificação **Proteger o meu computador e a rede limitando ou impedindo o acesso a este computador a partir da Internet** e, em seguida, clique em **OK**.
  
    **Nota** Se pretender activar a utilização de determinados programas e serviços para que comuniquem através da firewall, clique em **Definições** no separador **Avançadas** e, em seguida, seleccione os programas, protocolos e serviços necessários.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Excesso de Número Inteiro no Local Security Authority Subsystem Service - CVE-2009-2524
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de "negação de serviço". Um frame de autenticação especialmente concebido para o efeito poderia permitir que um intruso não autenticado deixasse de responder no processo de servidor LSASS e reiniciasse o computador.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade está presente porque a implementação NTLM do Windows no LSASS não processa correctamente pacotes mal formados específicos durante o processo de autenticação. Um pacote mal formado específico poderia resultar num fluxo insuficiente de números inteiros no processo de LSASS.
  
**O que é o LSASS?**    
O serviço LSASS (Local Security Authority Subsystem Service) proporciona uma interface para gestão da segurança local, autenticação de domínios e processos do serviço Active Directory. Este serviço processa a autenticação do cliente e do servidor. O LSASS também inclui funcionalidades que são utilizadas para suportar utilitários Active Directory.
  
**O que é o NTLM?**    
O protocolo de autenticação de Pergunta/Resposta do Windows (NT Lan Manager ou NTLM) é o protocolo de autenticação utilizado em redes que incluem sistemas que executam o sistema operativo Windows e em sistemas independentes. As credenciais NTLM são baseadas em dados obtidos durante o processo interactivo de início de sessão e consiste num nome de domínio, nome de utilizador, e hash único da palavra-passe do utilizador. O NTLM utiliza um protocolo encriptado de pergunta/resposta para autenticar um utilizador sem enviar a palavra-passe do utilizador através da rede. Em vez disso, o sistema que solicita autenticação tem de executar um cálculo que prove que tem acesso às credenciais NTLM seguras.
  
A autenticação interactiva NTLM através de uma rede envolve normalmente dois sistemas: um sistema cliente, onde o utilizador solicita autenticação, e um controlador de domínio, onde as informações relacionadas com a palavra-passe do utilizador são mantidas. A autenticação não-interactiva, que pode ser exigida para permitir que um utilizador com sessão iniciada aceda a um recurso, tal como uma aplicação de servidor, normalmente envolve três sistemas: um cliente, um servidor, e um controlador de domínio que faz o cálculo de autenticação em nome do servidor.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse tirar partido desta vulnerabilidade poderia fazer com que um sistema afectado reiniciasse automaticamente.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia explorar esta vulnerabilidade enviando um pacote malicioso durante o processo de autenticação NTLM.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
Os servidores que permitem que os clientes se autentiquem utilizando NTLM estão mais susceptíveis.
  
**O que faz a actualização?**    
A actualização garante que o LSASS processa correctamente valores específicos utilizados no processo de autenticação.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.
