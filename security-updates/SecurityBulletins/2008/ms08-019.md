Security Bulletin

Microsoft Security Bulletin MS08-019 - Importante
=================================================

Vulnerabilidades no Microsoft Visio poderiam permitir Execução Remota de Código (949032)
----------------------------------------------------------------------------------------

Data de publicação: 8 de abril de 2008 | Updated: 23 de abril de 2008

**actualizada:** 1.5

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança corrige vulnerabilidades no Microsoft Visio comunicadas de forma privada que poderiam permitir execução remota de código se um utilizador abrisse um ficheiro Visio especialmente concebido para o efeito. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Esta actualização de segurança está classificada como Importante para o Microsoft Visio 2002 Service Pack 2, Microsoft Visio 2003 Service Pack 2, Microsoft Visio 2003 Service Pack 3, Microsoft Visio 2007 e Microsoft Visio 2007 Service Pack 1. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

Esta actualização de segurança resolve estas vulnerabilidades modificando a forma como o Microsoft Visio efectua validações ao abrir ficheiros Visio. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade.

**Questões conhecidas.** O [Artigo 949032 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/949032) documenta as questões actualmente conhecidas e soluções recomendadas que os clientes podem experimentar quando instalam esta actualização de segurança. Quando as questões actualmente conhecidas e soluções recomendadas software dizem respeito apenas às edições específicas deste software, este artigo não fornece ligações a outros artigos.

#### Software Afectado e Software Não Afectado

O software que se segue foi testado para determinar quais as versões ou edições afectadas. As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Para determinar o ciclo de vida de suporte da versão ou edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Perguntas mais frequentes relativas a esta actualização de segurança  
--------------------------------------------------------------------
  
<span></span>
**Por que razão decidiu a Microsoft efectuar revisões a este boletim a 18 de Abril de 2008?**    
Este boletim foi revisto para corrigir questões de detecção e instalação, que tinham afectado utilizadores de componentes do Microsoft Office 2007. Anteriormente, a actualização oferecia e instalava a actualização para o Visio 2007 para utilizadores que não tinham o Visio 2007 instalado nos seus sistemas. A Microsoft corrigiu a questão e actualizou este boletim para fazer referência à alteração. Essa alteração referia-se unicamente a uma alteração na detecção, que não afecta os ficheiros contidos na actualização inicial. Os sistemas com o Visio 2007 que já aplicaram a actualização não receberão a actualização novamente e os utilizadores que já instalaram a actualização não terão de a reaplicar nos seus sistemas.
  
**Qual é o impacto de segurança associado à instalação da actualização se eu não tiver o Visio 2007 instalado no meu sistema? O que devo fazer?**    
Nenhum. Instalar incorrectamente esta actualização num sistema que não tem o Visio 2007 instalado não irá aumentar o risco de segurança desse sistema. Os utilizadores podem desinstalar a actualização se esta já tiver sido aplicada. Contudo, a desinstalação da actualização não é necessária, uma vez que a actualização não terá um impacto negativo na segurança ou desempenho de um sistema.
  
**Utilizo o Microsoft Office 2003 Service Pack 2. Estão incluídas funcionalidades de segurança adicionais nesta actualização?**    
Sim, como parte do modelo de prestação de serviços do Microsoft Office 2003, quando os utilizadores do Microsoft Office 2003 Service Pack 2 instalam esta actualização, os seus sistemas serão actualizados para funcionalidade de segurança inicialmente lançada com o Microsoft Office 2003 Service Pack 3. Todas as actualizações disponibilizadas depois de 1 de Janeiro de 2008 para o Microsoft Office 2003 Service Pack 2 incluirão estas funcionalidades de segurança, introduzidas no [Microsoft Office 2003 Service Pack 3](http://www.microsoft.com/downloads/details.aspx?familyid=e25b7049-3e13-433b-b9d2-5e3c1132f206&displaylang=en). Esta actualização foi testada minuciosamente, mas tal como em todas as actualizações, recomendamos que os utilizadores executem os testes apropriados no respectivo ambiente e na configuração dos seus sistemas. Para mais informações sobre esta questão, consulte o [Artigo 951646 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/951646).

Os pormenores sobre as informações do ficheiro podem ser encontrados no [Artigo 949032 da Base de Dados de Conhecimento.](http://support.microsoft.com/kb/949032)

Vulnerabilidade em Cabeçalhos de Objectos Visio - CVE-2008-1089  
---------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código na forma como o Microsoft Visio valida dados de cabeçalhos de objectos especialmente concebidos para o efeito. Um intruso poderia explorar a vulnerabilidade enviando um ficheiro mal formado que poderia estar incluído como anexo numa mensagem de correio electrónico ou alojado num Web site corrompido ou especialmente concebido para o efeito.
  
Se um utilizador tivesse sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com direitos de utilização reduzidos no sistema podem ser menos afectados do que os utilizadores que trabalham com direitos de utilização de administrador.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-1089](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1089).
  
#### Factores Atenuantes para a Vulnerabilidade em Cabeçalhos de Objectos Visio - CVE-2008-1089
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   No cenário de um ataque baseado na Web, um intruso poderia alojar um Web site contendo uma página Web que fosse utilizada para explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos ou anúncios fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Contudo, em qualquer dos casos, o intruso não teria forma de forçar os utilizadores a visitar estes Web sites. Em vez disso, um intruso teria de convencer os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico, ou num programa de mensagens instantâneas, que levasse o utilizador para o Web site do intruso.  
-   Um intruso que explorasse com sucesso esta vulnerabilidade poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.  
-   A vulnerabilidade não pode ser automaticamente explorada através de correio electrónico. Para um ataque poder ser bem sucedido, é necessário que um utilizador abra um anexo recebido numa mensagem de correio electrónico.
  
#### Soluções Alternativas para a Vulnerabilidade em Cabeçalhos de Objectos Visio - CVE-2008-1089
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   Utilize o Microsoft Visio 2003 Viewer ou o Microsoft Visio 2007 Viewer para abrir e visualizar ficheiros Visio. O Microsoft Visio 2003 Viewer e o Microsoft Visio 2007 Viewer não são afectados por esta questão.  
-   Não abra ou guarde ficheiros Visio que tenha recebido de fontes não fidedignas, ou que recebeu de fontes fidedignas mas que de alguma forma não correspondem ao que é habitual. É possível tirar partido desta vulnerabilidade quando um utilizador abre um ficheiro especialmente concebido para o efeito.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade em Cabeçalhos de Objectos Visio - CVE-2008-1089
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado remotamente. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios.
  
**O que provoca a vulnerabilidade?**    
O Microsoft Visio não valida correctamente dados de cabeçalhos de objectos ao abrir ficheiros Visio.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia executar código arbitrário em nome do utilizador com sessão iniciada. Se um utilizador tiver sessão iniciada com direitos de utilizador administrativos, um intruso poderia obter controlo total sobre o sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Esta vulnerabilidade requer que um utilizador abra um ficheiro Visio mal formado com uma versão afectada do Microsoft Visio.
  
Num cenário de ataque com base em correio electrónico, um intruso poderia explorar a vulnerabilidade enviando um ficheiro Visio mal formado ao utilizador e convencendo-o a abrir esse ficheiro.
  
Num cenário de ataque com base na Web, um intruso teria de alojar um Web site contendo um ficheiro Visio que fosse utilizado para tentar explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Um intruso não conseguiria obrigar os utilizadores a visitarem um Web site especialmente concebido para o efeito. Em vez disso, um intruso teria de persuadir os utilizadores a visitar o Web site, normalmente conseguindo que clicassem numa ligação que os levasse ao site do intruso.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
Os sistemas em que o Microsoft Visio é utilizado estão mais susceptíveis. Os servidores podem estar mais sujeitos a risco se os administradores permitirem que os utilizadores iniciem sessão nos servidores e executem programas. Contudo, os procedimentos recomendados desencorajam essa acção.
  
**O que faz a actualização?**    
A actualização remove a vulnerabilidade executando a validação ao atribuir o espaço para os dados de cabeçalhos de objectos ao abrir ficheiros Visio.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez. Este boletim de segurança resolve a vulnerabilidade divulgada em meios privados e outras questões descobertas através de investigações internas.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.
  
Vulnerabilidade de Validação de Memória no Visio – CVE-2008-1090  
----------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código na forma como o Microsoft Visio valida atribuições de memória ao carregar ficheiros .DXF especialmente concebidos para o efeito do disco para a memória. Um intruso poderia explorar a vulnerabilidade enviando um ficheiro mal formado que poderia estar incluído como anexo numa mensagem de correio electrónico ou alojado num Web site corrompido ou especialmente concebido para o efeito.
  
Se um utilizador tivesse sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com direitos de utilização reduzidos no sistema podem ser menos afectados do que os utilizadores que trabalham com direitos de utilização de administrador.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-1090](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1090).
  
#### Factores Atenuantes para a Vulnerabilidade de Validação de Memória no Visio - CVE-2008-1090
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Por predefinição, os ficheiros de desenho AutoCAD (extensão de ficheiro .DXF) não são registados pelo Visio; assim sendo, um ficheiro .DXF especialmente concebido para o efeito não seria aberto clicando duas vezes sobre ele. Um intruso teria de convencer um utilizador a abrir um ficheiro .DXF executando o Visio, seleccionando Abrir e, de seguida, alterando o tipo de ficheiro predefinido de "Todos os ficheiros Visio" para " Desenho AutoCAD".  
-   No cenário de um ataque baseado na Web, um intruso poderia alojar um Web site contendo uma página Web que fosse utilizada para explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos ou anúncios fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Contudo, em qualquer dos casos, o intruso não teria forma de forçar os utilizadores a visitar estes Web sites. Em vez disso, um intruso teria de convencer os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico, ou num programa de mensagens instantâneas, que levasse o utilizador para o Web site do intruso.

Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.  
-   A vulnerabilidade não pode ser automaticamente explorada através de correio electrónico. Para um ataque poder ser bem sucedido, é necessário que um utilizador abra um anexo recebido numa mensagem de correio electrónico.
  
#### Soluções Alternativas para a Vulnerabilidade de Validação de Memória no Visio - CVE-2008-1090
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização.     **Para Windows Vista**  
    Takeown.exe /f "%ProgramFiles%\\microsoft office\\visio11\\dll\\dwgdp.dll"

#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Validação de Memória no Visio - CVE-2008-1090
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado remotamente. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios.
  
**O que provoca a vulnerabilidade?**    
O Microsoft Visio não valida adequadamente atribuições de memória ao carregar ficheiros .DTX especialmente concebidos para o efeito do disco para a memória.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia executar código arbitrário em nome do utilizador com sessão iniciada. Se um utilizador tiver sessão iniciada com direitos de utilizador administrativos, um intruso poderia obter controlo total sobre o sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Esta vulnerabilidade requer que um utilizador abra um ficheiro .DXF mal formado com uma versão afectada do Microsoft Visio.
  
Num cenário de ataque com base em correio electrónico, um intruso poderia explorar a vulnerabilidade enviando um ficheiro Visio mal formado ao utilizador e convencendo-o a abrir esse ficheiro.
  
Num cenário de ataque com base na Web, um intruso teria de alojar um Web site contendo um ficheiro Visio que fosse utilizado para tentar explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Um intruso não conseguiria obrigar os utilizadores a visitarem um Web site especialmente concebido para o efeito. Em vez disso, um intruso teria de persuadir os utilizadores a visitar o Web site, normalmente conseguindo que clicassem numa ligação que os levasse ao site do intruso.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
Os sistemas em que o Microsoft Visio é utilizado estão mais susceptíveis. Os servidores podem estar mais sujeitos a risco se os administradores permitirem que os utilizadores iniciem sessão nos servidores e executem programas. Contudo, os procedimentos recomendados desencorajam essa acção.
  
**O que faz a actualização?**    
A actualização remove a vulnerabilidade executando uma validação adequada de atribuições de memória ao carregar ficheiros Visio do disco para a memória.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez. Este boletim de segurança resolve a vulnerabilidade divulgada em meios privados e outras questões descobertas através de investigações internas.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.
