Security Bulletin

Microsoft Security Bulletin MS08-006 - Importante
=================================================

Vulnerabilidade no Internet Information Services poderia permitir Execução Remota de Código (942830)
----------------------------------------------------------------------------------------------------

Data de publicação: 12 de fevereiro de 2008 | Updated: 20 de fevereiro de 2008

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização importante resolve uma vulnerabilidade comunicada de forma privada no Internet Information Services (IIS). Existe uma vulnerabilidade de execução remota de código na forma como o IIS gere a introdução de dados em páginas Web ASP. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia executar acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity). Por predefinição, o WPI está configurado com os privilégios da conta do Serviço de Rede. Os servidores IIS com páginas ASP cujas zonas da aplicação estão configuradas com um WPI que utilize uma conta com privilégios de administração poderiam ser afectados com mais gravidade do que os servidores IIS cuja zona da aplicação está configurada com as definições WPI predefinidas.

A actualização de segurança está classificada como importante para o Microsoft Internet Information Services em todas as edições suportadas do Windows XP e Windows Server 2003. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

Para mais informações sobre estas vulnerabilidades, consulte a subsecção de Perguntas mais frequentes na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade.

**Questões conhecidas.** Nenhum

As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Vulnerabilidade ASP - CVE-2008-0075  
-----------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código na forma como o Internet Information Services processa a introdução de dados em páginas Web ASP. Um intruso poderia explorar a vulnerabilidade introduzindo dados maliciosos numa página ASP de um Web site. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia efectuar quaisquer acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity), que, por predefinição, está configurado com privilégios de conta do Serviço de Rede.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-0075](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-0075).
  
#### Factores Atenuantes para a Vulnerabilidade ASP - CVE-2008-0075
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   O IIS 5.1 não faz parte da instalação predefinida do Windows XP Professional Service Pack 2.  
-   Nas edições suportadas do Windows Server 2003, o IIS não está instalado nem activado por predefinição.  
-   Nas edições suportadas do Windows Server 2003, se o IIS estiver activado, o ASP clássico não está instalado nem activado por predefinição.  
-   Nas edições suportadas do Windows Server 2003, se o IIS estiver activado e for usado o ASP clássico, um intruso que conseguisse explorar esta vulnerabilidade só poderia obter privilégios de conta do Serviço de Rede predefinidos. Por predefinição, os privilégios de conta do Serviço de Rede têm os mesmos direitos de utilizador de um utilizador autenticado.  
-   O ASP.NET não é afectado por esta vulnerabilidade. Os clientes que tenham apenas o ASP.NET instalado e não o ASP não estão susceptíveis a esta vulnerabilidade.
  
#### Soluções Alternativas para a Vulnerabilidade ASP - CVE-2008-0075
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **No Windows Server 2003, desactive o ASP clássico**
  
    Para desactivar o ASP clássico, siga estes passos:
  
    1. Clique em **Iniciar**, clique em **Ferramentas administrativas** e depois clique em **Internet Information Services (IIS) Manager**.
  
    2. Clique no **símbolo de adição** (+) junto do nome do servidor e depois clique na pasta **Extensões do Serviço Web**.
  
    3. Clique para realçar **Páginas de Servidor Activas** no painel direito e depois clique em **Proibir.**
  
    **Impacto da solução alternativa.** Os utilizadores não poderão utilizar páginas ASP clássicas.
  
    **Como anular esta solução alternativa.** Para anular a solução alternativa, siga os passos 1 a 3 acima e clique em **Permitir** quando seleccionar as **Páginas de Servidor Activas** (ASP).
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade ASP - CVE-2008-0075
  
**Qual é a abrangência desta vulnerabilidade?**  
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia executar acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity), que, por predefinição, está configurado com privilégios de conta do Serviço de Rede.
  
**O que provoca a vulnerabilidade?**  
A vulnerabilidade é causada pela forma como o Internet Information Services processa incorrectamente introduções de dados em páginas ASP.
  
**Como poderia um intruso utilizar a vulnerabilidade?**  
Um intruso que conseguisse tirar partido desta vulnerabilidade poderia efectuar acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity), que, por predefinição, está configurado com privilégios de conta do Serviço de Rede. Os serviços configurados com privilégios de conta do Serviço de Rede obtêm acesso ao nível de um utilizador autenticado, e não acesso ao nível de administrador. Para mais informações, consulte [Conta do Serviço de Rede](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx). Os servidores IIS cuja zona da aplicação está configurada com um WPI que utilize uma conta com privilégios de administração poderiam ser afectados com mais gravidade do que os servidores IIS cuja zona da aplicação está configurada com as definições WPI predefinidas.
  
A conta do Serviço de Rede utiliza as credenciais do computador quando o autentica de forma remota, mas tem um nível de privilégios reduzido e, por conseguinte, não tem privilégios de administrador local. Para mais informações, consulte a [NetworkService Account](http://msdn2.microsoft.com/en-us/library/ms684272(vs.85).aspx) (Conta de Serviço de Rede), [IIS and Built-in Accounts (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/3648346f-e4f5-474b-86c7-5a86e85fa1ff.mspx) (Contas IIS e integradas (IIS 6.0)), [Configuring Worker Process Identities (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/76be66e2-1db1-432e-95be-14ff59c2ec75.mspx) (Configurar o Worker Process Identities (IIS 6.0)) e [Configuring Application Pools in IIS 6.0](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/2a231dcb-d786-4b6d-b3ba-bda04061a5e7.mspx) (IIS 6.0) (Configurar zonas de aplicações no IIS 6.0 (IIS 6.0)).
  
**Como poderia um intruso explorar a vulnerabilidade?**  
Um intruso poderia introduzir dados especialmente concebidos para o efeito numa página ASP. A página ASP poderia já existir ou ser carregada por um utilizador num ambiente alojado.
  
**O que são processos de trabalho para IIS?**  
Os processos de trabalho são processos que executam as tarefas de processamento baseadas no servidor para um servidor Web**.** Para obter informações sobre processos de trabalho e o modo como são implementados no IIS 6.0, consulte [Worker Processes (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/29f53968-0115-451f-b26d-5ad58d87b5d1.mspx). Para obter mais informações sobre processos de trabalho e como executam aplicações baseadas na Web isoladamente, consulte [Worker Process Isolation Mode (IIS 6.0)](http://www.microsoft.com/technet/prodtechnol/windowsserver2003/library/iis/333f3410-b566-49bc-8399-57bcc404db8c.mspx) (Modo de isolamento de processos de trabalho (IIS 6.0)).
  
**O que é o ASP?**  
As Páginas de Servidor Activas da Microsoft (ASP) são uma tecnologia de execução de scripts baseados no servidor, que pode ser usada para criar aplicações Web dinâmicas e interactivas. Uma página ASP é uma página HTML que contém scripts baseados no servidor que são processados pelo servidor Web antes de serem enviados para o browser do utilizador.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**  
Os servidores Web do Windows Server 2003 e do Windows XP Professional estão susceptíveis se o IIS e o ASP estiverem activados. Os fornecedores de alojamento na Web que permitem aos clientes carregar páginas ASP para os Web sites alojados apresentam um risco acrescido.
  
**O que faz a actualização?**  
A actualização remove a vulnerabilidade alterando a forma como o Internet Information Services processa a introdução de dados em páginas ASP.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios de que esta vulnerabilidade estava a ser explorada?**  
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

**Verificar se a actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "Orientações e ferramentas de detecção e implementação" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias versões do Microsoft Windows, os passos que se seguem podem ser diferentes no seu computador. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da versão do sistema operativo, ou dos programas instalados, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no computador, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

-   **Verificação da Chave de Registo**

    Poderá também verificar os ficheiros instalados por esta actualização de segurança consultando as chaves de registo listadas na **Tabela de referência**, nesta secção.

    Estas chaves de registo poderão não conter uma lista completa dos ficheiros instalados. Além disso, estas chaves de registo poderão não ser criadas correctamente se um administrador ou um OEM integrar ou associar esta actualização de segurança aos ficheiros de origem de instalação do Windows.

#### Informações de Implementação

**Instalação da actualização**

Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.

Caso tenha instalado anteriormente um hotfix para actualizar um destes ficheiros, o programa de instalação copia os ficheiros RTMQFE, SP1QFE ou SP2QFE para o sistema. Caso contrário, o programa de instalação copia os ficheiros RTMGDR, SP1GDR ou SP2GDR para o sistema. As actualizações de segurança podem não conter todas as variantes destes ficheiros. Para mais informações sobre este comportamento, consulte o [Artigo 824994 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824994/pt).

Para mais informações sobre o programa de instalação, visite o [Web site Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684/pt).

**Verificar se a Actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "Orientações e ferramentas de detecção e implementação" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

-   **Verificação da Chave de Registo**

    Poderá também verificar os ficheiros instalados por esta actualização de segurança consultando as chaves de registo listadas na **Tabela de referência**, nesta secção.

    Estas chaves de registo poderão não conter uma lista completa dos ficheiros instalados. Além disso, estas chaves de registo poderão não ser criadas correctamente se um administrador ou um OEM integrar ou associar esta actualização de segurança aos ficheiros de origem de instalação do Windows.
