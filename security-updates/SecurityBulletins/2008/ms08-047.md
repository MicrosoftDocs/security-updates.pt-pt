Security Bulletin

Microsoft Security Bulletin MS08-047 - Importante
=================================================

Vulnerabilidade no Processamento da Política IPsec poderia permitir a Divulgação de Informações (953733)
--------------------------------------------------------------------------------------------------------

Data de publicação: 12 de agosto de 2008 | Updated: 13 de agosto de 2008

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização resolve uma vulnerabilidade, comunicada de forma privada, na forma como são aplicadas certas regras de segurança do Protocolo Internet do Windows (IPsec). Esta vulnerabilidade poderia fazer com que sistemas ignorassem políticas IPsec e transmitissem tráfego de rede como texto não encriptado. Por sua vez, isso divulgaria informações que deviam estar encriptadas na rede. Um intruso que visualizasse tráfego na rede poderia visualizar e possivelmente modificar os conteúdos do tráfego. Tenha em atenção que esta vulnerabilidade não permitiria que um intruso executasse o código ou elevasse os seus direitos de utilizador directamente. Poderia ser usada para recolher informações úteis para tentar comprometer ainda mais o sistema ou rede afectado.

Esta actualização está classificada como Importante para todas as versões suportadas do Windows Vista e Windows Server 2008. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção**.**

A actualização de segurança resolve a vulnerabilidade garantindo que as regras IPsec são adequadamente processadas. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade.

**Questões conhecidas.** Nenhum

Vulnerabilidade de Divulgação de Informação na Política IPsec - CVE-2008-2246  
-----------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de divulgação de informação na forma como as políticas IPsec são importadas para domínios do Windows Server 2008 a partir de domínios do Windows Server 2003. Esta vulnerabilidade poderia fazer com que sistemas ignorassem políticas IPsec e transmitissem tráfego de rede como texto não encriptado. Por sua vez, isso divulgaria informações que deviam estar encriptadas na rede. Um intruso que interceptasse tráfego na rede poderia visualizar e possivelmente modificar os conteúdos do tráfego. Tenha em atenção que esta vulnerabilidade não permitiria que um intruso executasse o código ou elevasse os seus direitos de utilizador directamente. Poderia ser usada para recolher informações úteis para tentar comprometer ainda mais o sistema afectado.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-2246](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-2246).
  
#### Factores Atenuantes para a Vulnerabilidade de Divulgação de Informação na Política IPsec - CVE-2008-2246
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   No Windows Vista e Windows Server 2008, a regra de resposta predefinida não está activada automaticamente. A regra de resposta predefinida só é activada automaticamente quando cria uma política IPsec através de versões do Windows anteriores ao Windows Vista e ao Windows Server 2008.
  
#### Soluções Alternativas para a Vulnerabilidade de Divulgação de Informação na Política IPsec - CVE-2008-2246
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   Não seleccione "Regra de Resposta Predefinida" durante a criação de políticas IPsec ou desactive esta regra em políticas existentes, uma vez que esta regra já não é válida no Windows Vista e no Windows Server 2008 e só se aplica a versões anteriores do Windows. Para emular esta regra no Windows Vista e no Windows Server 2008, consulte o [Artigo 942964 da Base de Dados de Conhecimento Microsoft](http://support.microsoft.com/kb/942964/).
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Divulgação de Informação na Política IPsec - CVE-2008-2246
  
**Qual é a abrangência desta vulnerabilidade?**    
Trata-se de uma vulnerabilidade ao nível da divulgação de informações. Um intruso que explorasse esta vulnerabilidade com sucesso poderia ler conteúdos do tráfego de rede que de outro modo estariam encriptados. Tenha em atenção que esta vulnerabilidade não permitiria que um intruso executasse o código ou elevasse os seus direitos de utilizador directamente. Poderia ser usada para recolher informações úteis para tentar comprometer ainda mais o sistema afectado.
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade resulta de um erro quando a política IPsec predefinida é importada de um domínio do Windows Server 2003 para um domínio do Windows Server 2008. Em determinadas circunstâncias, este erro poderia fazer com que todas as regras IPsec fossem ignoradas.
  
**O que é a IPsec?**    
A segurança de Protocolo Internet (IPsec) é uma estrutura de padrões abertos destinada a garantir comunicações privadas e seguras nas redes de Protocolo Internet (IP) através da utilização de serviços de segurança criptográficos. O grupo de trabalho para IPsec da Internet Engineering Task Force (IETF) define os padrões IPsec.
  
A IPsec é a solução de longo prazo para redes seguras. Fornece uma protecção agressiva contra ataques a redes privadas e Internet, através de segurança extremo a extremo. Os únicos computadores que têm de conhecer a protecção IPsec são o remetente e o destinatário da comunicação. A IPsec permite proteger as comunicações entre grupos de trabalho, computadores de rede local, clientes e servidores de domínio, escritórios de filiais (que podem estar fisicamente ligados à rede remota), extranets e clientes com localizações móveis. Para mais informações sobre a IPsec, consulte o seguinte [Web site da Microsoft](http://www.microsoft.com/technet/network/ipsec/ipsecfaq.mspx).
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade com sucesso poderia visualizar e, em alguns casos, modificar tráfego de rede em texto não encriptado que deveria estar encriptado.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Esta vulnerabilidade poderia provocar o mau funcionamento de uma funcionalidade de segurança. Para que esta questão fosse aproveitada, seria necessário que um intruso estivesse a monitorizar o tráfego de rede. O intruso poderia ler e talvez modificar os conteúdos do tráfego de rede observados a partir da sua posição estratégica. Em circunstâncias normais, as políticas IPsec protegem este tráfego contra intercepções através da encriptação de conteúdos. Um intruso não poderia provocar a ocorrência desta situação se não tivesse anteriormente obtido permissões administrativas para o controlador de domínio. Em vez disso, ficaria a depender que um administrador, inconscientemente, configurasse mal as regras IPsec e fizesse com que informações fossem transmitidas sem encriptação.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
As estações de trabalho com Windows Vista e os servidores com Windows Server 2008 abrangidos por políticas IPsec baseadas num domínio do Windows Server 2008 actualizado a partir de um domínio do Windows Server 2003 estariam mais susceptíveis.
  
**O que faz a actualização?**    
A actualização remove a vulnerabilidade garantindo que as regras IPsec são processadas do modo adequado.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.
