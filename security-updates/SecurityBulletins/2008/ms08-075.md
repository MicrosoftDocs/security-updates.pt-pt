Security Bulletin

Microsoft Security Bulletin MS08-075 - Crítica
==============================================

Vulnerabilidades na Pesquisa do Windows poderiam permitir Execução Remota de Código (959349)
--------------------------------------------------------------------------------------------

Data de publicação: 9 de dezembro de 2008 | Updated: 10 de dezembro de 2008

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve duas vulnerabilidades na Pesquisa do Windows, comunicadas de forma privada. Estas vulnerabilidades poderiam permitir a execução remota de código se um utilizador abrisse e guardasse um ficheiro de pesquisa guardada especialmente concebido para o efeito no Explorador do Windows ou se um utilizador clicasse num URL de pesquisa especialmente concebido para o efeito. Um intruso que conseguisse tirar partido destas vulnerabilidades poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

A vulnerabilidade mais grave está classificada como Crítica para todas as edições suportadas do Windows Vista e Windows Server 2008. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

Esta actualização de segurança resolve as vulnerabilidades modificando a forma como o Explorador do Windows liberta memória ao guardar ficheiros de Pesquisa do Windows e modificando a forma como o Explorador do Windows interpreta os parâmetros ao analisar o protocolo search-ms. Para mais informações sobre as vulnerabilidades, consulte a subsecção de Perguntas Mais Frequentes quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes apliquem de imediato a actualização.

**Questões conhecidas.** Nenhum

#### Software Afectado e Software Não Afectado

O software que se segue foi testado para determinar quais as versões ou edições afectadas. As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Para determinar o ciclo de vida de suporte da versão ou edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Vulnerabilidade em Pesquisas Guardadas no Windows - CVE-2008-4268  
-----------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código ao guardar ficheiros de pesquisa especialmente concebidos para o efeito no Explorador do Windows. Esta operação faz com o Explorador do Windows seja encerrado e reinicie num modo susceptível de ser explorado.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-4268](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4268).
  
#### Factores Atenuantes para a Vulnerabilidade em Pesquisas Guardadas no Windows - CVE-2008-4268
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   A vulnerabilidade não pode ser automaticamente explorada através de correio electrónico. Para um ataque poder ser bem sucedido, é necessário que um utilizador abra e guarde um anexo recebido numa mensagem de correio electrónico.  
-   No cenário de um ataque baseado na Web, um intruso poderia alojar um Web site contendo um ficheiro de pesquisa guardada (.search-ms) especialmente concebido para o efeito. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos ou anúncios fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Contudo, em qualquer dos casos, o intruso não teria forma de forçar os utilizadores a visitar estes Web sites. As tentativas de exploração desta vulnerabilidade requerem interacção com o utilizador.  
-   Um intruso que explorasse com sucesso esta vulnerabilidade poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.  
-   Os utilizadores que executam o Internet Explorer 7 receberão um pedido através de uma caixa de diálogo indicando que uma página da Web está a tentar abrir conteúdo no Explorador do Windows. Esta caixa de diálogo solicita ao utilizador que seleccione "Permitir" ou "Não Permitir" o conteúdo, mas também permite que utilizadores não voltem a visualizar este tipo de mensagens. Os utilizadores que tenham seleccionado anteriormente "Não voltar a mostrar o aviso para este programa" no Explorador do Windows não receberão este pedido.
  
#### Soluções Alternativas para a Vulnerabilidade em Pesquisas Guardadas no Windows - CVE-2008-4268
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   Alterar temporariamente o tipo de ficheiros associado à extensão de ficheiros “.search-ms”. Introduza o seguinte comando numa linha de comandos com privilégios elevados:
  
    assoc .search-ms=xmlfile
  
    **Impacto da solução alternativa.** Os ficheiros de pesquisa guardada abrirão como ficheiros XML no Internet Explorer.
  
    **Como anular esta solução alternativa.** Introduza o seguinte comando numa linha de comandos com privilégios elevados:
  
    assoc .search-ms=SearchFolder
  
-   Modificar a chave de registo para impedir que os utilizadores possam abrir ficheiros de pesquisa guardada ou acedam à pasta de pesquisas guardadas.
  
    Para modificar a chave de registo, siga estes passos:
  
    **Nota** A utilização incorrecta do Editor de Registo pode provocar problemas graves que poderão conduzir à reinstalação do sistema operativo. A Microsoft não pode garantir que os problemas resultantes da utilização incorrecta do Editor de Registo possam ser resolvidos. A utilização do Editor de Registo é da sua responsabilidade. Para obter informações sobre como editar o registo, deve consultar o tópico de Ajuda "Alterar Chaves e Valores" do Editor de Registo (Regedit.exe) ou os tópicos de Ajuda "Adicionar e Eliminar Informações do Registo" e "Editar Dados do Registo" do Regedt32.exe.
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Localize e expanda a subchave de registo seguinte:  
        HKEY\_CLASSES\_ROOT\\SearchFolder  
        Clique em **Editar** e a seguir clique em **Permissões**.  
        **Nota** Tome nota das permissões que são apresentadas nesta caixa de diálogo, para que possa restaurar os seus valores originais mais tarde.  
    3.  Clique em **Avançadas.**  
    4.  Clique para limpar a caixa de verificação **Incluir permissões herdáveis do principal deste objecto**. É-lhe pedido que clique em **Copiar**, **Remover**, ou **Cancelar**. Clique em **Remover** e a seguir em **OK**.  
    5.  Recebe uma mensagem que o informa que nenhum utilizador poderá aceder a esta chave de registo. Clique em **Sim** e a seguir clique em **OK** para fechar a caixa de diálogo **Permissões para SearchFolder**.
  
    **Impacto da solução alternativa.** Os utilizadores não poderão abrir ficheiros de pesquisas guardadas ou aceder à pasta de pesquisas guardadas.
  
    **Como anular esta solução alternativa.** Reponha as permissões configuradas anteriormente na chave de registo, marcando com um clique a caixa de verificação **Incluir permissões herdáveis do principal deste objecto** e depois clicando em **OK** duas vezes.
  
-   Anular o registo do tipo de ficheiro Procurar na Pasta.
  
    Para modificar a chave de registo, siga estes passos:
  
    **Nota** A utilização incorrecta do Editor de Registo pode provocar problemas graves que poderão conduzir à reinstalação do sistema operativo. A Microsoft não pode garantir que os problemas resultantes da utilização incorrecta do Editor de Registo possam ser resolvidos. A utilização do Editor de Registo é da sua responsabilidade. Para obter informações sobre como editar o registo, deve consultar o tópico de Ajuda "Alterar Chaves e Valores" do Editor de Registo (Regedit.exe) ou os tópicos de Ajuda "Adicionar e Eliminar Informações do Registo" e "Editar Dados do Registo" do Regedt32.exe.
  
    **Utilizar o Método Interactivo**
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Localize e clique na subchave de registo seguinte:  
        HKEY\_CLASSES\_ROOT\\SearchFolder  
    3.  Clique no menu **Ficheiro** e seleccione **Exportar**.  
    4.  Na caixa de diálogo **Exportar ficheiro de registo,** introduza **SearchFolder\_file\_association\_registry\_backup.reg** e clique em **Guardar**.  
        **Nota** Isto criará uma cópia de segurança desta chave de registo, que, por predefinição, ficará na pasta **Os meus documentos**.  
    5.  Prima a tecla **Delete** no teclado para apagar a chave de registo. Quando lhe for solicitado que elimine a chave de registo através da caixa de diálogo **Confirmar Eliminação da Chave**, clique em **Sim**.
  
    **Utilizar a Gestão de Script de Implementação**
  
    1.  Crie uma cópia de segurança das chaves de registo utilizando a gestão de script de implementação que contém os seguintes comandos:  
        Regedit.exe /e SearchFolder\_registry\_backup.reg  
        HKEY\_CLASSES\_ROOT\\SearchFolder  
    2.  Depois, guarde o que se segue num ficheiro com a extensão .REG, como por exemplo, Eliminar\_associacao\_ficheiro\_ProcurarPasta.reg:  
        Editor de registo do Windows versão 5.00  
        \[-HKEY\_CLASSES\_ROOT\\SearchFolder\]  
    3.  Execute o script de registo acima no computador visado, usando o seguinte comando numa linha de comandos com privilégios elevados:  
        Regedit.exe /s Delete\_SearchFolder\_file\_association.reg
  
    **Impacto da solução alternativa.** Clicar duas vezes numa pasta de pesquisas guardadas deixará de abrir uma janela que apresenta os resultados de pesquisa.
  
    **Como anular esta solução alternativa:**
  
    **Utilizar o Método Interactivo**
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Clique no menu **Ficheiro** e seleccione **Importar**.  
    3.  Na caixa de diálogo Importar ficheiro de registo, seleccione **SearchFolder\_file\_association\_registry\_backup.reg** e clique em **Abrir**.
  
    **Utilizar a Gestão de Script de Implementação**
  
    Reponha o estado original executando o seguinte comando:  
    Regedit.exe /s SearchFolder\_registry\_backup.reg
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade em Pesquisas Guardadas no Windows - CVE-2008-4268
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
**O que provoca a vulnerabilidade?**    
O Explorador do Windows não liberta memória correctamente ao guardar ficheiros de Pesquisa do Windows.
  
**O que é a Pesquisa do Windows?**    
A Pesquisa do Windows é um componente de série do Windows Vista e do Windows Server 2008 que está activado por predefinição. A Pesquisa do Windows permite capacidades de pesquisa instantânea dos ficheiros e tipos de dados mais comuns, como correio electrónico, contactos, calendário, agenda, documentos, fotografias, multimédia e outros formatos fornecidos por outras empresas. Estas capacidades permitem que os consumidores e os trabalhadores da informação possam encontrar, gerir e organizar de forma eficaz a crescente quantidade de dados comuns em ambientes domésticos e empresariais. Para mais informações, consulte a [página inicial de Pesquisa do Windows](http://www.microsoft.com/windows/products/winfamily/desktopsearch/default.mspx).
  
**O componente de Pesquisa do Windows disponível para o Windows XP é afectado por esta questão?**  
Não. Embora a Pesquisa do Windows seja um suplemento opcional para o Windows XP, não é afectada por esta questão.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia executar código arbitrário no sistema de um utilizador. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Esta vulnerabilidade requer que um utilizador abra e guarde um ficheiro de pesquisa guardada especialmente concebido para o efeito numa versão afectada do Explorador do Windows.  
Num cenário de ataque com base em correio electrónico, um intruso poderia explorar a vulnerabilidade enviando um ficheiro de pesquisa guardada especialmente concebido para o efeito ao utilizador e convencendo-o a abrir o ficheiro. Depois de abrir o ficheiro, o utilizador teria de guardar o ficheiro de pesquisa guardada para que a exploração ocorresse. A exploração ocorre independentemente do local em que a pesquisa é guardada.  
Num cenário de ataque com base na Web, um intruso teria de alojar um Web site contendo um ficheiro de pesquisa guardada que fosse utilizado para tentar explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Um intruso teria de convencer o utilizador a abrir e a guardar ficheiros de pesquisa especialmente concebidos para o efeito. Um intruso não conseguiria obrigar os utilizadores a visitarem um Web site especialmente concebido para o efeito. Em vez disso, um intruso teria de convencê-los a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico ou num programa de mensagens instantâneas que os levasse para o site do intruso.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Os sistemas afectados são principalmente estações de trabalho e servidores de terminais. Os servidores podem estar mais sujeitos a risco se os administradores permitirem que os utilizadores iniciem sessão nos servidores e executem programas. Contudo, os procedimentos recomendados desencorajam essa acção.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade modificando a forma como o Explorador do Windows liberta memória ao guardar ficheiros de Pesquisa do Windows.

Vulnerabilidade na Análise da Pesquisa do Windows - CVE-2008-4269  
-----------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código no Explorador do Windows que permite que um intruso construa uma página Web maliciosa com uma chamada para o processador de protocolos search-ms. O processador de protocolos, por sua vez, passa dados não fidedignos para o Explorador do Windows.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-4269](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-4269).
  
#### Factores Atenuantes para a Vulnerabilidade na Análise da Pesquisa do Windows - CVE-2008-4269
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   No cenário de um ataque baseado na Web, um intruso poderia alojar um Web site contendo um URL (search-ms://) de Pesquisa do Windows especialmente concebido para o efeito. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos ou anúncios fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Contudo, em qualquer dos casos, o intruso não teria forma de forçar os utilizadores a visitar estes Web sites. As tentativas de exploração desta vulnerabilidade requerem interacção com o utilizador.  
-   Um intruso que explorasse com sucesso esta vulnerabilidade poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.  
-   Os clientes que utilizam o Internet Explorer 7 com as suas predefinições receberão um aviso pop-up solicitando aos utilizadores que confirmem se querem permitir que a página Web comunique com o Explorador do Windows. De acordo com as predefinições, a comunicação não é permitida.  
-   Os sistemas Windows Vista e Windows Server 2008 apresentarão uma caixa de diálogo de aviso antes de executar um programa do exterior da intranet. Este aviso não será apresentado se o intruso tentar fazer referências e executáveis dentro da intranet ou no computador visado.
  
#### Soluções Alternativas para a Vulnerabilidade na Análise da Pesquisa do Windows - CVE-2008-4269
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Desactivar o processador de protocolos search-ms dentro do Explorador do Windows**
  
    **Nota** Reparar o Office ou instalar uma actualização de segurança do Office pode anular esta solução alternativa.
  
    **Aviso** Se utilizar o Editor de Registo incorrectamente, pode causar problemas sérios que podem vir a obrigar à reinstalação do seu sistema operativo. A Microsoft não pode garantir que consiga resolver problemas que resultam da utilização incorrecta do Editor de Registo. A utilização do Editor de Registo é da sua responsabilidade. Para obter informações sobre como editar o registo, deve consultar o tópico de Ajuda "Alterar Chaves e Valores" do Editor de Registo (Regedit.exe) ou os tópicos de Ajuda "Adicionar e Eliminar Informações do Registo" e "Editar Dados do Registo" do Regedt32.exe.
  
    **Nota** Recomendamos a realização de uma cópia de segurança do registo antes de o editar.
  
    Para modificar a chave de registo, siga estes passos:
  
    **Utilizar o Método Interactivo**
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva "regedit" (sem as aspas) na caixa Abrir e clique em **OK**.  
    2.  Localize e expanda a subchave de registo seguinte:  
    3.  **HKEY\_CLASSES\_ROOT\\search-ms\\shell\\open**.  
    4.  Clique em **command**.  
    5.  Clique no menu **Ficheiro** e seleccione **Exportar**.  
    6.  Na caixa de diálogo Exportar ficheiro de registo, introduza "Search-ms\_pluggable\_protocol\_registry\_backup.reg" e clique em **Guardar**.  
    7.  Edite o valor de registo (Predefinição) fazendo duplo clique sobre ele. Apague todos os dados do campo Dados do valor: campo.  
    8.  Clique em **OK.**
  
    **Utilizar a Gestão de Script de Implementação**
  
    1.  Crie uma cópia de segurança das chaves de registo utilizando a gestão de script de implementação que contém o seguinte comando:  
        regedit /e Search-ms\_pluggable\_protocol\_registry\_backup.reg HKEY\_CLASSES\_ROOT\\search-ms\\shell\\open\\command.  
    2.  Depois, guarde o que se segue num ficheiro com a extensão .REG, como por exemplo  
        "Disable\_search-ms\_pluggable\_protocol.reg":  
        Editor de registo do Windows versão 5.00  
        \[HKEY\_CLASSES\_ROOT\\search-ms\\shell\\open\\command\]  
        @=""  
    3.  Execute o script de registo acima no computador visado, usando o seguinte comando numa linha de comandos com privilégios elevados:  
        Regedit.exe /s Disable\_ search-ms\_pluggable\_protocol.reg
  
    **Impacto da solução alternativa.**
  
    As páginas Web não poderão iniciar pesquisas no Windows.
  
    **Como anular esta solução alternativa.**
  
    Utilizar o Método Interactivo

    2.  Clique no menu Ficheiro e seleccione Importar.  
    3.  Na caixa de diálogo Importar ficheiro de registo, seleccione "Search-ms\_pluggable\_protocol\_registry\_backup.reg" e clique em **Abrir**.
