Security Bulletin

Microsoft Security Bulletin MS08-076 - Importante
=================================================

Vulnerabilidades nos Componentes Windows Media poderiam permitir Execução Remota de Código (959807)
---------------------------------------------------------------------------------------------------

Data de publicação: 9 de dezembro de 2008 | Updated: 24 de novembro de 2009

**actualizada:** 5.0

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve duas vulnerabilidades nos seguintes componentes Windows Media, comunicadas de forma privada: Windows Media Player, Runtime do Formato Windows Media e Windows Media Services. A vulnerabilidade mais grave poderia permitir a execução remota de código. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Esta actualização de segurança está classificada como Importante para o Windows Media Player 6.4, Runtime do Formato Windows Media 7.1, Runtime do Formato Windows Media 9.0, Runtime do Formato Windows Media 9.5, Runtime do Formato Windows Media 11, Windows Media Services 4.1, Windows Media Services 9 Series e Windows Media Services 2008. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a primeira vulnerabilidade modificando a forma como as respostas de autenticação do Windows Media são validadas. A actualização de segurança resolve a segunda vulnerabilidade garantindo que os clientes Windows Media lidam com os servidores que usam endereços ISATAP como se fossem externos. Para mais informações sobre as vulnerabilidades, consulte a subsecção de Perguntas Mais Frequentes quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade.

**Questões conhecidas.** Nenhum

#### Software Afectado e Software Não Afectado

O software que se segue foi testado para determinar quais as versões ou edições afectadas. As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Para determinar o ciclo de vida de suporte da versão ou edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Perguntas mais frequentes relativas a esta actualização de segurança
--------------------------------------------------------------------

<span></span>
**Por que razão foi relançado este boletim a 24 de Novembro de 2009?**  
A Microsoft relançou este boletim para disponibilizar um novo pacote de actualização específico para sistemas Windows XP Embedded de forma a resolver questões de detecção e instalação nos pacotes de actualização originais do Windows XP incluídos com este boletim.

Para além disso, a Microsoft reviu os pacotes de actualização existentes para o Windows XP Service Pack 2 (KB952069) e Windows XP Service Pack 3 (KB952069) de forma a evitar que os pacotes sejam disponibilizados a sistemas Windows XP Embedded. Os clientes que já tenham instalado esta actualização de segurança estão protegidos e não necessitam de efectuar qualquer acção.

Os clientes que utilizam o sistema operativo Windows XP Embedded devem aplicar a actualização na primeira oportunidade. Os clientes podem também transferir manualmente a actualização para o [Windows XP Embedded Service Pack 2](http://www.microsoft.com/downloads/details.aspx?familyid=c8f59b79-7d09-4976-a678-9aef5975c186) (KB972187) ou a actualização para o [Windows XP Embedded Service Pack 3](http://www.microsoft.com/downloads/details.aspx?familyid=502dcf76-aec3-48bf-930d-0c02b1604cd1) (KB972187). Nenhuma outra actualização é afectada por este relançamento.

**O que é o sistema operativo Windows XP Embedded?**  
O Microsoft Windows XP Embedded é uma versão do sistema operativo Windows XP Professional dividida em componentes, que proporciona o amplo conjunto de funcionalidades do Windows XP Professional a dispositivos incorporados. O Windows XP Embedded é optimizado para a indústria de vendas a retalho, com capacidades de rede integradas e suporte plug-and-play baseado em padrões. O Windows XP Embedded permite-lhe ligar uma variedade de dispositivos periféricos de venda a retalho ou hotelaria a um único sistema de ponto de serviço (POS). Para obter mais informação sobre os sistemas operativos Windows XP Embedded, consulte o [Microsoft Windows Embedded Developer Center](http://msdn.microsoft.com/en-us/windowsembedded/default.aspx).

**Por que razão foi relançado este boletim a 13 de Janeiro de 2009?**  
A Microsoft relançou este boletim para oferecer novos pacotes de actualização para o Runtime do Formato Windows Media 9.5 no Windows XP Service Pack 2 (KB952069) e no Windows XP Service Pack 3 (KB952069). Os novos pacotes de actualização resolvem uma questão de instalação que poderia resultar numa instalação incompleta nos sistemas com o Windows XP que executam o Runtime do Formato Windows Media 9.5 nos casos em que o cliente tenha instalado a actualização separada KB944110 antes de instalar a KB952069. Esses clientes ainda estão vulneráveis e necessitam de transferir e instalar os pacotes de actualização relançados. Nenhum outro pacote de actualização é afectado por este relançamento. Os clientes que executem todas as outras versões suportadas e afectadas do Windows Media Player, Runtime do Formato Windows Media e Windows Media Services que já tenham instalado com sucesso os pacotes de actualização originais não necessitam de efectuar qualquer outra acção. Os pacotes de actualização de segurança relançados estão disponíveis nos mesmos canais de distribuição que a actualização de segurança original, incluindo Actualizações Automáticas, Windows Update e Windows Server Update Services.

A Microsoft também reviu este boletim para listar o Windows Media Player 6.4 e o Windows Media Services 4.1 como afectados em todas as edições do Microsoft Windows 2000 Service Pack 4 (não apenas no Microsoft Windows 2000 Server Service Pack 4). Aplica-se a mesma actualização; não existem novos pacotes de actualização para este sistema operativo. No entanto, os clientes que tiveram acesso às actualizações KB954600 para o Windows Media Player 6.4 ou KB952068 para o Windows Media Services 4.1, e que não as tenham aplicado, devem fazê-lo.

**Onde estão os pormenores sobre as informações do ficheiro?**  
Os pormenores sobre as informações do ficheiro podem ser encontrados no [Artigo 959807 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/959807).

**Por que razão esta actualização resolve diversas vulnerabilidades de segurança comunicadas?**  
Esta actualização contém suporte para diversas vulnerabilidades, porque as modificações necessárias para resolver essas questões estão incluídas nos ficheiros relacionados. Assim, em vez de instalarem diversas actualizações quase iguais, os clientes podem instalar apenas esta actualização.

**Estou a usar uma edição anterior do software discutido neste boletim de segurança. O que devo fazer?**  
O software afectado incluído neste boletim foi testado para determinar quais as edições afectadas. As outras edições ultrapassaram o respectivo ciclo de vida de suporte. Para determinar o ciclo de vida de suporte da edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Os utilizadores que tenham edições anteriores do software deviam considerar a migração para edições suportadas como uma prioridade, de modo a poderem proteger os seus sistemas contra a potencial exposição a vulnerabilidades. Para mais informações sobre o Ciclo de Vida dos Produtos Windows, visite o Web site do [Ciclo de Vida de Suporte Microsoft.](http://support.microsoft.com/lifecycle/) Para mais informações sobre o período de suporte alargado a actualizações de segurança para estas versões ou edições de software, visite o Web site de [Suporte Técnico da Microsoft](http://go.microsoft.com/fwlink/?linkid=33328).

Os clientes que necessitem de suporte personalizado para edições anteriores devem contactar o seu representante de equipa de contas da Microsoft, o seu Gestor Técnico de Contas ou o parceiro ou representante Microsoft adequado para obter mais informações sobre opções de suporte personalizado. Os clientes que não disponham de um Contrato Alliance, Premier ou Autorizado podem contactar a subsidiária local da Microsoft. Para obter informações de contacto, visite o [Microsoft Worldwide Information](http://go.microsoft.com/fwlink/?linkid=33329), seleccione o país e, em seguida, clique em **Go** para ver uma lista de números de telefone. Quando telefonar, peça para falar com o Gestor de Vendas de Suporte Premier local. Para mais informações, consulte as [Perguntas Mais Frequentes de suporte sobre o ciclo de vida de produto, relativas aos sistemas operativos Windows](http://go.microsoft.com/fwlink/?linkid=33330).

Vulnerabilidade SPN - CVE-2008-3009
-----------------------------------

<span></span>
Existe uma vulnerabilidade de reflexão de credenciais em componentes Windows Media que poderia permitir que um intruso executasse código com os mesmos direitos que o utilizador local ou com as credenciais de distribuição do Windows Media Services. A vulnerabilidade existe devido a falhas nas implementações de nome do principal do serviço (SPN) para componentes Windows Media.

Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-3009](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3009).

#### Factores Atenuantes para a Vulnerabilidade SPN - CVE-2008-3009

A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:

-   O Windows Media Services é um componente opcional em todas as plataformas Windows Server e não está instalado por predefinição.
-   Num cenário de ataque com base na Web, um intruso teria de alojar um Web site contendo uma página Web que fosse utilizada para tirar partido desta vulnerabilidade. Um intruso não conseguiria obrigar os utilizadores a visitarem um Web site mal intencionado. Em vez disso, um intruso teria de persuadi-los a visitar o Web site, levando-os normalmente a clicar numa hiperligação que os levasse ao Web site do intruso. Depois de clicarem numa ligação, ser-lhes-ia solicitado que executassem várias acções. Um ataque só poderia ocorrer depois de terem executado estas acções.
-   Para sistemas Windows Server 2008 e Windows Server 2008 baseados em x64, o Runtime do Formato Windows Media 11 só está presente no sistema quando o Desktop Experience está instalado.

#### Soluções Alternativas para a Vulnerabilidade SPN - CVE-2008-3009

A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.

#### Perguntas Mais Frequentes sobre a Vulnerabilidade SPN - CVE-2008-3009

**Qual é a abrangência desta vulnerabilidade?**  
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade com sucesso poderia executar código com os mesmos direitos do utilizador com sessão iniciada. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.  Um intruso que conseguisse tirar partido desta vulnerabilidade poderia também executar código com credenciais de distribuição do Windows Media Services. Nesta instância, um intruso poderia executar código com os mesmos privilégios que a conta NETWORK SERVICE.

**O que provoca a vulnerabilidade?**  
Os componentes Windows Media não dão a permissão correcta às protecções de reflexão de credenciais NTLM para garantir que as credenciais do utilizador não são reflectidas e usadas para se fazer passar pelo utilizador.

**O que são nomes do principal do serviço (SPNs)?**  
Um nome do principal do serviço (SPN) é o nome pelo qual um cliente identifica exclusivamente uma instância de um serviço. O SPN fornece um tipo de identidade de serviço utilizado para autenticar o serviço no cliente. Quando um cliente pretende estabelecer ligação a um serviço, localiza uma instância do serviço, compõe um SPN para essa instância, liga-se ao serviço e apresenta o SPN para ser autenticado pelo serviço. Para mais informações sobre o SPN, consulte [Autenticação e Identidade de Serviço](http://msdn.microsoft.com/en-us/library/ms733130.aspx). Para mais informações sobre como formatar correctamente SPNs, consulte [Formatos de Nomes para SPNs Únicos](http://msdn.microsoft.com/en-us/library/ms677601(vs.85).aspx).

**Como poderia um intruso utilizar a vulnerabilidade?**  
Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia obter os mesmos privilégios que o utilizador com sessão iniciada. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

**Como poderia um intruso explorar a vulnerabilidade?**  
Esta vulnerabilidade requer que um utilizador com uma versão afectada de componentes Windows Media aceda a um servidor malicioso. Um intruso poderia alojar uma partilha de servidor maliciosa ou Web site que exija autenticação. Quando o cliente se liga e autentica no servidor malicioso, o intruso pode utilizar as credenciais fornecidas pelo cliente, reflectindo-as de volta para o mesmo ou reencaminhando-as para outros servidores, fazendo-se passar pelo cliente. Um intruso não conseguiria obrigar os utilizadores a visitarem uma área de partilha de servidor ou um Web site especialmente concebido para o efeito. Em vez disso, um intruso teria de convencê-los a visitar a área de partilha de servidor ou o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico ou num programa de mensagens instantâneas que os levasse para o site do intruso.

**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**  
Qualquer sistema que utilize uma versão afectada do Windows Media Player, Runtime do Formato Windows Media ou Windows Media Services está susceptível a esta vulnerabilidade. Qualquer sistema que seja utilizado frequentemente por estes componentes, como as estações de trabalho ou servidores de terminal, está mais susceptível a esta vulnerabilidade.

**O que faz a actualização?**  
A actualização de segurança resolve a vulnerabilidade modificando a forma como as respostas de autenticação do componente Windows Media são validadas, evitando a repetição de credenciais.

**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.

**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**  
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

Vulnerabilidade ISATAP - CVE-2008-3010
--------------------------------------

<span></span>
Existe uma vulnerabilidade de divulgação de informações em versões suportadas de componentes Windows Media que poderia resultar na divulgação de credenciais NTLM. Qualquer componente Windows Media que aceda a um URL que utilize um endereço ISATAP poderá provocar a fuga de credenciais NTLM do utilizador para o servidor que aloja o URL. Isto poderia permitir a um intruso que está fora da zona de intranet apoderar-se das credenciais NTLM de um ambiente de empresa.

Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-3010](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-3010).

#### Factores Atenuantes para a Vulnerabilidade ISATAP - CVE-2008-3010

A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:

-   Os procedimentos recomendados de firewall e as configurações de firewall padrão predefinidas podem ajudar a proteger os recursos de rede de ataques provenientes do exterior do perímetro da empresa. É recomendado que os sistemas ligados à Internet tenham um número mínimo de portas e protocolos expostos.
-   Num cenário de ataque com base na Web, um intruso teria de alojar um Web site contendo uma página Web que fosse utilizada para tirar partido desta vulnerabilidade. Um intruso não conseguiria obrigar os utilizadores a visitarem um Web site mal intencionado. Em vez disso, um intruso teria de persuadi-los a visitar o Web site, levando-os normalmente a clicar numa hiperligação que os levasse ao Web site do intruso. Depois de clicarem numa ligação, ser-lhes-ia solicitado que executassem várias acções. Um ataque só poderia ocorrer depois de terem executado estas acções.
-   Para sistemas Windows Server 2008 e Windows Server 2008 baseados em x64, o Runtime do Formato Windows Media 11 só está presente no sistema quando o Desktop Experience está instalado.

#### Soluções Alternativas para a Vulnerabilidade ISATAP - CVE-2008-3010

Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:

-   **Bloquear o Protocolo IP Tipo 41 (ISATAP) na Firewall**

    O protocolo ISATAP é definido como protocolo tipo 41 em [RFC4214: Intra-Site Automatic Tunnel Addressing Protocol (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt). O bloqueio deste tipo de protocolo na firewall ajudará a impedir que os sistemas protegidos por essa firewall sejam atacados por tentativas de explorar esta vulnerabilidade. A Microsoft recomenda que bloqueie todas as comunicações de entrada não solicitadas a partir da Internet para ajudar a impedir ataques que possam utilizar outras portas ou protocolos.

#### Perguntas Mais Frequentes sobre a Vulnerabilidade ISATAP - CVE-2008-3010

**Qual é a abrangência desta vulnerabilidade?**  
Trata-se de uma vulnerabilidade ao nível da divulgação de informações. Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia obter as credenciais NTLM do utilizador com sessão iniciada. Estas credenciais podem ser posteriormente reflectidas no sistema do utilizador para executar código com os mesmos direitos do utilizador com sessão iniciada.

**O que provoca a vulnerabilidade?**  
Os componentes Windows Media afectados lidam incorrectamente com recursos da Internet, considerando-os como uma parte da zona de intranet local aquando da ligação a um servidor que utiliza um endereço ISATAP.

**O que é o ISATAP?**  
O Intra-Site Automatic Tunnel Addressing Protocol (ISATAP) disponibiliza conectividade IPv6 dentro de uma Intranet IPv4. O RFC para ISATAP é alojado pela Internet Engineering Task Force (IETF). Para mais informações sobre o ISATAP, consulte [RFC4214: Intra-Site Automatic Tunnel Addressing Protocol (ISATAP)](http://www.ietf.org/rfc/rfc4214.txt).

**O que é o NTLM?**  
O NTLM é um protocolo de autenticação baseado num mecanismo de pergunta/resposta.

**Como poderia um intruso utilizar a vulnerabilidade?**  
Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter as credenciais NTLM do utilizador.  Estas credenciais podem ser posteriormente reflectidas no sistema do utilizador para executar código com os mesmos direitos do utilizador com sessão iniciada.

**Como poderia um intruso explorar a vulnerabilidade?**  
Esta vulnerabilidade requer que um utilizador com uma versão afectada de componentes Windows Media aceda a um servidor malicioso. Um intruso poderia alojar uma partilha de servidor maliciosa ou Web site que exija autenticação. Um intruso não conseguiria obrigar os utilizadores a visitarem uma área de partilha de servidor ou um Web site especialmente concebido para o efeito. Em vez disso, um intruso teria de convencê-los a visitar a área de partilha de servidor ou o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico ou num programa de mensagens instantâneas que os levasse para o site do intruso.

**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**  
Qualquer sistema que utilize uma versão afectada do Windows Media Player, Runtime do Formato Windows Media ou Windows Media Services está susceptível a esta vulnerabilidade. Qualquer sistema que seja utilizado frequentemente por estes componentes, como as estações de trabalho ou servidores de terminal, está mais susceptível a esta vulnerabilidade.

**O que faz a actualização?**  
A actualização remove a vulnerabilidade garantindo que os clientes de componentes Windows Media lidam com os servidores que usam endereços ISATAP como se fossem externos e que não transmitem as credenciais de utilizador.

**Verificar se a actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

#### Informações de Implementação

**Instalação da actualização**

Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.

Caso tenha instalado anteriormente um hotfix para actualizar um destes ficheiros, o programa de instalação copia os ficheiros SP2QFE ou SP3QFE para o sistema. Caso contrário, o programa de instalação copia os ficheiros SP2GDR ou SP3GDR para o sistema. As actualizações de segurança podem não conter todas as variantes destes ficheiros. Para mais informações sobre este comportamento, consulte o [Artigo 824994 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824994).

Para mais informações sobre o programa de instalação, visite o [Web site Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

    Estas chaves de registo poderão não conter uma lista completa dos ficheiros instalados. Além disso, estas chaves de registo poderão não ser criadas correctamente se um administrador ou um OEM integrar ou associar esta actualização de segurança aos ficheiros de origem de instalação do Windows.

#### Informações de Implementação

**Instalação da actualização**

Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.

Caso tenha instalado anteriormente um hotfix para actualizar um destes ficheiros, o programa de instalação copia os ficheiros SP1QFE ou SP2QFE para o sistema. Caso contrário, o programa de instalação copia os ficheiros SP1GDR ou SP2GDR para o sistema. As actualizações de segurança podem não conter todas as variantes destes ficheiros. Para mais informações sobre este comportamento, consulte o [Artigo 824994 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824994).

Para mais informações sobre o programa de instalação, visite o [Web site Microsoft TechNet](http://go.microsoft.com/fwlink/?linkid=38951).

Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
