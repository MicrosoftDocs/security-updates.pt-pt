Security Bulletin

Microsoft Security Bulletin MS08-049 - Importante
=================================================

Vulnerabilidades no Sistema de Eventos poderiam permitir a Execução Remota de Código (950974)
---------------------------------------------------------------------------------------------

Data de publicação: 12 de agosto de 2008 | Updated: 10 de setembro de 2008

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização resolve duas vulnerabilidades no Sistema de Eventos do Microsoft Windows, comunicadas de forma privada, que poderiam permitir execução remota de código. Um intruso que conseguisse tirar partido destas vulnerabilidades poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios administrativos.

Esta actualização de segurança está classificada como Importante para todas as edições suportadas do Windows 2000, Windows XP, Windows Vista, Windows Server 2003 e Windows Server 2008. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve as vulnerabilidades alterando a forma como o Sistema de Eventos processa as subscrições por utilizador. Para mais informações sobre estas vulnerabilidades, consulte a subsecção de Perguntas Mais Frequentes sobre a vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.**  A Microsoft recomenda que os clientes instalem esta actualização na primeira oportunidade.

**Questões conhecidas.** O [Artigo 950974 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/950974) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões. Para mais informações sobre a instalação destas actualizações, consulte a subsecção **Perguntas Mais Frequentes Relativas a esta Actualização de Segurança**, nesta secção.

Vulnerabilidade no Sistema de Eventos - CVE-2008-1457  
-----------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código porque o Sistema de Eventos do Windows não valida correctamente pedidos de subscrição do utilizador quando estes são criados. A vulnerabilidade poderia permitir a execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios administrativos.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-1457](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1457).
  
#### Factores Atenuantes para a Vulnerabilidade no Sistema de Eventos - CVE-2008-1457
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso tem de se autenticar para poder explorar esta vulnerabilidade.
  
#### Soluções Alternativas para a Vulnerabilidade no Sistema de Eventos - CVE-2008-1457
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Anular o registo do es.dll**
  
    Siga os seguintes passos:
  
    1.  Inicie sessão como utilizador com privilégios de administrador.  
    2.  Clique em **Iniciar**, depois clique em **Executar**, digite **cmd** e depois clique em **OK**.  
    3.  Escreva o seguinte comando na linha de comandos:  
        regsvr32 /-u %WINDIR%\\SYSTEM32\\es.dll
  
    **Impacto da solução alternativa:** O Sistema de Eventos não será executado.
  
    **Como anular esta solução alternativa.** Volte a registar o ficheiro es.dll. Escreva o seguinte comando na linha de comandos:
  
    regsvr 32 %WINDIR%\\SYSTEM32\\es.dll
  
-   **Interrompa e desactive os serviços de Notificação de Eventos do Sistema e Sistema de Eventos COM+**
  
    Desactivar estes serviços ajuda a proteger os sistemas afectados contra tentativas de explorar esta vulnerabilidade.
  
    -   Interactivamente
  
        Siga os seguintes passos:
  
        1.  Clique em **Iniciar** e, em seguida, clique em **Painel de Controlo**. Alternativamente, seleccione **Definições** e depois clique em **Painel de Controlo**.  
        2.  Faça duplo clique em **Ferramentas Administrativas**. Como alternativa, clique em **Mudar para a vista clássica** e a seguir clique duas vezes sobre **Ferramentas administrativas**.  
        3.  Faça duplo clique em **Serviços**.  
        4.  Faça duplo clique em **Notificação de Eventos do Sistema**.  
        5.  Na lista **Tipo de arranque**, clique em **Desactivado**.  
        6.  Clique em **Parar** e a seguir em **OK**.
  
        Também pode interromper e desactivar o serviço de **Notificação de Eventos do Sistema** usando o seguinte comando na linha de comandos (disponível no Windows XP e no Microsoft Windows 2000 Resource Kit):
  
        sc stop sens & sc config sens start= disabled
  
        Em seguida, repita os passos acima para o **Sistema de Eventos COM+.**
  
        Também pode interromper e desactivar o serviço **Sistema de Eventos COM+** usando o seguinte comando na linha de comandos (disponível no Windows XP e no Microsoft Windows 2000 Resource Kit):
  
        sc stop eventsystem & sc config eventsystem start= disabled
  
    -   Por Políticas de grupos:
  
        Desactive estes serviços utilizando as definições das Políticas de Grupo. Pode desactivar o arranque deste serviço a nível local, de site, domínio ou unidade organizacional, utilizando a funcionalidade objecto de Políticas de Grupos nos ambientes de domínio Microsoft Windows 2000 ou Windows Server 2003.
  
        **Nota:** Poderá ainda consultar o manual Windows Server 2003 Security Guide. Este manual contém informações sobre como desactivar os serviços.
  
        Para mais informações sobre as Políticas de Grupos, consulte os seguintes Web sites da Microsoft:
  
        -   [Conjunto de Política de Grupos](http://technet2.microsoft.com/windowsserver/en/library/6d7cb788-b31d-4d17-9f1e-b5ddaa6deecd1033.mspx?mfr=true)  
        -   [O que é o Editor de Objectos da Política de Grupos?](http://technet2.microsoft.com/windowsserver/en/library/47ba1311-6cca-414f-98c9-2d7f99fca8a31033.mspx?mfr=true)  
        -   [Ferramentas e definições da Política de Grupos Central](http://technet2.microsoft.com/windowsserver/en/library/e926577a-5619-4912-b5d9-e73d4bdc94911033.mspx?mfr=true)
  
    **Impacto da solução alternativa.** Os programas que requerem estes serviços não serão executados.
  
    **Como anular esta solução alternativa:** Use os passos anteriores para definir o **tipo de arranque** como **Automático** e iniciar o serviço.
  
-   **Modifique a Lista de Controlo de Acesso no ficheiro es.dll**
  
    Para modificar a Lista de Controlo de Acesso (ACL) no es.dll para a tornar mais restritiva, siga estes passos:
  
    1.  Inicie sessão como utilizador com privilégios de administrador.  
    2.  Clique em **Iniciar**, depois clique em **Executar**, digite **cmd** e depois clique em **OK**.  
    3.  Tome nota das ACLs actuais que estão no ficheiro (incluindo definições mais antigas) para referência futura no caso de ter de anular esta modificação. Para ver as ACLs, escreva o seguinte:  
        cacls %WINDIR%\\SYSTEM32\\es.DLL  
    4.  Para negar o acesso do grupo "todos" ao ficheiro, escreva o seguinte comando na linha de comandos:  
        echo y| cacls %WINDIR%\\SYSTEM32\\es.DLL /E /P everyone:N
  
    **Impacto da solução alternativa:** Os programas que requerem o Sistema de Eventos não serão executados.
  
    **Como anular esta solução alternativa:** Escreva o seguinte comando na linha de comandos:
  
    echo y| cacls %WINDIR%\\SYSTEM32\\es.DLL /E /R everyone  
    Para modificar a Lista de Controlo de Acesso (ACL) no es.dll para a tornar menos restritiva, siga estes passos:
  
    1.  Inicie sessão como utilizador com privilégios de administrador.  
    2.  Clique em **Iniciar**, depois clique em **Executar**, digite **cmd** e depois clique em **OK**.  
    3.  Tome nota das ACLs actuais que estão no ficheiro (incluindo definições mais antigas) para referência futura no caso de ter de anular esta modificação. Para ver as ACLs, escreva o seguinte:  
        cacls %WINDIR%\\SYSTEM32\\es.DLL  
    4.  Para permitir o acesso do grupo "todos" ao ficheiro, escreva o seguinte comando na linha de comandos:  
        takeown.exe /f %WINDIR%\\SYSTEM32\\es.DLL

#### Perguntas Mais Frequentes sobre a Vulnerabilidade no Sistema de Eventos - CVE-2008-1457
  
**Qual é a abrangência desta vulnerabilidade?**    
Existe uma vulnerabilidade de execução remota de código no Sistema de Eventos do Microsoft Windows. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios administrativos.
  
**O que provoca a vulnerabilidade?**    
O Sistema de Eventos do Microsoft Windows não processa subscrições por utilizador correctamente, o que poderia permitir a execução de código arbitrário no contexto do sistema local.
  
**O que é o Sistema de Eventos do Microsoft Windows?**    
O Sistema de Eventos do Microsoft Windows é um serviço que gere chamadas de método e subscrições de eventos entre o Windows e aplicações existentes no sistema.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade poderia obter controlo total sobre o sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios administrativos.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Para explorar esta vulnerabilidade, um intruso teria de estar autenticado. Nesse caso, um intruso poderia enviar um pedido de subscrição de evento especialmente concebido para o efeito para o Sistema de Eventos do Microsoft Windows, o que permitiria que o intruso executasse um código com privilégios de sistema e obtivesse controlo total sobre o sistema afectado.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Todos os sistemas que executem versões afectadas do Sistema de Eventos do Microsoft Windows estão susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização remove a vulnerabilidade modificando a forma como o Sistema de Eventos do Microsoft Windows processa subscrições por utilizador, de modo a que estas não executem código no contexto de segurança de um publicador de eventos.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

-----------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código porque o Sistema de Eventos do Microsoft Windows não valida correctamente a gama de índices ao solicitar um array de ponteiros de funções. A vulnerabilidade poderia permitir a execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios administrativos.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2008-1456](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2008-1456).
  
#### Factores Atenuantes para a Vulnerabilidade no Sistema de Eventos - CVE-2008-1456
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso tem de se autenticar para poder explorar esta vulnerabilidade.
  
#### Soluções Alternativas para a Vulnerabilidade no Sistema de Eventos - CVE-2008-1456
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Anular o registo do es.dll**
  
    Siga os seguintes passos:
  
    1.  Inicie sessão como utilizador com privilégios de administrador.  
    2.  Clique em **Iniciar**, depois clique em **Executar**, digite **cmd** e depois clique em **OK**.  
    3.  Escreva o seguinte comando na linha de comandos:  
        regsvr32 /-u %WINDIR%\\SYSTEM32\\es.dll
  
    **Impacto da solução alternativa:** O Sistema de Eventos não será executado.
  
    **Como anular esta solução alternativa.** Volte a registar o ficheiro es.dll. Escreva o seguinte comando na linha de comandos:
  
    regsvr 32 %WINDIR%\\SYSTEM32\\es.dll
  
-   **Interrompa e desactive os serviços de Notificação de Eventos do Sistema e Sistema de Eventos COM+**
  
    Desactivar estes serviços ajuda a proteger os sistemas afectados contra tentativas de explorar esta vulnerabilidade.
  
    -   Interactivamente
  
        Siga os seguintes passos:
  
        1.  Clique em **Iniciar** e, em seguida, clique em **Painel de Controlo**. Alternativamente, seleccione **Definições** e depois clique em **Painel de Controlo**.  
        2.  Faça duplo clique em **Ferramentas Administrativas**. Como alternativa, clique em **Mudar para a vista clássica** e a seguir clique duas vezes sobre **Ferramentas administrativas**.  
        3.  Faça duplo clique em **Serviços**.  
        4.  Faça duplo clique em **Notificação de Eventos do Sistema**.  
        5.  Na lista **Tipo de arranque**, clique em **Desactivado**.  
        6.  Clique em **Parar** e a seguir em **OK**.

        sc stop sens & sc config sens start= disabled

#### Perguntas Mais Frequentes sobre a Vulnerabilidade no Sistema de Eventos - CVE-2008-1456
  
**Qual é a abrangência desta vulnerabilidade?**    
Existe uma vulnerabilidade de execução remota de código no Sistema de Eventos do Microsoft Windows. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios administrativos.
  
**O que provoca a vulnerabilidade?**    
O Sistema de Eventos do Microsoft Windows utiliza um parâmetro de métodos como índice num array de ponteiros de funções estático. A gama de índices não é validada correctamente, o que poderia permitir a execução de código arbitrário no contexto do sistema local.
  
**O que é o Sistema de Eventos do Microsoft Windows?**    
O Sistema de Eventos do Microsoft Windows é um serviço que gere chamadas de método e subscrições de eventos entre o Windows e aplicações existentes no sistema.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade poderia obter controlo total sobre o sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas com todos os privilégios administrativos.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Para explorar esta vulnerabilidade, um intruso teria de estar autenticado. Nesse caso, o intruso teria de chamar o Sistema de Eventos do Microsoft Windows e fornecer um pedido especialmente concebido para o efeito para além dos limites do array. Encontrando um ponteiro para uma localização previsível relativa ao array, um intruso poderia executar código com privilégios elevados e obter o controlo total sobre o sistema afectado.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Todos os sistemas que executem versões afectadas do Sistema de Eventos do Microsoft Windows estão susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização remove a vulnerabilidade modificando a forma como o Sistema de Eventos do Microsoft Windows valida gamas de índices, de modo a que as subscrições não executem código no contexto de segurança de um publicador de eventos.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

#### Informações de Implementação
  
**Instalação da actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.
  
Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684/pt).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro         | Descrição                                                                                                                                            |  
|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**        | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte o [Artigo 934307 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/934307).
  
**Verificar se a actualização foi aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização em **Iniciar pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
