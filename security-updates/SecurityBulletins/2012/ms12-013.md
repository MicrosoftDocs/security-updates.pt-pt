Security Bulletin

Microsoft Security Bulletin MS12-013 - Crítica
==============================================

Vulnerabilidade na Biblioteca de Runtime em C Poderia Permitir Execução Remota de Código (2654428)
--------------------------------------------------------------------------------------------------

Data de publicação: 14 de fevereiro de 2012

**actualizada:** 1.0

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade no Microsoft Windows, comunicada de forma privada. A vulnerabilidade poderia permitir execução remota de código se um utilizador abrisse um ficheiro multimédia especialmente concebido para o efeito alojado num Web site ou enviado como anexo de correio electrónico. Um intruso que explorasse a vulnerabilidade com sucesso poderia obter os mesmos direitos que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Esta actualização de segurança está classificada como Crítica para todas as edições suportadas do Windows Vista, Windows Server 2008, Windows 7 e Windows Server 2008 R2. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade modificando a forma como a biblioteca de ligação dinâmica (DLL) msvcrt calcula o tamanho de estruturas de dados na memória. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para obter informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem imediatamente a actualização, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões conhecidas.** O [Artigo 2654428 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/2654428) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

Vulnerabilidade de Sobrecarga de Memória Intermédia na Msvcrt.dll - CVE-2012-0150  
---------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código na forma como a DLL msvcrt calcula o tamanho de uma memória intermédia na memória, permitindo que sejam copiados dados para a memória que não foram correctamente atribuídos. Esta vulnerabilidade poderia permitir execução remota de código se um utilizador abrisse um ficheiro multimédia especialmente concebido para o efeito. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2012-0150](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2012-0150).
  
#### Factores Atenuantes para a Vulnerabilidade de Sobrecarga de Memória Intermédia na Msvcrt.dll - CVE-2012-0150
  
O factor atenuante refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso que explorasse com sucesso esta vulnerabilidade poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.  
-   No cenário de um ataque baseado na Web, um intruso poderia alojar um Web site contendo uma página Web que fosse utilizada para explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos ou anúncios fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Contudo, em qualquer dos casos, o intruso não teria forma de forçar os utilizadores a visitar estes Web sites. Em vez disso, um intruso teria de convencer os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico ou num programa de mensagens instantâneas, que levasse o utilizador para o Web site do intruso.
  
#### Soluções Alternativas para a Vulnerabilidade de Sobrecarga de Memória Intermédia na Msvcrt.dll - CVE-2012-0150
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Sobrecarga de Memória Intermédia na Msvcrt.dll - CVE-2012-0150
  
**Qual é a abrangência desta vulnerabilidade?**  
Esta é uma vulnerabilidade de execução remota de código no contexto do utilizador local.
  
**O que provoca a vulnerabilidade?**  
A vulnerabilidade é causada quando a msvcrt.dll calcula incorrectamente o tamanho de uma memória intermédia, permitindo que sejam copiados dados para a memória que não foram correctamente atribuídos.
  
**O que é a** **msvcrt.dll?**  
A Msvcrt.dll é uma biblioteca de ligação dinâmica multithread de runtime em C que é utilizada por componentes ao nível do sistema. Para mais informações sobre a msvcrt.dll, consulte o artigo MSDN sobre [Bibliotecas de Runtime em C](http://msdn.microsoft.com/en-us/library/abx4dbyh.aspx).
  
**Como poderia um intruso utilizar a vulnerabilidade?**  
Um intruso não autenticado que conseguisse tirar partido desta vulnerabilidade com êxito poderia controlar totalmente o sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
**Como poderia um intruso explorar a** **vulnerabilidade?**  
Num cenário de ataque com base em correio electrónico, um intruso não autenticado poderia explorar a vulnerabilidade enviando a um utilizador uma mensagem de correio electrónico contendo um ficheiro multimédia especialmente concebido para o efeito e convencendo-o a abrir o ficheiro multimédia.
  
No cenário de um ataque baseado na Web, um intruso não autenticado teria de alojar um Web site que contivesse um ficheiro multimédia especialmente concebido para o efeito. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam ser utilizados para explorar esta vulnerabilidade. Um intruso não teria como obrigar os utilizadores a visitar um Web site que alojasse o ficheiro multimédia especialmente concebido para o efeito. Em vez disso, um intruso teria de convencer os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico ou num programa de mensagens instantâneas, que levasse o utilizador para o Web site do intruso.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**  
As estações de trabalho e os servidores estão mais susceptíveis a esta vulnerabilidade.
  
**As aplicações de outros fabricantes são afectadas por esta vulnerabilidade?**  
Sim, qualquer aplicação que utiliza a msvcrt.dll poderia ser afectada por esta vulnerabilidade. A Microsoft recomenda a aplicação desta actualização de segurança para ajudar a solucionar esta vulnerabilidade em edições afectadas do Microsoft Windows. Para mais informações sobre esta questão, consulte a secção Questões Conhecidas com esta actualização de segurança, no [Artigo 2654428 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/2654428).
  
**O que faz a actualização?**  
A actualização resolve a vulnerabilidade modificando a forma como a DLL msvcrt calcula o tamanho de estruturas de dados na memória.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação coordenada de vulnerabilidades.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**  
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.

Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
