Security Bulletin

Microsoft Security Bulletin MS07-005 - Importante
=================================================

Vulnerabilidade no Step-by-Step Interactive Training Poderia Permitir Execução Remota de Código (923723)
--------------------------------------------------------------------------------------------------------

Data de publicação: 13 de fevereiro de 2007 | Updated: 8 de maio de 2007

**actualizada:** 1.1

#### Resumo

**Quem deve ler este documento:** Os clientes que utilizam o Microsoft Windows e têm o Step-by-Step Interactive Training instalado

**Impacto da Vulnerabilidade:** Execução Remota de Código

**Classificação de Gravidade Máxima:** Importante

**Recomendação:** Os clientes devem aplicar a actualização na primeira oportunidade

**Substituição de Actualizações de Segurança:** Este boletim substitui uma actualização de segurança anterior. Consulte a secção de Perguntas mais frequentes deste boletim para obter a lista completa.

**Avisos:** Nenhum

**Localizações de Transferência de Actualizações de Segurança e Software Testado:**

**Software Afectado:**

-   Step-by-Step Interactive Training quando instalado no Microsoft Windows XP Service Pack 2 — [Transferir a actualização](http://www.microsoft.com/downloads/details.aspx?familyid=128c57af-663a-4476-92f5-aab394cfc91a)

### Informações Gerais

Resumo Executivo
----------------

<span></span>
**Resumo Executivo:**

Esta actualização resolve uma vulnerabilidade recentemente descoberta e comunicada de forma privada. O Step-by-Step Interactive Training tem uma vulnerabilidade de execução remota de código que poderia permitir que um intruso obtivesse controlo total sobre um sistema afectado. A vulnerabilidade está documentada na secção "Detalhes sobre a Vulnerabilidade" deste boletim.

Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Recomendamos que os clientes apliquem a actualização na primeira oportunidade.

Detalhes Sobre a Vulnerabilidade
--------------------------------

<span></span>
#### Vulnerabilidade no Interactive Training - CVE-2006-3448:

Existe uma vulnerabilidade de execução remota de código no Step-by-Step Interactive Training devido à forma como este gere ficheiros de ligação de marcadores. Um intruso poderia explorar a vulnerabilidade criando uma ligação de marcadores especialmente concebida para o efeito, que poderia potencialmente permitir a execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. No entanto, é necessária interacção por parte do utilizador para explorar esta vulnerabilidade.

#### Factores Atenuantes para a Vulnerabilidade no Interactive Training - CVE-2006-3448:

-   No cenário de um ataque baseado na Web, um intruso poderia alojar um Web site contendo uma página Web que fosse utilizada para explorar esta vulnerabilidade. Além disso, Web sites que aceitam ou alojam publicidade ou conteúdos fornecidos pelo utilizador e Web sites nocivos podem conter conteúdos nocivos que poderiam explorar esta vulnerabilidade. Contudo, em qualquer dos casos, o intruso não teria forma de forçar os utilizadores a visitar estes Web sites. Em vez disso, um intruso teria de persuadir os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico, ou numa mensagem de um programa de mensagens instantâneas, que levasse o utilizador para o Web site do intruso.
-   Um intruso que explorasse com sucesso esta vulnerabilidade poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
-   A vulnerabilidade não pode ser automaticamente explorada através de correio electrónico. Para que um ataque tenha sucesso, um utilizador tem de abrir um anexo que é enviado numa mensagem de correio electrónico ou tem de clicar numa ligação fornecida numa mensagem de correio electrónico.

#### Alternativas para a Vulnerabilidade no Interactive Training - CVE-2006-3448:

A Microsoft testou as seguintes soluções alternativas. Apesar de estas soluções alternativas não corrigirem a vulnerabilidade subjacente, elas ajudam a bloquear vectores de ataque conhecidos. Na secção seguinte identificam-se os casos em que uma solução alternativa reduz a funcionalidade.

-   **Desactivar o processador de ficheiros de ligação de marcadores do Step-by-Step Interactive Training, removendo as chaves de registo relacionadas.**
    Elimine estas chaves de registo para ajudar a reduzir os ataques. Esta solução alternativa ajuda a reduzir os ataques impedindo que o Step-by-Step Interactive Training abra automaticamente os tipos de ficheiros afectados. Ainda assim, o conteúdo pode ser aberto a partir da interface de utilizador do Step-by-Step Interactive Training.
    **Importante** Este boletim contém informações sobre a forma de modificar o registo. Certifique-se de que faz uma cópia de segurança do registo antes de o modificar. Certifique-se de que sabe como restaurar o registo se ocorrer um problema. Para mais informações sobre criação de cópias de segurança, restauro e modificação de registos, consulte o [Artigo 256986 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/256986).
    **Aviso** Podem ocorrer problemas sérios se modificar o registo incorrectamente usando o Editor de Registo ou usando outro método. Estes problemas podem obrigar à reinstalação do seu sistema operativo. A Microsoft não pode garantir que esses problemas possam ser resolvidos. A modificação da chave de registo é da sua responsabilidade.

    1.  Clique em **Iniciar**, clique em **Executar**, escreva **regedt32** e, depois, clique em **OK**.
    2.  No Editor de Registo, localize o seguinte registo:

        `HKEY_CLASSES_ROOT\.cbl (para “Microsoft Press Interactive Training”)HKEY_CLASSES_ROOT\.cbm (para “Interactive Training”)HKEY_CLASSES_ROOT\.cbo (para “Microsoft Interactive Training ”)`

    3.  Para cada subchave encontrada, clique na subchave e, depois, clique em **DELETE**.
    4.  Na caixa de diálogo Confirmar Eliminação da Chave, clique em **OK**.

    Estas acções também podem ser executadas numa linha de comandos utilizando os seguintes comandos por esta ordem:

    `reg.exe export HKCR\.cbl c:\cbl.regreg.exe delete HKCR\.cbl /freg.exe export HKCR\.cbm c:\cbm.regreg.exe delete HKCR\.cbm /freg.exe export HKCR\.cbo c:\cbo.regreg.exe delete HKCR\.cbo /f`

    **Impacto da solução alternativa:** Os ficheiros de marcadores do Step-by-Step Interactive Training já não podem ser abertos. Ainda assim, o conteúdo pode ser aberto a partir da interface de utilizador do Step-by-Step Interactive Training.

-   **Não abra ou guarde ficheiros de ligação de marcadores do Step-by-Step Interactive Training (.cbo, .cbl, .cbm) que receba de fontes não fidedignas.**
    Esta vulnerabilidade poderia ser explorada quando um utilizador abrisse um ficheiro .cbo, .cbl, ou .cbm. Não abra ficheiros que utilizem estas extensões de nome de ficheiro. Esta solução alternativa não cobre outros vectores de ataque, como o browser da Web.
-   **Remova o Step-by-Step Interactive Training utilizando a ferramenta Adicionar ou Remover Programas no Painel de Controlo.**
    Para remover manualmente o Step-by-Step Interactive Training de um sistema, siga estes passos.

    1.  Clique em **Iniciar**, seleccione **Definições** e clique no **Painel de Controlo**.
    2.  Clique duas vezes em **Adicionar ou Remover Programas**.
    3.  Na caixa de diálogo **Adicionar ou Remover Programas**, clique no nome do programa afectado e, depois, clique em **Remover**.

    **Nota** As versões afectadas são a "Microsoft Press Interactive Training" e a "Interactive Training". Contudo, remover estes programas pode não ser uma solução alternativa completa, porque o "Microsoft Interactive Training" não cria uma entrada em Adicionar ou Remover Programas. O "Microsoft Interactive Training" baseia-se no ficheiros Orun32.exe. Portanto, também tem de verificar manualmente se o ficheiro Orun32.exe não está presente no seu sistema.

-   Siga as instruções para completar a remoção.

    **Impacto da solução alternativa:** Após remover a aplicação Step-by-Step Interactive Training, quaisquer aplicações que dependam desta irão falhar.

-   **Remova o Step-by-Step Interactive Training.**
    Remover o Step-by-Step Interactive Training ajudará a impedir ataques.Para remover o Step-by-Step Interactive Training, siga os seguintes passos:

    -   Clique em **Iniciar**, de seguida em **Executar** e depois **escreva**:

        %windir%\\IsUninst.exe -x -y -a -f"%windir%\\orun32.isu"

    **Nota** Pode ter de substituir "orun32.isu" por "mrun32.isu" ou "lrun32.isu", dependendo da versão do Step-by-Step Interactive Training instalada. Se tiver várias destas versões instaladas, tem de removê-las todas.

    **Impacto da solução alternativa:** Após remover a aplicação Step-by-Step Interactive Training, quaisquer aplicações que dependam desta irão falhar.

-   **Elimine ou dê outro nome ao ficheiro .ini do programa Step-by-Step Interactive Training.**
    Se o Step-by-Step Interactive Training não puder ser removido utilizando os métodos que estão documentados nesta secção do boletim de segurança, pode ajudar a impedir ataques eliminando ou dando outro nome ao ficheiro físico. Elimine ou dê outro nome ao ficheiro %windir%\\Orun32.ini.

    **Nota** Pode ter de substituir "Orun32.ini" por "Lrun32.ini" ou "Mrun32.ini", dependendo da versão do Step-by-Step Interactive Training instalada.

    **Impacto da solução alternativa:** Após desactivar a aplicação Step-by-Step Interactive Training, quaisquer aplicações que dependam do Step-by-Step Interactive Training poderão falhar.

#### Perguntas mais frequentes sobre a Vulnerabilidade no Interactive Training - CVE-2006-3448:

**Qual é a abrangência desta vulnerabilidade?**
Esta é uma vulnerabilidade de execução remota de código. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

**O que provoca a vulnerabilidade?**
Uma memória intermédia não verificada no processo que o Step-by-Step Interactive Training utiliza para validar ficheiros de ligação de marcadores.

**O que é um ficheiro de ligação de marcadores?**
Os ficheiros de ligação de marcadores são criados utilizando a interface de utilizador do Step-by-Step Interactive Training. Estes ficheiros permitem que um utilizador se ligue rápida e facilmente a um determinado tópico. Os ficheiros de ligação de marcadores são ficheiros de texto que contêm as informações necessárias para que o Step-by-Step Interactive Training visualize um tópico.

**O que é o Step-by-Step Interactive Training?**
O Step-by-Step Interactive Training é utilizado como motor para centenas de títulos de formação interactiva, fornecidos pela Microsoft Press e por outros vendedores. A lista de títulos conhecidos que contêm este software é fornecida no [Artigo 898458 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/898458). Para obter mais informações sobre outros títulos da Microsoft Press disponíveis que possam conter este software, consulte o Web site da Microsoft Press. Também pode utilizar as informações fornecidas na pergunta mais frequente "Como posso saber se tenho o Step-by-Step Interactive Training instalado no meu sistema?" para procurar ficheiros afectados na sua empresa.

**Como poderia um intruso utilizar a vulnerabilidade?**
Um intruso que tirasse partido desta vulnerabilidade com êxito poderia controlar totalmente o sistema afectado.

**Quem poderia explorar a vulnerabilidade?**
Um intruso que pudesse construir um ficheiro especialmente concebido e depois persuadir um utilizador a visitar um Web site malicioso que abrisse este ficheiro, ou um intruso que pudesse persuadir um utilizador a abrir um anexo especialmente concebido para o efeito fornecido numa mensagem de correio electrónico, poderia tentar explorar esta vulnerabilidade.

**Como poderia um intruso explorar a vulnerabilidade?**
Um intruso poderia tentar explorar a vulnerabilidade criando uma mensagem especialmente concebida para o efeito e enviando a mensagem ao sistema afectado. A mensagem poderia, então, fazer com que o sistema afectado executasse código.

Existem várias formas adicionais através das quais um intruso poderia tentar explorar esta vulnerabilidade. No entanto, em qualquer uma desta formas é necessária interacção do utilizador para explorar esta vulnerabilidade. Seguem-se alguns exemplos:

-   Um intruso poderia explorar a vulnerabilidade construindo um ficheiro nocivo de marcador do Step-by-Step Interactive Training (um ficheiro .cbo, cbl, ou .cbm) e depois persuadindo o utilizador a abrir o ficheiro.
-   Um intruso poderia enviar um ficheiro nocivo como anexo a um utilizador através de correio electrónico e depois convencê-lo a abrir o anexo.
-   Um intruso poderia alojar um Web site mal intencionado concebido para tirar partido desta vulnerabilidade através do Internet Explorer, persuadindo depois um utilizador a ver o Web site em questão.
-   Num cenário de ataque com base na Web, um intruso teria de alojar um Web site contendo uma página Web que fosse utilizada para tirar partido desta vulnerabilidade. Um intruso também poderia tentar comprometer um Web site fazendo-o apresentar uma página Web com conteúdo nocivo, para tentar explorar esta vulnerabilidade. Um intruso não teria como obrigar os utilizadores a visitar um Web site. Em vez disso, um intruso teria de os persuadir a visitar o Web site, normalmente fazendo-os clicar numa ligação que os leva ao Web site do intruso ou a um Web site que tenha sido comprometido pelo intruso.

**Quais são os sistemas mais susceptíveis face a esta vulnerabilidade?**
Qualquer sistema operativo onde o Step-by-Step Interactive Training esteja instalado está susceptível a esta vulnerabilidade. Como, normalmente, este software só está instalado em sistemas clientes, os servidores não estariam susceptíveis a esta vulnerabilidade.

**O que faz a actualização?**
A actualização remove a vulnerabilidade modificando a forma como o Step-by-Step Interactive Training valida os conteúdos de um ficheiro de marcadores antes de os copiar para a memória intermédia atribuída.

**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.

**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios de que esta vulnerabilidade estava a ser explorada?**
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

        **Nota** Dependendo da versão do sistema operativo, ou dos programas instalados, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no computador, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

-   **Verificação da Chave de Registo**

    Poderá também verificar os ficheiros instalados por esta actualização de segurança revendo a seguinte chave de registo:

    HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\StepbyStepInteractiveTraining\\KB923723\\Filelist

    **Nota** Esta chave de registo poderá não conter uma lista completa dos ficheiros instalados. Além disso, esta chave de registo poderá não ser criada correctamente se um administrador ou um OEM integrar ou associar a actualização de segurança 923723 aos ficheiros de origem de instalação do Windows.
