Security Bulletin

Microsoft Security Bulletin MS07-047 - Importante
=================================================

Vulnerabilidades no Windows Media Player poderiam permitir a Execução Remota de Código (936782)
-----------------------------------------------------------------------------------------------

Data de publicação: 14 de agosto de 2007 | Updated: 12 de agosto de 2008

**actualizada:** 2.0

### Informações Gerais

#### Resumo Executivo

Esta importante actualização de segurança resolve duas vulnerabilidades comunicadas de forma privada. Estas vulnerabilidades poderiam permitir a execução de código se um utilizador visse um ficheiro especialmente concebido para o efeito no Windows Media Player. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Esta é uma actualização de segurança importante para as versões suportadas do Windows Media Player 7.1, 9, 10 e 11. Para mais informações consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

Para mais informações sobre as vulnerabilidades, consulte a subsecção de Perguntas Mais Frequentes quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes apliquem a actualização.

**Questões conhecidas.** O [Artigo 936782 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/936782) documenta as questões actualmente conhecidas que os clientes podem experimentar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

#### Software Afectado e Software Não Afectado

O software aqui indicado foi testado para determinar quais as versões ou edições afectadas. As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Para determinar o ciclo de vida de suporte da versão ou edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

A Microsoft reviu este boletim de segurança para adicionar o Windows XP Service Pack 3 como software afectado, em que o Windows Media Player 9, o Windows Media Player 10 e o Windows Media Player 11 são executados. **Quais são as questões conhecidas que os clientes poderão experimentar quando instalam esta actualização de segurança?**

Vulnerabilidade de Execução de código na análise de máscaras no Windows Media Player – CVE-2007-3037  
----------------------------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução de código na análise de máscaras no Windows Media Player. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte o boletim [CVE-2007-3037](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2007-3037).
  
#### Factores atenuantes para a Vulnerabilidade de Execução de código na análise de máscaras no Windows Media Player – CVE-2007-3037
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Quando um utilizador tenta instalar um ficheiro de máscara do Windows Media Player, o utilizador vê uma caixa de diálogo de "Transferência" antes de a máscara ser aplicada.  
-   As tentativas de explorar a vulnerabilidade usando ficheiros WMZ e WMD exigem que o utilizador veja ou aplique a máscara depois de a ter transferido, para que o seu sistema fique vulnerável.
  
#### Soluções alternativas para a Vulnerabilidade de Execução de código na análise de máscaras no Windows Media Player – CVE-2007-3037
  
**Anular a associação das extensões de ficheiro WMZ e WMD**
  
A anulação da associação das extensões de ficheiro WMZ e WMD no Windows impede a pré-visualização ou abertura de ficheiros WMZ e WMD no Windows Media Player.
  
1.  Clique em **Iniciar**, seleccione a opção **Executar**, introduza "**regedit**" (sem as aspas) e clique em OK.  
2.  Expanda **HKEY\_CLASSES\_ROOT** e clique em .**WMZ**. A seguir, clique com o botão direito e seleccione **Exportar**. Nota: Isto criará uma cópia de segurança desta chave de registo, que, por predefinição, ficará na pasta ‘Os meus documentos’.  
3.  Na janela **Exportar ficheiro de registo**, escreva “Cópia de segurança de registo com a associação de extensão WMZ.reg” e prima **Guardar**.  
    Isto criará uma cópia de segurança desta chave de registo, que, por predefinição, ficará na pasta ‘Os meus documentos’.  
4.  Prima a tecla **Delete** no teclado para apagar a chave de registo. Seleccione **Sim** para confirmar a eliminação da chave de registo.  
5.  Repita os passos de 2 a 5 para a chave **WMD**.
  
**Impacto da solução alternativa**: Esta solução alternativa impede que os utilizadores apliquem ficheiros de máscaras ao Windows Media Player clicando duas vezes sobre eles. Os utilizadores continuam a poder aplicar ficheiros de máscaras que estejam na pasta de "máscaras" predefinida.
  
**Anular o registo Wmp.dll**
  
1.  Anular a chave de registo Wmp.dll ajuda a proteger o sistema afectado contra tentativas de explorar esta vulnerabilidade. Para modificar a chave de registo Wmp.dl, execute os seguintes passos.  
2.  Clique em **Iniciar**, depois em **Executar**, escreva **"regsvr32 -u %windir%\\system32\\wmp.dll"** (sem as aspas) e clique em **OK**.  
3.  Quando aparecer uma caixa de diálogo que confirma que o processo foi concluído com êxito, clique em **OK**.  
4.  Seleccione o separador **Tipos de ficheiros**.
  
**Impacto da solução alternativa**: Esta solução alternativa desactiva o Windows Media Player e as aplicações que utilizam o Controlo ActiveX do Windows Media.
  
#### Perguntas mais frequentes para a Vulnerabilidade de Execução de código na análise de máscaras no Windows Media Player – CVE-2007-3037
  
**Qual é a abrangência desta vulnerabilidade? **  
Existe uma vulnerabilidade de execução de código na análise de máscaras no Windows Media Player. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Em seguida, poderia instalar programas ou visualizar, alterar ou apagar dados.
  
**O que provoca a vulnerabilidade? **  
O Windows Media Player processa de forma incorrecta a informação de cabeçalho contida nos ficheiros de máscara.
  
**O que é um ficheiro de máscara? **  
As máscaras são conjuntos de scripts, criações artísticas, recursos multimédia e ficheiros de texto que podem ser combinados para criar uma nova imagem global para o Windows Media Player. A utilização de máscaras permite não só mudar o aspecto, mas também a forma de funcionamento do Windows Media Player. Os ficheiros de máscara do Windows Media Player são distribuídos em ficheiros WMZ e WMD.
  
**Como poderia um intruso utilizar a vulnerabilidade? **  
Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Os utilizadores cujas contas são configuradas para terem menos direitos de utilizador no sistema operativo convidado sofrem menos impacto do que os utilizadores que funcionam com direitos administrativos no sistema operativo convidado.
  
**Como poderia um intruso explorar a vulnerabilidade? **  
Num cenário de ataque pela Web, um intruso poderia alojar um ficheiro de máscara especialmente concebido para o efeito para explorar esta vulnerabilidade através do Windows Media Player e depois convencer um utilizador a ver os ficheiros de máscara. Contudo, em nenhuma situação um intruso poderia obrigar os utilizadores a visitar estes Web sites. Em vez disso, o intruso teria de persuadir os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação numa mensagem de correio electrónico, ou num pedido de mensagem instantânea que leva o utilizador para o Web site do intruso.
  
Num cenário de ataque com base em correio electrónico, um intruso poderia explorar a vulnerabilidade enviando um ficheiro de máscara especialmente concebido para o efeito ao utilizador e persuadindo-o a abrir o ficheiro.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade? **  
Para que qualquer acção nociva seja executada, esta vulnerabilidade requer que o utilizador tenha sessão iniciada e abra o ficheiro de máscara nocivo no Windows Media Player. Deste modo, qualquer sistema onde o Windows Media Player seja frequentemente utilizado, como as estações de trabalho ou servidores de terminal dos utilizadores, está mais susceptível a esta vulnerabilidade.
  
**O que faz a actualização? **  
A actualização remove a vulnerabilidade ao alterar a forma como o Windows Media Player processa a informação de cabeçalho contida nos ficheiros de máscara.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente? **  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.

Num cenário de ataque pela Web, um intruso poderia alojar um ficheiro de máscara especialmente concebido para o efeito para explorar esta vulnerabilidade através do Windows Media Player e depois convencer um utilizador a ver os ficheiros de máscara. Contudo, em nenhuma situação um intruso poderia obrigar os utilizadores a visitar estes Web sites. Em vez disso, o intruso teria de persuadir os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação numa mensagem de correio electrónico, ou num pedido de mensagem instantânea que leva o utilizador para o Web site do intruso.
  
Num cenário de ataque com base em correio electrónico, um intruso poderia explorar a vulnerabilidade enviando um ficheiro de máscara especialmente concebido para o efeito ao utilizador e persuadindo-o a abrir o ficheiro.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade? **  
Para que qualquer acção nociva seja executada, esta vulnerabilidade requer que o utilizador tenha sessão iniciada e abra o ficheiro de máscara nocivo no Windows Media Player. Deste modo, qualquer sistema onde o Windows Media Player seja frequentemente utilizado, como as estações de trabalho ou servidores de terminal dos utilizadores, está mais susceptível a esta vulnerabilidade.
  
**O que faz a actualização? **  
A actualização remove a vulnerabilidade ao alterar a forma como o Windows Media Player processa a informação de cabeçalho contida nos ficheiros de máscara.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente? **  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

**Verificar se a actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "Orientações e ferramentas de detecção e implementação" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

    Poderá também verificar os ficheiros instalados por esta actualização de segurança consultando as chaves de registo listadas na **Tabela de referência**, nesta secção.

    Estas chaves de registo poderão não conter uma lista completa dos ficheiros instalados. Além disso, estas chaves de registo poderão não ser criadas correctamente se um administrador ou um OEM integrar ou associar esta actualização de segurança aos ficheiros de origem de instalação do Windows.
