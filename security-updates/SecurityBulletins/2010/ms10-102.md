=================================================

Vulnerabilidade em Hyper-V Pode Permitir Negação de Serviço (2345316)
---------------------------------------------------------------------

Data de publicação: 14 de dezembro de 2010 | Updated: 12 de janeiro de 2011

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade, comunicada de forma privada, no Windows Server 2008 Hyper-V e no Windows Server 2008 R2 Hyper-V. A vulnerabilidade poderia permitir negação de serviço se um pacote especialmente concebido fosse enviado ao VMBus por um utilizador autenticado numa das máquinas virtuais convidadas alojadas pelo Hyper-V Server. Um intruso deve ter credenciais de início de sessão válidas e ser capaz de enviar conteúdo especialmente concebido de uma máquina virtual convidada para explorar esta vulnerabilidade. A vulnerabilidade não poderia ser explorada remotamente ou por utilizadores anónimos.

Esta actualização de segurança está classificada como Importante para todas as edições baseadas em x64 suportadas do Windows Server 2008 e Windows Server 2008 R2. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade corrigindo a forma como o servidor Hyper-V valida pacotes mal formados enviados ao VMBus no interior das suas máquinas virtuais convidadas. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais, ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a actualização na primeira oportunidade, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões Conhecidas.**  O [Artigo 2345316 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/2345316) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

Perguntas Mais Frequentes relativas a Esta Actualização de Segurança
--------------------------------------------------------------------

<span></span>
**Não tenho o Hyper-V activado. Posso instalar esta actualização?** 
Não. A actualização apenas pode ser instalada em sistemas operativos afectados com a função do Hyper-V activada. Os sistemas sem a função do Hyper-V activada não são afectados pela vulnerabilidade.

**Vou receber esta actualização mesmo que não tenha o Hyper-V activado?**  
Esta actualização apenas será oferecida através das actualizações automáticas se a função do Hyper-V estiver activada. Isto aplica-se se o sistema operativo afectado foi ou não instalado utilizando a opção de instalação Server Core.

**Esta actualização tem de ser aplicada a máquinas virtuais convidadas do Hyper-V?**  
Esta actualização não deve ser aplicada em máquinas convidadas do Hyper-V, mas apenas nos servidores que têm a função do Hyper-V activada. Será oferecida através das actualizações automáticas apenas ao sistema do servidor Hyper-V e não no interior das máquinas virtuais convidadas.

**O que é o Hyper-V Server e é também afectado por esta vulnerabilidade?**  
O Microsoft Hyper-V Server é um produto independente que fornece uma solução de virtualização fiável e optimizada, que permite que as organizações melhorem a utilização do servidor e reduzam os custos. Visto que o Hyper-V Server é um produto independente dedicado, que contém apenas o Windows Hypervisor, o modelo de controlador do Windows Server e componentes de virtualização, proporciona uma ocupação pequena e custos gerais mínimos.

O Microsoft Hyper-V Server 2008, Microsoft Hyper-V Server 2008 Service Pack 2 e Microsoft Hyper-V Server 2008 R2 são afectados por esta vulnerabilidade. As respectivas actualizações do Windows Server 2008 e Windows Server 2008 R2 devem ser aplicadas para ajudar a resolver esta vulnerabilidade. Os clientes que têm as actualizações automáticas activadas receberão automaticamente estas actualizações para instalação.

**Quais são as questões conhecidas que os clientes poderão experimentar quando desinstalam esta actualização de segurança?**  
A desinstalação desta actualização exige dois reinícios, devido à presença de determinados binários oferecidos na actualização. Os sistemas que não são reiniciados duas vezes depois de desinstalar a actualização, poderão sifrer perda de conectividade de rede. Para mais informações sobre esta questão conhecida, consulte o [Artigo 2345316 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/2345316).

Vulnerabilidade de VMBus de Hyper-V - CVE-2010-3960  
---------------------------------------------------
  
<span></span>
Há uma vulnerabilidade no Windows Server 2008 Hyper-V e Windows Server 2008 R2 Hyper-V que poderia permitir uma negação de serviço, se um pacote especificamente concebido fosse enviado ao VMBus por um utilizador autenticado numa das máquinas virtuais convidadas alojadas pelo servidor Hyper-V. Um intruso tem que ter credenciais de início de sessão válidas e conseguir iniciar a sessão localmente numa máquina virtual convidada para explorar esta vulnerabilidade. A vulnerabilidade não poderia ser explorada remotamente ou por utilizadores anónimos.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-3960](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3960).
  
#### Factores atenuantes para a Vulnerabilidade de VMBus de Hyper-V - CVE-2010-3960
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso tem que ter credenciais de início de sessão válidas e conseguir iniciar a sessão localmente numa máquina virtual do Hyper-V para explorar esta vulnerabilidade. A vulnerabilidade não poderia ser explorada remotamente ou por utilizadores anónimos.
  
#### Soluções alternativas para a Vulnerabilidade de VMBus de Hyper-V - CVE-2010-3960
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas mais frequentes sobre a Vulnerabilidade de VMBus de Hyper-V - CVE-2010-3960
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de "negação de serviço". Um intruso que explorasse esta vulnerabilidade poderia fazer com que o Hyper-V Server afectado deixasse de responder e fosse necessário reiniciá-lo. Tenha em atenção que a vulnerabilidade na negação de serviço não permitiria que um intruso executasse o código ou elevasse os seus direitos de utilizador, mas poderia fazer com que o sistema afectado deixasse de aceitar pedidos.
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade é provocada quando o servidor Hyper-V valida incorrectamente pacotes especialmente encapsulados enviados ao VMBus de Hyper-V. Devido a esta falta de validação, o processamento destes pacotes pode fazer com que a aplicação do servidor Hyper-V deixe de responder.
  
**O que é o Hyper-V?**    
O Hyper-V é uma tecnologia baseada em hipervisor, uma funcionalidade essencial do Windows Server 2008 e do Windows Server 2008 R2. Fornece uma plataforma de virtualização escalável, fiável e altamente disponível. Um ambiente de virtualização é útil porque os utilizadores podem criar e gerir máquinas virtuais, podendo executar vários sistemas operativos num computador físico e isolar os sistemas operativos entre si.
  
**O que é o VMBus?**    
O VMBus é um canal de comunicações de inter-partições lógico. Para obter mais informações sobre o VMBus e a arquitectura do Hyper-V, consulte o artigo de MSDN, [Arquitectura do Hyper-V](http://msdn.microsoft.com/en-us/library/cc768520(bts.10).aspx).
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade com sucesso poderia fazer com que o sistema de um utilizador deixasse de responder até ser reiniciado. Tenha em atenção que a exploração desta vulnerabilidade poderia fazer com que o próprio Hyper-V Server deixasse de responder, incluindo todas as máquinas virtuais convidadas que sejam alojadas por esse servidor.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso teria que ser um utilizador autenticado numa das máquinas virtuais convidadas alojadas pelo Hyper-V Server e necessitaria de ter capacidade para executar código arbitrário no sistema. Um intruso teria que criar um controlador personalizado que criasse pacotes especialmente concebidos para serem transmitidos ao VMBus, fazendo com que o servidor Hyper-V deixasse de responder.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
Os servidores Hyper-V que alojam máquinas virtuais convidadas que permitem acesso a utilizadores não fidedignos estão particularmente susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade corrigindo a forma como o servidor Hyper-V valida e processa pacotes especialmente criados transmitidos de máquinas virtuais convidadas para o VMBus de Hyper-V.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação coordenada de vulnerabilidades.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.

#### Informações de Implementação
  
**Instalação da Actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.
  
Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro         | Descrição                                                                                                                                            |  
|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**        | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte o [Artigo 934307 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/934307).
  
**Verificar Se a Actualização Foi Aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização na caixa **Iniciar Pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
