Security Bulletin

Microsoft Security Bulletin MS10-031 - Crítica
==============================================

Vulnerabilidade no Microsoft Visual Basic for Applications Poderia Permitir Execução Remota de Código (978213)
--------------------------------------------------------------------------------------------------------------

Data de publicação: 11 de maio de 2010 | Updated: 19 de maio de 2010

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade no Microsoft Visual Basic for Applications, comunicada de forma privada. A vulnerabilidade poderia permitir a execução remota de código se uma aplicação anfitriã abrisse e passasse um ficheiro especialmente concebido para o efeito para o runtime do Visual Basic for Applications. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Esta actualização de segurança está classificada como Crítica para todas as versões suportadas do Microsoft Visual Basic for Applications SDK e aplicações de outras empresas que utilizem o Microsoft Visual Basic for Applications. Esta actualização de segurança também está classificada como Importante para todas as edições suportadas do Microsoft Office XP, Microsoft Office 2003 e Microsoft Office System 2007. Para mais informações consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização resolve a vulnerabilidade modificando a forma como o Visual Basic for Applications procura controlos ActiveX incorporados nos documentos. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A Microsoft recomenda que os clientes apliquem de imediato a actualização.

**Questões conhecidas.** O [Artigo 978213 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/978213) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões. Quando as questões actualmente conhecidas e soluções recomendadas software dizem respeito apenas às edições específicas deste software, este artigo não fornece ligações a outros artigos.

#### Software Afectado e Software Não Afectado

O software que se segue foi testado para determinar quais as versões ou edições afectadas. As outras versões ou edições são antigas, ultrapassaram o respectivo ciclo de vida de suporte ou não são afectadas. Para determinar o ciclo de vida de suporte da versão ou edição do seu software, visite o [Web site do Ciclo de Vida de Suporte Microsoft](http://support.microsoft.com/lifecycle/).

Perguntas Mais Frequentes relativas a Esta Actualização de Segurança  
--------------------------------------------------------------------
  
<span></span>
**Por que razão foi revisto este boletim a 19 de Maio de 2010?**    
A Microsoft reviu este boletim para indicar mais claramente que as actualizações para o Microsoft Office se aplicam a todas as suites suportadas do Microsoft Office e a outro software Microsoft Office que contenha uma versão vulnerável do VBE6.dll. Isto inclui, para além de outras possíveis, versões suportadas do Microsoft Office Visio e do Microsoft Office Project. Isto também inclui software Office que contém o VBE6.dll, mas que não acede ao código vulnerável, como versões suportadas do Microsoft Office Outlook e do Microsoft Office OneNote.
  
Esta é apenas uma alteração de boletim. Não foram efectuadas alterações à lógica de detecção nem aos ficheiros de actualização. Os clientes que já tenham actualizado os seus sistemas com sucesso, incluindo clientes com as actualizações automáticas activadas, não necessitam de efectuar qualquer acção. Os clientes que não tenham instalado previamente esta actualização podem necessitar de reavaliar se os seus sistemas necessitam desta actualização, com base na tabela de Software Afectado revista e na informação fornecida nas Perguntas Mais Frequentes sobre a Actualização.
  
**A actualização destina-se a um componente partilhado utilizado pelo Microsoft Office. A actualização será oferecida para o software Microsoft Office que inclui o componente partilhado, mesmo que o software não aceda ao código vulnerável?**    
Sim, a actualização será oferecida para sistemas em que o componente do Office partilhado vulnerável é detectado. Isto é verdade mesmo para casos em que o software Office inclui o VBE6.dll, mas não acede ao código vulnerável. Consulte a seguinte tabela para exemplos de software Office ao qual esta actualização se aplica.
  
| Microsoft Office                                                                            | O VBE6.dll está incluído neste software? | O software acede ao código vulnerável?           |  
|---------------------------------------------------------------------------------------------|------------------------------------------|--------------------------------------------------|  
| Microsoft Office XP, Microsoft Office 2003, 2007 Microsoft Office System                    | Sim                                      | Sim, componentes como o Word, Excel e PowerPoint |  
| Microsoft Office OneNote 2003, Microsoft Office OneNote 2007                                | Sim                                      | Não                                              |  
| Microsoft Office Outlook 2003, Microsoft Office Outlook 2007                                | Sim                                      | Não                                              |  
| Microsoft Office Project 2002, Microsoft Office Project 2003, Microsoft Office Project 2007 | Sim                                      | Sim                                              |  
| Microsoft Office Visio 2002, Microsoft Office Visio 2003, Microsoft Office Visio 2007       | Sim                                      | Sim                                              |
  
**Nota** Utilize esta tabela apenas como uma orientação. Inclui uma amostra do software para o qual foram colocadas mais perguntas.
  
Em casos em que a actualização é oferecida, a Microsoft recomenda que os clientes instalem a actualização, independentemente de o software aceder ou não ao código vulnerável. Instalar a actualização ajudará a assegurar que os clientes executam a versão mais actualizada do VBE6.dll utilizada pelo software Microsoft Office.
  
**Esta oferta de actualização de uma versão não vulnerável do software constitui uma falha no mecanismo de actualização da Microsoft?**    
Não, o mecanismo de actualização está a funcionar correctamente na medida em que detecta uma versão inferior dos ficheiros no sistema comparativamente ao pacote de actualizações e, por conseguinte, fornece a actualização.
  
**Onde estão os pormenores sobre as informações do ficheiro?**    
Consulte as tabelas de referência na secção **Implementação da Actualização de Segurança** para obter a localização dos pormenores sobre as informações do ficheiro.
  
**Apliquei as actualizações de segurança necessárias da Microsoft, mas ainda tenho no meu sistema uma versão afectada do runtime do Visual Basic for Applications (VBE6.dll). Como actualizo este DLL?**    
Há casos em que o seu sistema pode ainda ter uma versão afectada do VBE6.dll, mesmo depois de ter instalado as actualizações de segurança necessárias para o Microsoft Office e a actualização para o Microsoft Visual Basic for Applications descrita neste boletim.
  
Se o VBE6.dll tiver sido instalado no seu sistema por uma versão suportada do Microsoft Office, a aplicação da actualização de segurança para a versão afectada do Microsoft Office substituirá o VBE6.dll pela versão actualizada que resolve a vulnerabilidade descrita neste boletim. Contudo, se o VBE6.dll tiver sido instalado no seu sistema por uma aplicação de outra empresa, poderá ter de instalar uma actualização para esse programa.
  
Há dois cenários possíveis para actualizar o VBE6.dll de aplicações de outras empresas, consoante a implementação do VBA dessa aplicação de outra empresa. Se souber que a aplicação de outra empresa está em conformidade com as melhores práticas recomendadas para utilizar um componente partilhado como uma assemblagem lado a lado, a aplicação da actualização do Microsoft Visual Basic for Applications (KB974945) substituirá o VBE6.dll na localização partilhada pela versão actualizada que resolve a vulnerabilidade descrita neste boletim.
  
Por outro lado, se a aplicação de outra empresa não colocar o VBE6.dll na localização partilhada de acordo com as melhores práticas recomendadas, deverá contactar o programador da aplicação da outra empresa e pedir que lhe seja fornecida uma versão actualizada da sua aplicação que contenha uma versão mais recente do VBE6.dll que resolva a vulnerabilidade descrita neste boletim. Tenha em atenção que nem todas as aplicações de outras empresas que instalam o VBE6.dll são afectadas pela vulnerabilidade descrita neste boletim, uma vez que a aplicação pode não utilizar o VBE6.dll para procurar controlos ActiveX de forma passível de exploração. No entanto, apenas o programador dessa aplicação poderá determinar isso.
  
**Esta actualização de segurança aplica-se apenas a software da Microsoft. Como posso detectar se uma aplicação de outra empresa implementou uma versão afectada do runtime do Visual Basic for Applications (VBE6.dll) no meu sistema?**    
As aplicações de outras empresas que suportam o VBA podem implementar o VBE6.dll numa localização que não seja actualizada por esta actualização de segurança. No caso de ter uma aplicação de outra empresa cujo fornecimento tenha sido efectuado com a sua própria cópia do VBE6.dll, deverá contactar directamente o programador ou fornecedor responsável pelo apoio à aplicação da outra empresa, para o ajudar a assegurar que o seu sistema está completamente protegido contra a vulnerabilidade descrita neste boletim.
  
Para detalhes sobre como procurar cópias de VBE6.dll no seu sistema, consulte o [Artigo 978213 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/978213). A análise de todas as cópias do VBE6.dll no seu sistema pode ajudar a identificar cópias potencialmente vulneráveis que podem ter sido instaladas por aplicações de outros fabricantes.
  
**Faço desenvolvimento de aplicações e uso o runtime do Microsoft Visual Basic for Applications nas minhas aplicações. As minhas aplicações são vulneráveis? Como é que as posso actualizar?**    
Os programadores que redistribuem o runtime VBE6.dll do Microsoft Visual Basic for Applications devem certificar-se de que actualizam a versão do runtime do Microsoft Visual Basic for Applications que têm instalada na sua aplicação transferindo uma versão actualizada do Microsoft Visual Basic for Applications SDK a partir da [Summit Software Company](http://www.summsoft.com/). Para mais informações sobre as melhores práticas de uso de componentes redistribuídos, consulte o [Artigo 835322 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/835322) e o artigo MSDN sobre [Aplicações Isoladas e Assemblagens Lado a Lado](http://msdn.microsoft.com/en-us/library/aa375193.aspx).
  
**Sou um ISV. Onde está a actualização para o Microsoft Visual Basic for Applications SDK?**    
A actualização está disponível na [Summit Software Company](http://www.summsoft.com/). A Summit Software Company é um fornecedor global de produtos de software de personalização de aplicações e serviços de apoio a integração para fornecedores de software independentes (ISV) e programadores empresariais. Em Junho de 1996, a Summit Software acordou com a Microsoft a venda do Microsoft Visual Basic for Applications e de serviços e tecnologias de valor acrescentado relacionados. A Summit continua a vender e a prestar assistência ao Microsoft VBA.
  
**Por que razão esta actualização é Crítica para o Microsoft Visual Basic for Applications, mas apenas Importante para o Microsoft Office?**    
O Microsoft Office XP e versões posteriores dispõem de uma funcionalidade incorporada que solicita a permissão do utilizador para Abrir, Guardar ou Cancelar antes de abrir um documento. Este factor atenuante reduz a classificação da vulnerabilidade de Crítica para Importante, porque requer mais do que a acção de um único utilizador para que a exploração possa ser efectuada.
  
**Utilizo o Microsoft Office 2007 Service Pack 1. Estão incluídas funcionalidades de segurança adicionais nesta actualização?**     
Sim, como parte do modelo de prestação de serviços do Microsoft Office 2007, quando os utilizadores do Microsoft Office 2007 Service Pack 1 instalam esta actualização, os seus sistemas serão actualizados para a funcionalidade de segurança inicialmente lançada com o Microsoft Office 2007 Service Pack 2. Todas as actualizações disponibilizadas depois de 24 de Abril de 2009 para o Microsoft Office 2007 incluirão estas funcionalidades de segurança, introduzidas no Microsoft Office 2007 Service Pack 2. Esta actualização foi testada minuciosamente, mas tal como em todas as actualizações, recomendamos que os utilizadores executem os testes apropriados no respectivo ambiente e na configuração dos seus sistemas.
  
**Estou a usar uma edição anterior do software discutido neste boletim de segurança. O que devo fazer?**    
O software afectado incluído neste boletim foi testado para determinar quais as edições afectadas. As outras edições ultrapassaram o respectivo ciclo de vida de suporte. Para mais informações sobre o ciclo de vida dos produtos, visite o Web site do [Ciclo de Vida de Suporte da Microsoft](http://support.microsoft.com/lifecycle/).
  
Os utilizadores que tenham edições anteriores do software deviam considerar a migração para edições suportadas como uma prioridade, de modo a poderem proteger os seus sistemas contra a potencial exposição a vulnerabilidades. Para determinar o ciclo de vida de suporte para a edição do seu software, consulte [Seleccionar um produto para obter informações sobre o ciclo de vida](http://go.microsoft.com/fwlink/?linkid=169555). Para mais informações sobre service packs para estas edições de software, consulte [Service Packs com Suporte do Ciclo de Vida](http://go.microsoft.com/fwlink/?linkid=89213).
  
Os clientes que necessitem de suporte personalizado para software anterior devem contactar o seu representante de equipa de contas da Microsoft, o seu Gestor Técnico de Contas ou o parceiro ou representante Microsoft adequado para mais informações sobre opções de suporte personalizado. Os clientes que não disponham de um Contrato Alliance, Premier ou Autorizado podem contactar a subsidiária local da Microsoft. Para obter informações de contacto, visite o [Web site Microsoft Worldwide Information](http://go.microsoft.com/fwlink/?linkid=33329), seleccione o país na lista de informações de contacto e, em seguida, clique em **Go** para ver uma lista de números de telefone. Vulnerabilidade de Corrupção na Memória de Pilha VBE6.dll - CVE-2010-0815
-------------------------------------------------------------------------

<span></span>
Existe uma vulnerabilidade de execução remota de código na forma como o Microsoft Visual Basic for Applications procura controlos ActiveX. Esta vulnerabilidade poderia permitir a execução remota de código se uma aplicação anfitriã abrisse e passasse um ficheiro especialmente concebido para o efeito para o runtime do Visual Basic for Applications. Se um utilizador tiver sessão iniciada com privilégios administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-0815](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0815).

#### Factores Atenuantes para a Vulnerabilidade de Corrupção na Memória de Pilha VBE6.dll - CVE-2010-0815

A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:

-   Por predefinição, o Microsoft Office System 2007 alerta o utilizador com um aviso de segurança antes de activar um controlo ActiveX incorporado num documento do Office. Os utilizadores que optem por não activar o controlo estão protegidos por esta predefinição. Para mais informações, consulte o artigo do Microsoft Office Online sobre como [Activar ou desactivar controlos ActiveX em documentos do Office](http://office.microsoft.com/en-us/help/ha100310671033.aspx).
-   Um intruso que explorasse com sucesso esta vulnerabilidade poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
-   No cenário de um ataque baseado na Web, um intruso poderia alojar um Web site contendo uma página Web que fosse utilizada para explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos ou anúncios fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Contudo, em qualquer dos casos, o intruso não teria forma de forçar os utilizadores a visitar estes Web sites. Um intruso teria de convencer os utilizadores a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico, ou numa mensagem de um programa de mensagens instantâneas, que levasse os utilizadores para o Web site do intruso e, em seguida, convencendo-os a abrir o ficheiro Office especialmente concebido para o efeito.
-   A vulnerabilidade não pode ser automaticamente explorada através de correio electrónico. Para um ataque poder ser bem sucedido, é necessário que um utilizador abra um anexo recebido numa mensagem de correio electrónico.

#### Soluções Alternativas para a Vulnerabilidade de Corrupção na Memória de Pilha VBE6.dll - CVE-2010-0815

Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:

-   **Desactivar controlos ActiveX no Microsoft Office System 2007**

    Pode desactivar controlos ActiveX no Microsoft Office System 2007 utilizando o **Centro de Fidedignidade**. Consulte as instruções para saber como "Desactivar todos os controlos sem notificação", no artigo do Microsoft Office Online sobre como [Activar ou desactivar controlos ActiveX em documentos do Office](http://office.microsoft.com/en-us/help/ha100310671033.aspx).

    **Impacto da solução alternativa.** Os controlos ActiveX incorporados (como as macros) não serão executados em documentos do Office.

-   **Restringir o acesso ao VBE6.dll**

    Para o Microsoft Windows 2000, Windows XP e Windows Server 2003, introduza o seguinte numa linha de comandos:

    `cacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /E /P everyone:N`

    Para o Windows Vista, Windows Server 2008, Windows 7 e Windows Server 2008 R2, introduza o seguinte numa linha de comandos com privilégios elevados:

    `takeown /f "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll"icacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /save %TEMP%\VBE6_ACL.TXTicacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /deny everyone:(F)`

    **Impacto da solução alternativa.** Os controlos ActiveX incorporados (como as macros) não serão executados em documentos do Office. Por exemplo, os utilizadores não poderão inserir objectos em documentos do Office.

    **Como anular esta solução alternativa.**

    Para o Microsoft Windows 2000, Windows XP e Windows Server 2003, introduza o seguinte numa linha de comandos:

    `cacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\vbe6.dll" /E /R everyone`

    Para o Windows Vista, Windows Server 2008, Windows 7 e Windows Server 2008 R2, introduza o seguinte numa linha de comandos:

    `icacls "C:\Program Files\Common Files\Microsoft Shared\VBA\VBA6\" /restore %TEMP%\VBE6_ACL.TXT`

-   **Utilize a funcionalidade Microsoft Office Isolated Conversion Environment (MOICE) ao abrir ficheiros que provenham de fontes desconhecidas ou não fidedignas**

    O Microsoft Office Isolated Conversion Environment (MOICE) ajuda a proteger a instalação do Office 2003 ao abrir com mais segurança os ficheiros de formato binário do Word, Excel e PowerPoint.

    Para instalar o MOICE, necessita de ter instalado o Office System 2003 ou 2007.

    Para instalar o MOICE, necessita de ter o Pack de compatibilidade para os formatos de ficheiros Word, Excel e PowerPoint 2007. Este Pack de compatibilidade encontra-se disponível no Centro de Transferências da Microsoft:

    [Transfira agora o pacote FileFormatConverters.exe](http://www.microsoft.com/downloads/details.aspx?familyid=941b3470-3ae9-4aee-8f43-c6bb74cd1466&amp;displaylang=en)

    O MOICE necessita de todas as actualizações recomendadas para todos os programas do Office. Visite o Microsoft Update para instalar todas actualizações recomendadas:

    <http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=en-us>

    Para activar o MOICE, altere o processador registado para os formatos de ficheiro do Office. A tabela seguinte descreve o comando para activar ou desactivar o MOICE para os formatos de ficheiro do Office:

Além disso, os documentos com palavras-passe ou que estejam protegidos através de Gestão de Direitos Digitais não podem ser convertidos.

**Como anular esta solução alternativa. **Para obter informações sobre como anular esta solução alternativa, consulte o [Artigo 935865 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/935865).

-   Não abra ou guarde ficheiros do Microsoft Office que receba de fontes não-fidedignas, ou que recebeu de fontes fidedignas, mas que de alguma forma não correspondem ao que é habitual. É possível tirar partido desta vulnerabilidade quando um utilizador abre um ficheiro especialmente concebido para o efeito.

#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Corrupção na Memória de Pilha VBE6.dll - CVE-2010-0815

**Qual é a abrangência desta vulnerabilidade?**  
Esta é uma vulnerabilidade de execução remota de código no Visual Basic for Applications (VBA). Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

**O que provoca a vulnerabilidade?**  
A vulnerabilidade é provocada pela forma como o Visual Basic for Applications procura controlos ActiveX num documento que suporte o VBA. Por esse motivo, é possível que uma aplicação anfitriã, como o Microsoft Office ou uma aplicação de outra empresa, desenvolvida para programação do Visual Basic, seja transmitida para um documento especialmente concebido para o efeito com controlos ActiveX incorporados para o runtime do VBA, criando uma condição que poderia permitir a execução de código arbitrário.

**O que é o Visual Basic for Applications (VBA)?**  
O Microsoft VBA é uma tecnologia de programação para o desenvolvimento de aplicações compactadas de ambiente de trabalho de clientes e para as integrar em dados e em sistemas existentes. O Microsoft VBA baseia-se no sistema de programação Microsoft Visual Basic. Os produtos Microsoft Office incluem o VBA e utilizam-no para executar determinadas funções. O VBA também pode ser usado para construir aplicações personalizadas baseadas numa aplicação alojadora existente.

**Como poderia um intruso utilizar a vulnerabilidade?**  
Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

**Como poderia um intruso explorar a vulnerabilidade?**  
Para que um ataque pudesse ter sucesso, um utilizador teria de abrir um documento especialmente concebido para o efeito que suportasse o VBA. Este documento poderia ser de qualquer tipo que suportasse o VBA, como um documento Word, uma folha de cálculo Excel ou uma apresentação em PowerPoint.

Num cenário de ataque com base em correio electrónico, um intruso poderia explorar a vulnerabilidade enviando um ficheiro Office especialmente concebido para o efeito ao utilizador e convencendo-o a abrir o ficheiro.

Num cenário de ataque com base na Web, um intruso teria de alojar um Web site contendo um documentos que suportasse o VBA, como um ficheiro do Office, que fosse utilizado para tentar explorar esta vulnerabilidade. Além disso, os Web sites afectados e os Web sites que aceitam ou alojam conteúdos fornecidos por utilizadores poderiam conter conteúdos especialmente concebidos que poderiam explorar esta vulnerabilidade. Um intruso não conseguiria obrigar os utilizadores a visitarem um Web site especialmente concebido para o efeito. Um intruso teria de os convencer a visitar o Web site, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico, ou numa mensagem de um programa de mensagens instantâneas, que os levasse para o Web site do intruso e, em seguida, convencendo-os a abrir o ficheiro Office especialmente concebido para o efeito.

**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**  
Os sistemas mais susceptíveis são as estações de trabalho e os servidores de terminais em que o Microsoft Visual Basic for Applications ou o Microsoft Office são utilizados. Os servidores podem estar mais sujeitos a risco se os administradores permitirem que os utilizadores iniciem sessão nos servidores e executem programas. Contudo, os procedimentos recomendados desencorajam essa acção.

**O que faz a actualização?**  
A actualização resolve a vulnerabilidade modificando a forma como o Microsoft Visual Basic for Applications procura controlos ActiveX incorporados nos documentos.

**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.

**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**  
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.

Esta actualização de segurança suporta os seguintes parâmetros de configuração.

| Parâmetro   | Descrição                                                                                                                 |
|-------------|---------------------------------------------------------------------------------------------------------------------------|
| **/q**      | Especifica o modo silencioso ou suprime mensagens quando os ficheiros estão a ser extraídos.                              |
| **/q:u**    | Especifica o modo silencioso de utilizador, que apresenta algumas caixas de diálogo ao utilizador.                        |
| **/q:a**    | Especifica o modo silencioso de administrador, que não apresenta qualquer caixa de diálogo ao utilizador.                 |
| **/t:path** | Especifica a pasta de destino para a extracção de ficheiros.                                                              |
| **/c**      | Extrai os ficheiros sem os instalar. Se **/t:path** não for especificado, ser-lhe-á pedida uma pasta de destino.          |
| **/c:path** | Substitui o comando de instalação definido pelo autor. Especifica o caminho e nome do ficheiro Setup.inf ou .exe.         |
| **/r:n**    | Nunca reinicia o sistema após a instalação.                                                                               |
| **/r:I**    | Solicita ao utilizador que reinicie o sistema, se for necessário, excepto quando este parâmetro é utilizado com **/q:a**. |
| **/r:a**    | Reinicia sempre o sistema após a instalação.                                                                              |
| **/r:s**    | Reinicia o sistema após a instalação sem solicitar a confirmação do utilizador.                                           |
| **/n:v**    | Sem verificação da versão - Instalar o programa sobre qualquer versão anterior.                                           |

**Nota** É possível combinar estes parâmetros no mesmo comando.
