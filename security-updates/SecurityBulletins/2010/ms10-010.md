Security Bulletin

Microsoft Security Bulletin MS10-010 - Importante
=================================================

Vulnerabilidade no Windows Server 2008 Hyper-V Poderia Permitir Negação de Serviço (977894)
-------------------------------------------------------------------------------------------

Data de publicação: 9 de fevereiro de 2010 | Updated: 10 de fevereiro de 2010

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade, comunicada de forma privada, no Windows Server 2008 Hyper-V e no Windows Server 2008 R2 Hyper-V. A vulnerabilidade poderia permitir negação de serviço se uma sequência mal formada de instruções da máquina fosse executada por um utilizador autenticado numa das máquinas virtuais convidadas que seja alojada pelo Hyper-V Server. Um intruso tem que ter credenciais de início de sessão válidas e conseguir iniciar a sessão localmente numa máquina virtual convidada para explorar esta vulnerabilidade. A vulnerabilidade não poderia ser explorada remotamente ou por utilizadores anónimos.

Esta actualização de segurança está classificada como Importante para todas as edições baseadas em x64 suportadas do Windows Server 2008 e Windows Server 2008 R2. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade corrigindo a forma como o Hyper-V Server valida codificação em instruções de máquinas executadas dentro das respectivas máquinas virtuais convidadas. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais, ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a actualização na primeira oportunidade, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

Esta actualização não deve ser aplicada a máquinas convidadas do Hyper-V, mas apenas à máquina que tem a função do Hyper-V activada. Será oferecida através das Actualizações Automáticas apenas ao sistema Hyper-V Server e não dentro das máquinas virtuais convidadas.

**O que é o Hyper-V Server e em que medida é afectado por esta vulnerabilidade?**
O Microsoft Hyper-V Server é um produto independente que fornece uma solução de virtualização fiável e optimizada, que permite que as organizações melhorem a utilização do servidor e reduzam os custos. Visto que o Hyper-V Server é um produto independente dedicado, que contém apenas o Windows Hypervisor, o modelo de controlador do Windows Server e componentes de virtualização, proporciona uma ocupação pequena e custos gerais mínimos.
O Microsoft Hyper-V Server 2008, o Microsoft Hyper-V Server 2008 Service Pack 2 e o Microsoft Hyper-V Server 2008 R2 são afectados por esta vulnerabilidade e as actualizações do Windows Server 2008 e do Windows Server 2008 R2, respectivamente, devem ser aplicadas para ajudar a resolver esta vulnerabilidade. Os clientes que têm as Actualizações Automáticas activadas receberão automaticamente estas actualizações para instalação.

Vulnerabilidade de Validação de Conjuntos de Instruções no Hyper-V - CVE-2010-0026  
----------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de negação de serviço no Hyper-V no Windows Server 2008 e Windows Server 2008 R2. A vulnerabilidade deve-se a uma validação insuficiente de sequências específicas de instruções da máquina por parte do Hyper-V. Um intruso que explorasse esta vulnerabilidade com sucesso poderia fazer com que o sistema Hyper-V afectado deixasse de responder. Isto afectaria todas as máquinas virtuais alojadas por esse sistema.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-0026](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-0026).
  
#### Factores Atenuantes para a Vulnerabilidade de Validação de Conjuntos de Instruções no Hyper-V - CVE-2010-0026
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso tem que ter credenciais de início de sessão válidas e conseguir iniciar a sessão localmente numa máquina virtual do Hyper-V para explorar esta vulnerabilidade. A vulnerabilidade não poderia ser explorada remotamente ou por utilizadores anónimos.
  
#### Soluções Alternativas para a Vulnerabilidade de Validação de Conjuntos de Instruções no Hyper-V - CVE-2010-0026
  
A Microsoft não identificou quaisquer soluções alternativas para esta vulnerabilidade.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Validação de Conjuntos de Instruções no Hyper-V - CVE-2010-0026
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de "negação de serviço". Um intruso que explorasse esta vulnerabilidade poderia fazer com que o Hyper-V Server afectado deixasse de responder e fosse necessário reiniciá-lo. Tenha em atenção que a vulnerabilidade na negação de serviço não permitiria que um intruso executasse o código ou elevasse os seus direitos de utilizador, mas poderia fazer com que o sistema afectado deixasse de aceitar pedidos.
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade é provocada pelo facto de o Hyper-V Server validar incorrectamente a codificação de instruções de máquinas específicas executadas dentro das máquinas virtuais convidadas. Devido a esta falta de validação, o processamento destas instruções pode fazer com que a aplicação do Hyper-V Server deixe de responder.
  
**O que é o Hyper-V?**    
O Hyper-V é uma tecnologia baseada em hipervisor, uma funcionalidade essencial do Windows Server 2008 e do Windows Server 2008 R2. Fornece uma plataforma de virtualização escalável, fiável e altamente disponível. Um ambiente de virtualização é útil porque os utilizadores podem criar e gerir máquinas virtuais, podendo executar vários sistemas operativos num computador físico e isolar os sistemas operativos entre si.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade com sucesso poderia fazer com que o sistema de um utilizador deixasse de responder até ser reiniciado. Tenha em atenção que a exploração desta vulnerabilidade poderia fazer com que o próprio Hyper-V Server deixasse de responder, incluindo todas as máquinas virtuais convidadas que sejam alojadas por esse servidor.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso teria que ser um utilizador autenticado numa das máquinas virtuais convidadas alojadas pelo Hyper-V Server e necessitaria de ter capacidade para executar código arbitrário no sistema. Um intruso poderia então executar um executável não fidedigno no sistema que invocasse uma sequência mal formada de instruções da máquina e, assim, fazer com que o Hyper-V Server deixasse de responder.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Os servidores Hyper-V que alojam máquinas virtuais convidadas que permitem acesso a utilizadores não fidedignos estão particularmente susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade corrigindo a forma como o Hyper-V Server valida codificação em instruções de máquinas executadas dentro das respectivas máquinas virtuais convidadas.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.
