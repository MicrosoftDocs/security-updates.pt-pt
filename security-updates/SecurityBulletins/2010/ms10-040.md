Security Bulletin

Microsoft Security Bulletin MS10-040 - Importante
=================================================

Vulnerabilidade no Internet Information Services Poderia Permitir Execução Remota de Código (982666)
----------------------------------------------------------------------------------------------------

Data de publicação: 8 de junho de 2010

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade comunicada de forma privada no Internet Information Services (IIS). Esta vulnerabilidade poderia permitir a execução remota de código se um utilizador recebesse um pedido HTTP especialmente concebido para o efeito. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado.

Esta actualização de segurança está classificada como Importante para o IIS 6.0, IIS 7.0 e IIS 7.5. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade corrigindo a validação de autenticação. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais, ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a actualização na primeira oportunidade, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões conhecidas.**O [Artigo 982666 da Base de Conhecimentos da Microsoft](http://support.microsoft.com/kb/982666) reporta a questão conhecida, de que clientes após a instalação desta actualização de segurança podem verificar. O artigo também recomenda soluções para esta situação.

Perguntas Mais Frequentes relativas a Esta Actualização de Segurança
--------------------------------------------------------------------

<span></span>
**Que sistemas são afectados por esta vulnerabilidade?**  
Esta vulnerabilidade afecta os sistemas Windows 7 e Windows Server 2008 R2. Além disso, também afecta os sistemas Windows Server 2003, Windows Vista e Windows Server 2008 que tenham instalado a [KB973917](http://support.microsoft.com/kb/973917), uma actualização não relacionada com segurança que implementa a Protecção de Autenticação Alargada.

**O meu computador fica vulnerável se não tiver instalado a actualização KB973917?**  
Os sistemas que executam edições suportadas do Windows Server 2003, Windows Vista e Windows Server 2008 que não tenham a KB973917 instalada não são vulneráveis. Os sistemas que executam edições suportadas do Windows 7 e Windows Server 2008 R2 são vulneráveis.

Os sistemas que não têm a actualização KB973917 instalada não receberão esta actualização, uma vez que não são afectados por esta vulnerabilidade de segurança.

Vulnerabilidade de Corrupção de Memória na Autenticação IIS - CVE-2010-1256  
---------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código no Internet Information Services (IIS). A vulnerabilidade deve-se a uma análise incorrecta de informação de autenticação. Um intruso que explorasse esta vulnerabilidade com sucesso poderia executar código no contexto do WPI (Worker Process Identity).
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-1256](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-1256).
  
#### Factores Atenuantes para a Vulnerabilidade de Corrupção de Memória na Autenticação IIS - CVE-2010-1256
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Sem a instalação da KB973917 no Windows Server 2003, Windows Vista e Windows Server 2008, os sistemas não terão a funcionalidade de Protecção de Autenticação Alargada e não serão vulneráveis.  
-   A Protecção de Autenticação Alargada não está activada por predefinição nas plataformas afectadas, mesmo quando um sistema tem a KB973917 instalada. Os sistemas só são afectados quando esta funcionalidade está activada.
  
#### Soluções Alternativas para a Vulnerabilidade de Corrupção de Memória na Autenticação IIS - CVE-2010-1256
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Requerer Verificação por Token**
  
    Os sistemas que têm o Internet Information Services configurado para utilizar a Protecção Alargada para "Requerer" a verificação por token, como descrito no [Artigo 973917 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/973917), não expõem esta vulnerabilidade.
  
    **Impacto da solução alternativa.** Esta alteração de configuração não permitirá que os clientes que não implementem a Protecção Alargada se autentiquem.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Corrupção de Memória na Autenticação IIS - CVE-2010-1256
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso remoto autenticado que conseguisse tirar partido desta vulnerabilidade poderia executar acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity). Por predefinição, esta identidade está configurada para ser a conta Network Service no Windows Server 2003, Windows Vista e Windows Server 2008. Esta identidade está configurada para ser uma identidade virtual de conjuntos aplicacionais no Windows 7 e Windows Server 2008 R2.
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade é causada quando o servidor Web do Internet Information Services não atribui memória correctamente ao analisar informações de autenticação específicas recebidas do cliente.
  
**Qual é o componente afectado pela vulnerabilidade?**    
Esta vulnerabilidade afecta o servidor Web do Internet Information Services quando está configurado para suportar pedidos de autenticação utilizando apenas a Protecção de Autenticação Alargada. Para este componente ser vulnerável, a funcionalidade de Protecção Alargada deve estar configurada num estado não predefinido que permita a verificação por token.
  
**O que é a Protecção de Autenticação Alargada?**    
A Protecção de Autenticação Alargada ajuda a proteger contra ataques man-in-the-middle (MITM), em que um intruso intercepta as credenciais de um cliente e as reencaminha para um servidor. A Protecção de Autenticação Alargada está a ser implementada como actualizações não relacionadas com segurança que os administradores e utilizadores devem instalar para que lhes seja fornecida esta protecção adicional.
  
A Protecção Alargada está disponível como uma actualização não relacionada com segurança para o Windows Server 2003, Windows Vista e Windows Server 2008 (consulte o [Artigo 973917 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/973917)). O Windows 7 e o Windows Server 2008 R2 têm esta funcionalidade como um componente opcional no sistema operativo que é activada quando instala o serviço de função de Autenticação do Windows para servidor Web do Internet Information Services (IIS).
  
**Como verifico se tenho a Protecção de Autenticação Alargada activada?**    
Há várias formas de configurar a Protecção de Autenticação Alargada. Para informações sobre a configuração desta funcionalidade, que pode ser utilizada para efeitos de validação se estiver presente, consulte o [Artigo 973917 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/973917).
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso remoto autenticado que conseguisse tirar partido desta vulnerabilidade poderia executar acções no servidor IIS com os mesmos direitos que o WPI (Worker Process Identity). Por predefinição, o WPI está configurado com os privilégios da conta do Serviço de Rede. Os servidores IIS com páginas ASP cujas zonas da aplicação estão configuradas com um WPI que utilize uma conta com privilégios de administração poderiam ser afectados com mais gravidade do que os servidores IIS cuja zona da aplicação está configurada com as definições WPI predefinidas.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Existe uma vulnerabilidade de execução remota de código na forma como o servidor IIS processa tentativas de autenticação quando a Protecção de Autenticação Alargada não está activada no cliente, mas está activada no servidor. Um intruso poderia explorar esta vulnerabilidade criando um pedido HTTP especialmente concebido para o efeito.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Os servidores Web que alojam conteúdo que exige autenticação e suporte da Protecção de Autenticação Alargada são mais afectados por esta vulnerabilidade. Os sistemas só são afectados se tiverem instalada a actualização para a Protecção Alargada descrita no [Artigo 973917 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/973917).
  
**O que faz a actualização?**    
Esta actualização resolve a vulnerabilidade corrigindo a forma como o IIS valida informações específicas de autenticação transmitidas pelo cliente.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

**Instalação da actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.

**Verificar se a Actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.

#### Informações de Implementação
  
**Instalação da actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.
  
Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro         | Descrição                                                                                                                                            |  
|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**        | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte o [Artigo 934307 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/934307).
  
**Verificar se a actualização foi aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização em **Iniciar Pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
