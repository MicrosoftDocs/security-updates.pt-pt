Security Bulletin

Microsoft Security Bulletin MS10-061 - Crítica
==============================================

Vulnerabilidade no Serviço de Spooler de Impressão Poderia Permitir Execução Remota de Código (2347290)
-------------------------------------------------------------------------------------------------------

Data de publicação: 14 de setembro de 2010 | Updated: 29 de setembro de 2010

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança corrige uma vulnerabilidade no serviço de Spooler de Impressão, divulgada publicamente. A vulnerabilidade poderia permitir a execução remota de código se um intruso enviasse um pedido de impressão especialmente concebido para o efeito a um sistema vulnerável que tivesse uma interface de spooler de impressão exposta através de RPC. Por predefinição, as impressoras não são partilhadas em qualquer um dos sistemas operativos Windows actualmente suportados.

Esta actualização de segurança está classificada como Crítica para todas as edições suportadas do Windows XP e classificada como Importante para todas as edições suportadas do Windows Server 2003, Windows Vista, Windows Server 2008, Windows 7 e Windows Server 2008 R2. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade corrigindo a forma como o serviço de Spooler de Impressão valida permissões de utilizador. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem imediatamente a actualização, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões Conhecidas.** O [Artigo 2347290 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/2347290) documenta as questões actualmente conhecidas que os clientes podem verificar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estas questões.

Vulnerabilidade de Personificação no Serviço de Spooler de Impressão - CVE-2010-2729  
------------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código no serviço de Spooler de Impressão do Windows que poderia permitir que um intruso remoto não autenticado executasse código arbitrário num sistema Windows XP afectado. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas. Esta é uma vulnerabilidade de elevação de privilégios em todos os outros sistemas Microsoft Windows suportados.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-2729](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2729).
  
#### Factores Atenuantes para a Vulnerabilidade de Personificação no Serviço de Spooler de Impressão - CVE-2010-2729
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Por predefinição, as impressoras não são partilhadas em qualquer um dos sistemas operativos Windows actualmente suportados.  
-   Os sistemas apenas estão vulneráveis a ataques remotos quando partilham uma impressora e o intruso remoto acede à partilha de impressoras.  
-   Os procedimentos recomendados de firewall e as configurações de firewall padrão predefinidas podem ajudar a proteger as redes de ataques provenientes do exterior do perímetro da empresa. É recomendado que os sistemas ligados à Internet tenham um número mínimo de portas exposto.
  
#### Soluções Alternativas para a Vulnerabilidade de Personificação no Serviço de Spooler de Impressão - CVE-2010-2729
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Bloquear as portas associadas a RPC na firewall**
  
    -   Portas UDP 135, 137, 138 e 445 e portas TCP 135, 139, 445 e 593  
    -   Todo o tráfego de entrada não solicitado nas portas com um número superior a 1024  
    -   Qualquer outra porta RPC especificamente configurada
  
    Estas portas são utilizadas para iniciar uma ligação com RPC. O bloqueio destas portas na firewall ajuda a impedir que os sistemas protegidos por essa firewall sejam atacados por tentativas de explorar esta vulnerabilidade. Além disso, certifique-se de que bloqueia quaisquer outras portas RPC especificamente configuradas no sistema remoto. A Microsoft recomenda que bloqueie todas as comunicações de entrada não solicitadas a partir da Internet para ajudar a impedir ataques que possam utilizar outras portas. Para mais informações sobre portas, consulte o artigo Technet sobre [Atribuição de portas TCP e UDP](http://go.microsoft.com/fwlink/?linkid=21312).
  
    **Impacto da solução alternativa.** Vários serviços do Windows utilizam as portas afectadas. Bloquear a ligação às portas pode fazer com que diversas aplicações ou serviços não funcionem. Algumas das aplicações ou serviços que podem ser afectados encontram-se listados a seguir:
  
    -   Aplicações que utilizam o SMB (CIFS)  
    -   Aplicações que utilizam mailslots ou encaminhamentos com nome (named pipes) (RPC sobre SMB)  
    -   Servidor (Partilha de Ficheiros e Impressoras)  
    -   Política de Grupos  
    -   Net Logon  
    -   Sistema de Ficheiros Distribuídos (DFS)  
    -   Terminal Server Licensing  
    -   Spooler de Impressão  
    -   Browser de Computador  
    -   Localizador de Chamada de Procedimento Remoto  
    -   Serviço de Fax  
    -   Serviço de Indexação  
    -   Alertas e Registos de Desempenho  
    -   Systems Management Server  
    -   Serviço de Registo de Licenças
  
    **Como anular esta solução alternativa.** Desbloquear as portas UDP 135, 137, 138 e 445, e as portas TCP 135, 139, 445 e 593 na firewall. Para mais informações sobre portas, consulte [Atribuição de portas TCP e UDP](http://go.microsoft.com/fwlink/?linkid=21312).
  
-   **Desactivar a partilha de impressoras**
  
    Para desactivar a partilha de impressoras, efectue as seguintes acções:
  
    Em sistemas Windows XP e Windows 2003
  
    1.  Clique em **Iniciar** e clique em **Impressoras e faxes**.  
    2.  Clique com o botão direito do rato no ícone da impressora e seleccione **Partilhar**.  
    3.  Seleccione **Não partilhar esta impressora** e clique em **OK**.  
    4.  Repita este processo para cada impressora partilhada no sistema.
  
    Em todos os sistemas operativos Windows posteriores:
  
    1.  Clique em **Iniciar** e clique em **Dispositivos e impressoras**.  
    2.  Clique com o botão direito do rato no ícone da impressora e seleccione **Propriedades da impressora**.  
    3.  Seleccione o separador **Partilhar**.  
    4.  Certifique-se de que a opção **Partilhar esta impressora** não está seleccionada e clique em **OK**.  
    5.  Repita este processo para cada impressora partilhada no sistema.
  
    **Impacto da solução alternativa.** Ao desactivar esta funcionalidade, os utilizadores remotos não poderão imprimir para a impressora afectada.
  
    **Como anular esta solução alternativa::**
  
    Em sistemas Windows XP e Windows 2003:
  
    1.  Clique em **Iniciar** e clique em **Impressoras e faxes**.  
    2.  Clique com o botão direito do rato no ícone da impressora e seleccione **Partilhar**.  
    3.  Seleccione **Partilhar esta impressora** e clique em **OK**.  
    4.  Repita este processo para cada impressora partilhada no sistema.
  
    Em todos os sistemas operativos Windows posteriores:
  
    1.  Clique em **Iniciar** e clique em **Dispositivos e impressoras**.  
    2.  Clique com o botão direito do rato no ícone da impressora e seleccione **Propriedades da impressora**.  
    3.  Seleccione o separador **Partilhar**.  
    4.  Seleccione a opção **Partilhar esta impressora** e clique em **OK**.  
    5.  Repita este processo para cada impressora partilhada no sistema.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Personificação no Serviço de Spooler de Impressão - CVE-2010-2729
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas.
  
**O que provoca a vulnerabilidade?**    
Esta vulnerabilidade é provocada quando o Spooler de Impressão do Windows restringe de forma insuficiente o acesso aos spoolers de impressão com base nas permissões de utilizador.
  
**O que é o serviço de Spooler de Impressão?**    
O serviço de Spooler de Impressão é um ficheiro executável que é instalado como um serviço. O spooler é carregado quando o sistema operativo arranca e continua a ser executado até que o sistema operativo seja encerrado. O serviço de spooler de impressão gere o processo de impressão, que inclui tarefas como a recuperação da localização do controlador de impressora correcto, carregar esse controlador, fazer spool das chamadas de função de alto nível para um trabalho de impressão e a programação de trabalhos de impressão. Quando as tarefas de um determinado trabalho de impressão estiverem concluídas, o serviço de spooler de impressão passa o trabalho para o router de impressão. Para mais informações, consulte o artigo da TechNet sobre [Como Imprimir Trabalhos em Rede](http://technet2.microsoft.com/windowsserver/en/library/d58ce7b9-49cf-4f5e-95e9-1ade005c13e01033.mspx?mfr=true).
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse explorar esta vulnerabilidade com sucesso poderia executar código arbitrário no sistema de um utilizador com privilégios de sistema. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas, ver, alterar ou eliminar dados, ou ainda criar novas contas.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia explorar esta vulnerabilidade concebendo e enviando um pedido de impressão malicioso para um sistema vulnerável que tem uma interface de spooler de impressão exposta através de RPC. O sistema de destino não faria correctamente a validação, mesmo que o utilizador remoto tivesse permissões suficientes, e permitiria que o intruso remoto criasse um ficheiro num directório de sistema Windows. Quando deixados em localizações específicas, estes ficheiros podem ser executados automaticamente pelo sistema.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
As estações de trabalho e servidores de terminais que partilham uma impressora, ou servidores de impressão são mais susceptíveis. Os servidores podem estar mais sujeitos a risco se os administradores permitirem que os utilizadores iniciem sessão nos servidores e executem programas. Contudo, os procedimentos recomendados desencorajam essa acção.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade corrigindo a forma como o serviço de Spooler de Impressão valida permissões de utilizador para aceder a spoolers de impressão.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Sim. Esta vulnerabilidade foi divulgada publicamente. Foi-lhe atribuído o número de Vulnerabilidade e Exposição Comum [CVE-2010-2729](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2729).
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Sim. Na altura do lançamento deste boletim, esta vulnerabilidade estava a ser explorada numa escala limitada por malware.

Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.
