Security Bulletin

Microsoft Security Bulletin MS10-046 - Crítica
==============================================

Vulnerabilidade na Shell do Windows Poderia Permitir Execução Remota de Código (2286198)
----------------------------------------------------------------------------------------

Data de publicação: 2 de agosto de 2010

**actualizada:** 1.0

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade na Shell do Windows divulgada publicamente. A vulnerabilidade poderia permitir a execução remota de código se o ícone de um atalho especialmente concebido para o efeito fosse apresentado. Um intruso que explorasse com sucesso esta vulnerabilidade poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Esta actualização de segurança está classificada como Crítica para todas as edições suportadas do Microsoft Windows. Para mais informações consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade corrigindo a validação das referências dos ícones de atalhos. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

Esta actualização de segurança resolve a vulnerabilidade descrita previamente no [Aviso de Segurança da Microsoft 2286198](http://technet.microsoft.com/security/advisory/2286198).

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem imediatamente a actualização, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões Conhecidas.** Nenhum

Vulnerabilidade de Carregamento de Ícones de Atalhos - CVE-2010-2568  
--------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código em versões afectadas do Microsoft Windows. A vulnerabilidade existe porque o Windows analisa atalhos incorrectamente, de um modo em que poderia ser executado código malicioso quando o sistema operativo apresentasse o ícone de um ficheiro de atalho nocivo. Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia executar código arbitrário em nome do utilizador com sessão iniciada. Esta actualização resolve uma vulnerabilidade anteriormente discutida no [Aviso de Segurança da Microsoft 2286198](http://technet.microsoft.com/security/advisory/2286198).
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-2568](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2568).
  
#### Factores Atenuantes para a Vulnerabilidade de Carregamento de Ícones de Atalhos - CVE-2010-2568
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia obter os mesmos privilégios que o utilizador local. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.  
-   Para o vector de ataque do dispositivo USB amovível, o utilizador teria de abrir manualmente o Explorador do Windows ou uma aplicação semelhante quando a reprodução automática estivesse desactivada, e navegar até à pasta afectada do disco amovível.
  
#### Soluções Alternativas para a Vulnerabilidade de Carregamento de Ícones de Atalhos - CVE-2010-2568
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Desactivação da apresentação de ícones para atalhos**
  
    **Nota** Consulte o [Artigo 2286198 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/2286198) para utilizar a solução automática de **correcção da Microsoft** para activar ou desactivar esta solução alternativa. Para ser eficaz, esta solução de correcção requer um reinício após a conclusão. Esta solução de correcção implementa a solução alternativa, tendo assim o mesmo impacto para o utilizador. Recomendamos que os administradores consultem detalhadamente o artigo da Base de Dados de Conhecimento antes de implementar esta solução de correcção.
  
    **Nota** A utilização incorrecta do Editor de Registo pode provocar problemas graves que poderão conduzir à reinstalação do sistema operativo. A Microsoft não pode garantir que os problemas resultantes da utilização incorrecta do Editor de Registo possam ser resolvidos. A utilização do Editor de Registo é da sua responsabilidade. Para obter informações sobre como editar o registo, deve consultar o tópico de Ajuda "Alterar Chaves e Valores" do Editor de Registo (Regedit.exe) ou os tópicos de Ajuda "Adicionar e Eliminar Informações do Registo" e "Editar Dados do Registo" do Regedt32.exe.
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Localize e seleccione a chave de registo seguinte:  
        HKEY\_CLASSES\_ROOT\\lnkfile\\shellex\\IconHandler  
    3.  Clique no menu **Ficheiro** e, depois, em **Exportar**.  
    4.  Na caixa de diálogo **Exportar ficheiro de registo**, introduza **LNK\_Icon\_Backup.reg** e clique em **Guardar**.  
        **Nota** Isto criará uma cópia de segurança desta chave de registo, que, por predefinição, ficará na pasta Os meus documentos.  
    5.  Seleccione o valor **(Predefinição)** no painel do lado direito no Editor de registo. Prima **Enter** para editar o valor da chave. Elimine o valor, de modo a que o valor fique em branco, e prima **Enter**.  
    6.  Localize e seleccione a chave de registo seguinte:  
        HKEY\_CLASSES\_ROOT\\piffile\\shellex\\IconHandler  
    7.  Clique no menu **Ficheiro** e, depois, em **Exportar**.  
    8.  Na caixa de diálogo **Exportar ficheiro de registo**, introduza **PIF\_Icon\_Backup.reg** e clique em **Guardar**.  
        **Nota** Isto cria uma cópia de segurança desta chave de registo, que, por predefinição, ficará na pasta Os meus documentos.  
    9.  Seleccione o valor **(Predefinição)** no painel do lado direito no Editor de registo. Prima **Enter** para editar o valor da chave. Elimine o valor, de modo a que o valor fique em branco, e prima **Enter**.  
    10. Termine as sessões de todos os utilizadores e volte a iniciá-las, ou reinicie o computador.
  
    **Impacto da solução alternativa.** A desactivação da apresentação de ícones para atalhos evita que esta questão seja explorada nos sistemas afectados. Quando esta solução alternativa é implementada, o sistema pode apresentar a maior parte dos ícones como um ícone de objecto padrão "branco", o que tem consequências na forma de utilização. Recomendamos que os administradores do sistema testem minuciosamente esta solução alternativa antes da sua implementação. Todos os ícones voltarão a aparecer quando a solução alternativa for anulada.
  
    **Como anular esta solução alternativa.**
  
    Utilizar o método interactivo
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Clique no menu **Ficheiro** e, depois, em **Importar**.  
    3.  Na caixa de diálogo **Importar ficheiro de registo**, seleccione **LNK\_Icon\_Backup.reg** e clique em **Abrir**.  
    4.  Clique no menu **Ficheiro** e, depois, em **Importar**.  
    5.  Na caixa de diálogo **Importar ficheiro de registo**, seleccione **PIF\_Icon\_Backup.reg** e clique em **Abrir**.  
    6.  Saia do Editor de registo e reinicie o computador.
  
    Redefinição manual dos valores da chave de registo para os valores predefinidos
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Localize e clique na chave de registo seguinte:  
        HKEY\_CLASSES\_ROOT\\lnkfile\\shellex\\IconHandler  
    3.  Redefina o valor da chave de registo para:  
        **{00021401-0000-0000-C000-000000000046}**  
    4.  Localize e clique na chave de registo seguinte:  
        HKEY\_CLASSES\_ROOT\\piffile\\shellex\\IconHandler  
    5.  Redefina o valor da chave de registo para:  
        **{00021401-0000-0000-C000-000000000046}**  
    6.  Reinicie o computador
  
-   **Desactivar o serviço WebClient**
  
    A desactivação do serviço WebClient ajuda a proteger os sistemas afectados contra tentativas de explorar esta vulnerabilidade, bloqueando o vector de ataque remoto mais provável através do serviço de cliente WebDAV (Web Distributed Authoring and Versioning). Depois de aplicar esta solução alternativa, os intrusos remotos que consigam explorar esta vulnerabilidade com sucesso ainda poderão fazer com que o Microsoft Office Outlook execute programas instalados no computador do utilizador visado ou na rede local (LAN), mas os utilizadores receberão um pedido de confirmação antes de abrirem programas arbitrários da Internet.
  
    Para desactivar o serviço WebClient, siga estes passos:
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Services.msc** e a seguir clique em **OK**.  
    2.  Clique com o botão do lado direito do rato no serviço **WebClient** e seleccione **Propriedades**.  
    3.  Altere o tipo de arranque para **Desactivado**. Se o serviço estiver a ser executado, clique em **Parar**.  
    4.  Clique em **OK** e saia da aplicação de gestão.
  
    **Impacto da solução alternativa.** Quando o serviço WebClient estiver desactivado, os pedidos WebDAV (Web Distributed Authoring and Versioning) não são transmitidos. Além disso, quaisquer serviços que dependam explicitamente do serviço WebClient não serão iniciados e será enviada uma mensagem de erro para o registo do sistema. Por exemplo, as partilhas do WebDAV ficarão inacessíveis a partir do computador do cliente.
  
    **Como anular esta solução alternativa.**
  
    Para reactivar o serviço WebClient, siga estes passos:
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Services.msc** e a seguir clique em **OK**.  
    2.  Clique com o botão do lado direito do rato no serviço **WebClient** e seleccione **Propriedades**.  
    3.  Altere o tipo de arranque para **Automático**. Se o serviço não estiver a ser executado, clique em **Iniciar**.  
    4.  Clique em **OK** e saia da aplicação de gestão.
  
-   **Bloquear a transferência de ficheiros LNK e PIF da Internet**
  
    Bloquear a transferência de ficheiros LNK e PIF no Gateway para a Internet garante protecção contra a exploração remota destes ataques. Tenha em atenção que os ficheiros podem ser transferidos através de WebDAV, pelo que qualquer solução de bloqueio deve ter este protocolo em conta.
  
-   **Bloquear as ligações SMB de saída na firewall de perímetro**  
    Bloquear as ligações SMB de saída na firewall de perímetro reduz o risco de exploração remota através de partilhas de ficheiros.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Carregamento de Ícones de Atalhos - CVE-2010-2568
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios.
  
**O que provoca a vulnerabilidade?**    
Ao tentar carregar o ícone de um atalho, a Shell do Windows não valida correctamente parâmetros específicos do atalho.
  
**O que é a Shell do Windows?**    
A interface de utilizador do Windows (UI) fornece aos utilizadores acesso a uma grande variedade de objectos necessários para executar aplicações e gerir o sistema operativo. Os mais numerosos e familiares destes objectos são as pastas e ficheiros que residem nas unidades de disco do computador. Existem igualmente vários objectos virtuais que permitem que o utilizador efectue tarefas, tais como enviar ficheiros para impressoras remotas ou aceder à Reciclagem. A Shell organiza estes objectos num espaço de nome hierárquico e fornece aos utilizadores e aplicações uma forma consistente e eficiente de gerir e aceder a objectos.
  
**O que é um atalho?**    
Um atalho é uma ligação para um ficheiro ou programa, sendo representado por um ícone. Se clicar duas vezes num atalho, o ficheiro ou programa abre-se. O atalho é um mecanismo muitas vezes utilizado para manter os ficheiros de utilização frequente numa localização única e de fácil acesso, como uma pasta ou o ambiente de trabalho. Os atalhos são implementados como ficheiros com a extensão LNK. Quando estão relacionados com programas MS-DOS, os atalhos podem também aparecer como ficheiros PIF.
  
**Esta actualização de segurança desactiva a solução alternativa ou a solução de Correcção da Microsoft se a tiver implementado anteriormente?**    
Não, a solução alternativa e a solução de Correcção da Microsoft funcionam separadamente da actualização de segurança. Depois de a actualização de segurança ser implementada, os utilizadores que tiverem aplicado a solução alternativa terão de a anular.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia executar código arbitrário em nome do utilizador com sessão iniciada. Se um utilizador tiver sessão iniciada com direitos de utilizador administrativos, um intruso poderia obter controlo total sobre o sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia apresentar ao utilizador uma unidade amovível com um ficheiro de atalho nocivo e um binário malicioso associado. Quando o utilizador abrisse esta unidade no Explorador do Windows, ou em qualquer outra aplicação que analise o ícone do atalho, o binário malicioso executaria código seleccionado pelo intruso no sistema do alvo.
  
Um intruso poderia também configurar um Web site malicioso ou uma partilha de rede remota e colocar os componentes maliciosos nesta localização remota. Quando o utilizador navegasse no Web site com um browser da Web como o Internet Explorer ou um gestor de ficheiros como o Explorador do Windows, o Windows tentaria carregar o ícone do ficheiro de atalho, invocando o binário malicioso. Além disso, um intruso poderia incorporar uma exploração num documento que suportasse atalhos incorporados ou num controlo de browser alojado (como, mas não apenas, documentos do Microsoft Office).
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
Os sistemas de ambiente de trabalho que são utilizados para navegar em partilhas de rede não fidedignas ou em Web sites não fidedignos estão mais susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização resolve esta vulnerabilidade validando correctamente a referência de um ícone de um atalho.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Sim. Esta vulnerabilidade foi divulgada publicamente. Foi-lhe atribuído o número de Vulnerabilidade e Exposição Comum [CVE-2010-2568](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-2568).
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Na altura do lançamento deste boletim de segurança, esta vulnerabilidade estava a ser explorada por diversas famílias de malware.

#### Informações de Implementação

**Instalação da Actualização**

Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de um hotfix da Microsoft.

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, clique com o botão direito do rato no nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Propriedades**.
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no sistema, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.

Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Para mais informações sobre a terminologia que é usada neste boletim, tal como *hotfix*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro                            | Descrição                                                                                                                                            |  
|--------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help**                    | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**                           | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**                       | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |  
| **/warnrestart:&lt;seconds&gt;**     | Quando combinado com **/quiet**, o programa de instalação avisa o utilizador antes de dar lugar ao reinício.                                         |  
| **/promptrestart**                   | Quando combinado com **/quiet**, o programa de instalação solicitará confirmação antes de dar lugar ao reinício.                                     |  
| **/forcerestart**                    | Quando combinado com **/quiet**, o programa de instalação força o encerramento das aplicações e dá lugar ao reinício.                                |  
| **/log:&lt;file name&gt;**           | Activa o registo no ficheiro especificado.                                                                                                           |  
| **/extract:&lt;destination&gt;**     | Extrai o conteúdo do pacote para a pasta de destino.                                                                                                 |  
| **/uninstall /kb:&lt;KB Number&gt;** | Desinstala a actualização de segurança.                                                                                                              |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte a informação referente ao Programa de Instalação Independente do Windows Update no artigo TechNet sobre [Alterações Diversas no Windows 7](http://technet.microsoft.com/en-us/library/dd871148(ws.10).aspx).
  
**Verificar Se a Actualização Foi Aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e ferramentas de detecção e implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização em **Iniciar Pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
