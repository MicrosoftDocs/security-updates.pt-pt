Security Bulletin

Microsoft Security Bulletin MS10-075 - Crítica
==============================================

Vulnerabilidade no Serviço de Partilha de Rede do Media Player Poderia Permitir Execução Remota de Código (2281679)
-------------------------------------------------------------------------------------------------------------------

Data de publicação: 12 de outubro de 2010

**actualizada:** 1.0

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade comunicada de forma privada no Serviço de Partilha de Rede do Microsoft Windows Media Player. A vulnerabilidade poderia permitir a execução remota de código se um intruso enviasse um pacote RTSP especialmente concebido para o efeito para um sistema afectado. No entanto, o acesso ao multimédia doméstico através da Internet está desactivado por predefinição. Nesta configuração predefinida, a vulnerabilidade apenas pode ser explorada por um intruso dentro da mesma sub-rede.

Esta actualização de segurança está classificada como Crítica para as edições suportadas do Windows 7 e como Importante para todas as edições suportadas do Windows Vista. Para mais informações consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade corrigindo a forma como o Serviço de Partilha de Rede do Windows Media Player processa pacotes RTSP especialmente concebidos para o efeito. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem imediatamente a actualização, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://update.microsoft.com/microsoftupdate/v6/default.aspx?ln=pt-pt).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões Conhecidas.** Nenhum

Vulnerabilidade de Utilização Após Libertação RTSP - CVE-2010-3225  
------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade no Serviço de Partilha de Rede do Microsoft Windows Media Player que poderia permitir a um utilizador remoto enviar um pacote de rede especialmente concebido para o efeito para uma instância do serviço de transmissão em sequência na rede da aplicação e provocar a execução remota de código no contexto da aplicação actual.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-3225](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3225).
  
#### Factores Atenuantes para a Vulnerabilidade de Utilização Após Libertação RTSP - CVE-2010-3225
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Por predefinição, o acesso ao multimédia doméstico através da Internet está desactivado. Nesta configuração predefinida, a vulnerabilidade apenas pode ser explorada por computadores dentro da mesma sub-rede.  
-   Nas edições suportadas do Windows Vista e nas edições do Windows 7 Professional, Windows 7 Enterprise e Windows 7 Ultimate, o Serviço de Partilha de Rede do Windows Media Player não está activado por predefinição.
  
#### Soluções Alternativas para a Vulnerabilidade de Utilização Após Libertação RTSP - CVE-2010-3225
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Desactivar o Serviço de Partilha de Rede do Windows Media Player em sistemas afectados**
  
    **Método Interactivo**
  
    **Nota** A utilização incorrecta do Editor de Registo pode provocar problemas graves que poderão conduzir à reinstalação do sistema operativo. A Microsoft não pode garantir que os problemas resultantes da utilização incorrecta do Editor de Registo possam ser resolvidos. A utilização do Editor de Registo é da sua responsabilidade. Para obter informações sobre como editar o registo, deve consultar o tópico de Ajuda "Alterar Chaves e Valores" do Editor de Registo (Regedit.exe) ou os tópicos de Ajuda "Adicionar e Eliminar Informações do Registo" e "Editar Dados do Registo" do Regedt32.exe.
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Localize e seleccione a chave de registo seguinte:  
        HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\services\\WMPNetworkSvc  
    3.  Clique no menu **Ficheiro** e, depois, em **Exportar**.  
    4.  Na caixa de diálogo **Exportar ficheiro de registo**, introduza **WMPNetworkSvc\_Backup.reg** e clique em **Guardar**.  
        **Nota** Isto criará uma cópia de segurança desta chave de registo, que, por predefinição, ficará na pasta Os meus documentos.  
    5.  Seleccione o valor **Start** no painel do lado direito no Editor de registo. Prima **Enter** para editar o valor da chave. Altere o campo **Dados** para **4** e prima **Enter**.  
    6.  . Pare o serviço, introduzindo o seguinte comando a partir de uma linha de comandos elevada:  
        `sc stop WMPNetworkSvc`
  
    **Gestão de Script de Implementação**
  
    1.  Faça uma cópia de segurança da chave de registo utilizando o seguinte comando:  
        `regedit /e WMPNetworkSvc_Backup.reg HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\WMPNetworkSvc`  
    2.  Guarde o que se segue num ficheiro com a extensão .REG (por exemplo: Disable\_WMPNetworkSvc.reg):  
        Editor de Registo do Windows Versão 5.00  
        HKEY\_LOCAL\_MACHINE\\System\\CurrentControlSet\\Services\\service\_name\]  
        Start"=dword:00000004  
    3.  Execute o script de registo criado no passo 2 no computador visado, com o seguinte comando:  
        `Regedit /s Disable_WMPNetworkSvc.reg`  
    4.  Pare o serviço, introduzindo o seguinte comando a partir de uma linha de comandos elevada:  
        `sc stop WMPNetworkSvc`
  
    **Impacto da solução alternativa.** A Partilha de Multimédia do Windows é desactivada.
  
    **Como anular esta solução alternativa.**
  
    Utilizar o método interactivo:
  
    1.  Clique em **Iniciar**, clique em **Executar**, escreva **Regedit** na caixa **Abrir** e clique em **OK**.  
    2.  Clique no menu **Ficheiro** e, depois, em **Importar**.  
    3.  Na caixa de diálogo **Importar ficheiro de registo**, seleccione o ficheiro **WMPNetworkSvc\_Backup.reg** que foi criado acima no "Método Interactivo" acima e, em seguida, clique em **Abrir**.  
    4.  Saia do Editor de registo e reinicie o computador.
  
    Utilizar a gestão de scripts de implementação:
  
    1.  Execute o script de registo criado no passo 2 no computador visado, com o seguinte comando:  
        `Regedit /s WMPNetworkSvc_Backup.reg`  
    2.  Reinicie o computador.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Utilização Após Libertação RTSP - CVE-2010-3225
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso que explorasse esta vulnerabilidade com sucesso poderia executar código no contexto de segurança da conta do Serviço de Rede.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é provocada pela forma como o Serviço de Partilha de Rede do Microsoft Windows Media Player processa pacotes RTSP (Real Time Streaming Protocol).
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso poderia explorar a vulnerabilidade criando e enviando um pacote RTSP especialmente concebido para o efeito para um sistema afectado.
  
**O que é o RTSP?**    
O protocolo RTSP (Real Time Streaming Protocol) é um protocolo ao nível da aplicação para controlar a entrega de dados com propriedades de tempo real. O RTSP fornece uma estrutura extensível para permitir a entrega controlada e sob pedido de dados em tempo real, tais como áudio e vídeo. As fontes de dados tanto podem incluir feeds de dados em tempo real como clips armazenados. Este protocolo destina-se a controlar várias sessões de entrega de dados, a disponibilizar um meio para seleccionar canais de entrega, tais como UDP, UDP multicast e TCP, e a disponibilizar um meio para seleccionar mecanismos de entrega baseados em RTP (RFC 1889). O RTSP é definido em [RFC 2326](http://www.ietf.org/rfc/rfc2326.txt).
  
O RTSP suporta acções de controlo de reprodução, tais como parar, fazer pausa, recuar e avançar, em ficheiros Multimédia do Windows indexados. Pode utilizar o RTSP para transmitir conteúdo para computadores que executem o Windows Media Player Série 9 ou posterior ou o Windows Media Services Série 9 ou posterior.
  
**O que é a conta do Serviço de Rede?**    
A conta do Serviço de Rede é uma conta incorporada que tem mais acesso a recursos e objectos do que os membros do grupo de Utilizadores. Os serviços que são executados como conta do Serviço de Rede acedem aos recursos da rede utilizando as credenciais da conta do computador. O nome real da conta é NT AUTHORITY\\NETWORK SERVICE. Para mais informações, consulte o artigo MSDN sobre a Conta NetworkService.
  
**O que é o Serviço de Partilha de Rede do Windows Media Player?**    
O Windows Media Player permite-lhe partilhar ficheiros da sua biblioteca com outros utilizadores na sua rede doméstica e procurar ficheiros das bibliotecas que são partilhadas com outros utilizadores da sua rede. A Partilha de Multimédia do Windows utiliza o Serviço de Partilha da Rede do Windows Media Player.
  
**Como posso determinar se o Serviço de Partilha da Rede do Windows Media Player está a ser executado no meu sistema?**    
Clique em **Iniciar**, clique em **Executar**, escreva **Services.msc** e a seguir clique em **OK**. No painel de controlo de Serviços, confirme se o estado do Serviço de Partilha de Rede do Windows Media Player está "Iniciado".
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo de um sistema afectado no contexto de segurança da conta do Serviço de Rede.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
Os sistemas que executam o Serviço de Partilha de Rede do Windows Media Player (wmpnetwk.exe) para transmitir conteúdos multimédia são mais susceptíveis.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade corrigindo a forma como o Serviço de Partilha de Rede do Windows Media Player processa pacotes RTSP especialmente concebidos para o efeito.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação coordenada de vulnerabilidades.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.

**Update Compatibility Evaluator e Application Compatibility Toolkit**
  
É frequente as actualizações utilizarem os mesmos ficheiros e configurações de registo necessários para a execução das aplicações. Isso pode dar origem a incompatibilidades e aumentar o tempo necessário para a implementação de actualizações de segurança. Pode simplificar o teste e a validação de actualizações do Windows relativamente às aplicações instaladas com os componentes do [Update Compatibility Evaluator](http://technet2.microsoft.com/windowsvista/en/library/4279e239-37a4-44aa-aec5-4e70fe39f9de1033.mspx?mfr=true) incluído no [Application Compatibility Toolkit](http://www.microsoft.com/downloads/details.aspx?familyid=24da89e9-b581-47b0-b45e-492dd6da2971&displaylang=en).
  
O Application Compatibility Toolkit (ACT) contém as ferramentas e documentação necessárias para avaliar e atenuar questões de compatibilidade da aplicação antes de implementar o Microsoft Windows Vista, uma Actualização do Windows, uma Actualização de Segurança da Microsoft ou uma nova versão do Windows Internet Explorer no seu ambiente.
