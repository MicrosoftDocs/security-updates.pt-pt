Security Bulletin

Microsoft Security Bulletin MS06-041 - Crítica
==============================================

Vulnerabilidades na Resolução DNS poderiam permitir Execução Remota de Código (920683)
--------------------------------------------------------------------------------------

Data de publicação: 8 de agosto de 2006 | Updated: 13 de setembro de 2006

**actualizada:** 1.0

#### Resumo

**Quem deve ler este documento:** Clientes que utilizam o Microsoft Windows

**Impacto da Vulnerabilidade:** Execução Remota de Código

**Classificação de Gravidade Máxima:** Crítica

**Recomendação:** Os clientes deverão aplicar de imediato a actualização

**Substituição de Actualizações de Segurança:** Nenhum

**Avisos:** Nenhum

Detalhes Sobre a Vulnerabilidade  
--------------------------------
  
<span></span>
#### Vulnerabilidade no Nome de Alojamento Winsock - CVE-2006-3440:
  
Existe uma vulnerabilidade de execução remota de código no Winsock que poderia permitir que um intruso que explorasse esta vulnerabilidade com sucesso obtivesse controlo total sobre o sistema afectado. Para que um ataque tivesse sucesso, o intruso teria de forçar o utilizador a abrir um ficheiro ou a visitar um Web site especialmente concebidos para solicitar a API do Winsock afectada.
  
#### Factores atenuantes para a Vulnerabilidade no Nome de Alojamento Winsock - CVE-2006-3440:
  
-   Um intruso conseguiria tirar partido da vulnerabilidade se convencesse um utilizador a abrir um ficheiro ou a visualizar um Web site especialmente concebidos. Não existe forma de um intruso forçar um utilizador a abrir um ficheiro especialmente concebido, excepto potencialmente através da pré-visualização de uma mensagem de correio electrónico.
  
#### Soluções alternativas para a Vulnerabilidade no Nome de Alojamento Winsock - CVE-2006-3440:
  
A Microsoft testou as seguintes soluções alternativas. Embora estas soluções alternativas não corrijam a vulnerabilidade subjacente, elas ajudam a bloquear vectores de ataque conhecidos. Na secção seguinte identificam-se os casos em que uma solução alternativa reduz a funcionalidade.
  
-   **Modificar o Autodial DLL**
  
    Modificar o Autodial DLL no registo do Windows irá impedir que uma aplicação, um Web site especialmente concebido para o efeito, ou uma mensagem de correio electrónico contactem a API afectada e explorem esta vulnerabilidade. Se o valor de registo Autodial DLL não for localizado por pré-definição na localização especificada, recomendamos aos clientes que criem o valor REG\_SZ como adequado.  
    **Nota** A utilização incorrecta do Editor de registo pode provocar problemas graves que podem exigir a reinstalação do sistema operativo. A Microsoft não pode garantir que os problemas resultantes da utilização incorrecta do Editor de Registo possam ser resolvidos. A utilização do Editor de Registo é da sua responsabilidade. Para obter informações sobre como editar o registo, deve consultar o tópico de Ajuda "Alterar Chaves e Valores" do Editor de Registo (Regedit.exe) ou os tópicos de Ajuda "Adicionar e Eliminar Informações do Registo" e "Editar Dados do Registo" do Regedt32.exe.  
    **Nota** Recomendamos a realização de uma cópia de segurança do registo antes de o editar.
  
    -   Clique em **Iniciar**, seleccione a opção **Executar**, introduza "**regedit32**" (sem as aspas) e clique em **OK**.  
    -   No Editor de Registo, localize a seguinte chave de registo: HKEY\_LOCAL\_MACHINE\\SYSTEM\\CurrentControlSet\\Services\\WinSock2\\Parameters  
    -   Clique duas vezes sobre o valor REG\_SZ **AutodialDLL**  
    -   Defina o valor de dados para **kernel32.dll**  
    -   Feche o utilitário **regedt32** e reinicie o computador
  
#### Perguntas mais frequentes sobre a Vulnerabilidade no Nome de Alojamento Winsock - CVE-2006-3440:
  
**Qual é a abrangência desta vulnerabilidade?**  
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido da vulnerabilidade poderia obter o controlo total de um sistema afectado remotamente. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios.
  
**O que provoca a vulnerabilidade?**  
Uma memória intermédia não verificada na API do Winsock.
  
**O que é o Winsock?**  
O Windows Sockets 2 (Winsock) permite que os programadores criem aplicações para Internet, Intranet e outras aplicações de rede avançadas para transmitir dados através da rede, independentemente do protocolo usado. Com o Winsock, os programadores podem aceder a capacidades de rede Microsoft® Windows® avançadas como multicast e QOS (Quality of Service). Para mais informações sobre o Winsock, consulte o seguinte [Artigo MSDN](http://windowssdk.msdn.microsoft.com/en-us/library/ms740673.aspx).
  
**Como poderia um intruso utilizar a vulnerabilidade?**  
Um intruso que conseguisse tirar partido desta vulnerabilidade com êxito poderia controlar totalmente o sistema afectado.
  
**Como poderia um intruso explorar a vulnerabilidade?**  
Um intruso conseguiria tirar partido da vulnerabilidade se convencesse um utilizador a abrir um ficheiro ou a visualizar um Web site especialmente concebidos para o efeito. Não existe forma de um intruso forçar um utilizador a abrir um ficheiro especialmente concebido, excepto potencialmente através da pré-visualização de uma mensagem de correio electrónico. Além disso, se uma aplicação usa o API afectado, poderia haver exploração da vulnerabilidade num cenário normal de utilização que não exigisse acção do utilizador.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**  
Os servidores e as estações de trabalho estão mais susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**  
A actualização elimina a vulnerabilidade, ao modificar a forma como a função afectada valida a mensagem, antes de a transmitir para a memória intermédia atribuída.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**  
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**  
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.
  
#### Vulnerabilidade de Sobrecarga da Memória Intermédia do Cliente DNS - CVE-2006-3441:
  
Existe uma vulnerabilidade de execução remota de código no serviço de Cliente DNS que poderia permitir que um intruso que explorasse esta vulnerabilidade com sucesso obtivesse controlo total sobre o sistema afectado.
  
#### Factores atenuantes para a Vulnerabilidade de Sobrecarga da Memória Intermédia do Cliente DNS - CVE-2006-3441:
  
-   Para que um ataque tenha sucesso, o intruso teria de estar numa sub-rede entre o alojador e o servidor DNS ou forçar o alojador atacado a fazer um pedido DNS para receber uma resposta de registo especialmente concebida de um servidor nocivo.
  
#### Soluções alternativas para a Vulnerabilidade de Sobrecarga da Memória Intermédia do Cliente DNS - CVE-2006-3441:
  
A Microsoft testou as seguintes soluções alternativas. Embora estas soluções alternativas não corrijam a vulnerabilidade subjacente, elas ajudam a bloquear vectores de ataque conhecidos. Na secção seguinte identificam-se os casos em que uma solução alternativa reduz a funcionalidade.
  
-   Para que um ataque tenha sucesso, os intrusos teriam de estar numa sub-rede entre o alojador e o servidor DNS ou forçar o alojador atacado a fazer um pedido DNS para receber uma resposta de registo especialmente concebida de um servidor nocivo.  
-   **Bloquear registos relacionados com DNS nos gateways de rede**
  
    Bloquear os seguintes tipos de registos DNS em gateways de rede ajudará a proteger o sistema afectado contra tentativas de explorar esta vulnerabilidade.
  
    -   ATMA  
    -   TXT  
    -   X25  
    -   HINFO  
    -   ISDN DNS
  
#### Perguntas mais frequentes sobre a Vulnerabilidade de Sobrecarga da Memória Intermédia do Cliente DNS - CVE-2006-3441:
  
**Qual é a abrangência desta vulnerabilidade?**  
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido da vulnerabilidade poderia obter o controlo total de um sistema afectado remotamente. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios.
  
**O que provoca a vulnerabilidade?**  
Uma memória intermédia não verificada no patamar de cliente DNS.
  
**O que é o DNS?**  
O serviço de cliente Sistema de Nomes de Domínio (DNS) resolve e coloca em cache os nomes DNS. O serviço de cliente DNS tem de estar a ser executado em todos os computadores que vão processar a resolução de nome DNS. A capacidade de processar nomes DNS é crucial para a localização de controladores de domínios em domínios Active Directory. O serviço de cliente DNS também é crítico para a localização de dispositivos identificados através de resoluções de nome DNS. Para mais informações sobre o serviço cliente DNS, consulte o seguinte [Artigo Microsoft TechNet](http://technet2.microsoft.com/windowsserver/en/library/6715556b-dfe4-41c4-87a9-333d22e473b21033.mspx?mfr=true).
  
**Como poderia um intruso utilizar a vulnerabilidade?**  
Um intruso que conseguisse tirar partido desta vulnerabilidade com êxito poderia controlar totalmente o sistema afectado.
  
**Como poderia um intruso explorar a vulnerabilidade?**  
Um utilizador anónimo poderia explorar a vulnerabilidade enviando uma comunicação DNS especialmente concebida a um cliente afectado. Para que um ataque tenha sucesso, o intruso teria de estar numa sub-rede entre o alojador e o servidor DNS ou forçar o alojador atacado a fazer um pedido DNS para receber uma resposta de registo especialmente concebida de um servidor nocivo.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**  
Os servidores e as estações de trabalho estão mais susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**  
A actualização remove a vulnerabilidade validando a forma como o cliente DNS processa comunicações relacionadas com DNS.
  
**A vulnerabilidade poderia ser explorada a partir da Internet?**  
Um intruso poderia tentar explorar esta vulnerabilidade através da Internet. Os procedimentos recomendados em relação à utilização de firewalls e as predefinições padrão de uma firewall podem ajudar a proteger os sistemas contra ataques provenientes da Internet. A Microsoft disponibilizou informações sobre como pode ajudar a proteger o seu computador. Os utilizadores finais podem visitar o [Web site Proteja o Seu PC](http://www.microsoft.com/portugal/athome/security/default.mspx). Os profissionais de informática podem visitar o [Web site do Security Guidance Center](http://go.microsoft.com/fwlink/?linkid=21171).
  
**O que faz a actualização?**  
A actualização remove a vulnerabilidade validando a forma como o cliente DNS processa comunicações relacionadas com DNS.
  
**A desactivação do serviço de cliente DNS ou a configuração do cliente para usar um servidor DNS específico atenuaria a vulnerabilidade?**  
Não. A vulnerabilidade não pode ser atenuada com a desactivação do serviço de cliente DNS ou com a configuração para utilização de um servidor DNS fidedigno específico.

**Nota** A utilização do parâmetro **/quiet** irá suprimir todas as mensagens. Isto inclui a supressão de mensagens de falha. Os administradores também devem consultar o ficheiro KB920683.log para verificar a existência de quaisquer mensagens de falha quando utilizam este parâmetro.

Para instalar a actualização de segurança sem obrigar o sistema a reiniciar, utilize o seguinte comando numa linha de comandos do Windows Server 2003:

**Windowsserver2003-kb920683-x86-enu /norestart**

**Requisito de Reinício**

É necessário reiniciar o sistema depois de aplicar esta actualização de segurança.

Esta actualização de segurança suporta HotPatching. Se usar HotPatching, pode instalar actualizações de segurança em sistemas que estejam a executar o Microsoft Windows Server 2003 Service Pack 1, sem reiniciar os servidores. Para mais informações sobre HotPatching e sobre como implementar a actualização de segurança usando HotPatching, consulte o Artigo 897341 da Base de Dados de Conhecimento da Microsoft.

A actualização de segurança que está disponível no Windows Update, Microsoft Update e Actualizações Automáticas fazem uso do HotPatching e não requer o reinício do seu servidor.

**Nota** Nem todas as actualizações de segurança suportam HotPatching e algumas que o suportam poderão obrigar ao reinício do servidor depois da instalação da actualização de segurança. O HotPatching só é suportado se os ficheiros que serão substituídos pela actualização de segurança forem do tipo General Distribution Release (GDR).

    **Nota** Dado existirem várias versões do Microsoft Windows, os passos que se seguem podem ser diferentes no seu computador. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no computador, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

    **Nota** Dado existirem várias versões do Microsoft Windows, os passos que se seguem podem ser diferentes no seu computador. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, faça clique com o botão direito na tabela de informações do ficheiro adequada e clique em **Propriedades**.
        **Nota** Consoante a versão do sistema operativo ou programas instalados, alguns dos ficheiros que são apresentados na tabela de informações do ficheiro poderão não ser instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no computador, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

**Verificar se a Actualização foi aplicada**

-   **Microsoft Baseline Security Analyzer**

    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). A ferramenta MBSA permite aos administradores procurarem actualizações de segurança em falta e comuns erros de configuração de segurança em sistemas locais e remotos. Para mais informações acerca do MBSA, visite o [Web site do Microsoft Baseline Security Analyzer](http://go.microsoft.com/fwlink/?linkid=21134).

-   **Verificação das Versões dos Ficheiros**

    **Nota** Dado existirem várias versões do Microsoft Windows, os passos que se seguem podem ser diferentes no seu computador. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
    2.  No painel **Resultados da Procura**, clique em **Todos os ficheiros e pastas** no **Assistente de Pesquisa**.
    3.  Na caixa **Totalidade ou parte do nome de ficheiro**, escreva um nome de ficheiro a partir da tabela de informações de ficheiros adequada e, em seguida, clique em **Procurar**.
    4.  Na lista de ficheiros, faça clique com o botão direito na tabela de informações do ficheiro adequada e clique em **Propriedades**.
        **Nota** Consoante a versão do sistema operativo ou programas instalados, alguns dos ficheiros que são apresentados na tabela de informações do ficheiro poderão não ser instalados.
    5.  No separador **Versão**, determine a versão do ficheiro que está instalada no computador, comparando-a com a versão que é documentada na tabela de informações de ficheiros adequada.
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.

-   **Verificação da Chave de Registo**

    Poderá também verificar os ficheiros instalados por esta actualização de segurança revendo a seguinte chave de registo:

    HKEY\_LOCAL\_MACHINE\\SOFTWARE\\Microsoft\\Updates\\Windows 2000\\SP5\\KB920683\\Filelist

    **Nota** Esta chave de registo poderá não conter uma lista completa dos ficheiros instalados. Além disso, esta chave de registo poderá não ser criada correctamente quando um administrador ou um OEM integrar ou associar a actualização de segurança aos ficheiros de origem de instalação do Windows.

**Systems Management Server:**

O Microsoft Systems Management Server (SMS) fornece uma solução empresarial altamente configurável para gerir actualizações. O SMS permite aos administradores identificarem sistemas baseados no Windows que necessitem de actualizações de segurança e executar a implementação controlada dessas actualizações em toda a empresa, com um mínimo de incómodo para os utilizadores finais.
