Security Bulletin

Microsoft Security Bulletin MS06-034 - Importante
=================================================

Vulnerabilidade no Microsoft Internet Information Services a utilizar Páginas de Servidor Activas (ASP) poderia permitir Execução Remota de Código (917537)
-----------------------------------------------------------------------------------------------------------------------------------------------------------

Data de publicação: 11 de julho de 2006 | Updated: 19 de julho de 2006

**actualizada:** 1.2

#### Resumo

**Quem deve ler este documento:** Clientes que utilizam o Microsoft Internet Information Services (IIS)

**Impacto da Vulnerabilidade:** Execução Remota de Código

**Classificação de Gravidade Máxima:** Importante

**Recomendação:** Os clientes devem aplicar a actualização na primeira oportunidade.

**Substituição de Actualizações de Segurança:** Nenhuma

**Avisos:** [O artigo 917537 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/917537) documenta as questões actualmente conhecidas que os clientes podem experimentar quando instalam esta actualização de segurança. O artigo também documenta soluções recomendadas para estes problemas. Para mais informações, consulte o [artigo 917537 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/917537).

Resumo Executivo
----------------

<span></span>
**Resumo Executivo:**

Esta actualização resolve uma vulnerabilidade recentemente descoberta e comunicada de forma privada. A vulnerabilidade está documentada na secção "Detalhes sobre a Vulnerabilidade" deste boletim.

Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios.

Um intruso tem de ter credenciais de início de sessão válidas, mas se um servidor tiver sido configurado propositadamente para permitir que utilizadores, tanto anónimos como autenticados, carreguem conteúdos web como páginas .ASP para web sites, o servidor poderia ser explorado por esta vulnerabilidade.

Recomendamos que os clientes apliquem a actualização na primeira oportunidade.

Detalhes Sobre a Vulnerabilidade
--------------------------------

<span></span>
#### Vulnerabilidade no Microsoft Internet Information Services a utilizar Páginas de Servidor Activas (ASP) - CVE-2006-0026

Existe uma vulnerabilidade de execução remota de código no Internet Information Services (IIS). Um intruso poderia explorar a vulnerabilidade construindo um ficheiro com Páginas de Servidor Activas (ASP) especialmente concebido para o efeito, o que poderia potencialmente permitir a execução remota de código se o Internet Information Services (IIS) o processasse. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado.

#### Factores atenuantes para a Vulnerabilidade no Internet Information Services a utilizar Páginas de Servidor Activas (ASP) mal formadas - CVE-2006-0026:

-   No IIS 5.0 e no IIS 5.1, as aplicações com Páginas de Servidor Activas activadas são executadas por predefinição na aplicação 'Pooled Out of Process', o que significa que são executadas em DLLHOST.exe, que está a ser executado no contexto da conta com poucos privilégios IWAM\_&lt;machinename&gt;.
-   Como pré-definição, o IIS 5.1 no Windows XP Professional e o IIS 6.0 no Windows Server 2003 não estão activados.
-   Por predefinição, as Páginas de Servidor Activas não estão activadas no IIS 6.0. Se as Páginas de Servidor Activas estiverem activadas, são executadas no contexto de um processo de trabalhador W3WP.exe a ser executado como conta com poucos privilégios 'NetworkService'.
-   Seria necessário que um intruso tivesse credenciais de início de sessão válidas para explorar esta vulnerabilidade. Contudo, se um servidor tiver sido intencionalmente configurado para permitir que os utilizadores, sejam eles anónimos ou autenticados, transfiram conteúdo web, como páginas .ASP, para web sites, o servidor poderia ser atacado com sucesso através da exploração desta vulnerabilidade.

#### Soluções alternativas para a Vulnerabilidade no Microsoft Internet Information Services a utilizar Páginas de Servidor Activas (ASP) - CVE-2006-0026:

Não identificámos quaisquer alternativas para esta vulnerabilidade.

#### Perguntas mais frequentes sobre a Vulnerabilidade no Microsoft Internet Information Services a utilizar Páginas de Servidor Activas (ASP) - CVE-2006-0026:

**Qual é a abrangência desta vulnerabilidade?**
Há uma vulnerabilidade de execução remota de código no Internet Information Services (IIS) que resulta da forma como o IIS processa Páginas de Servidor Activas (ASP). Um intruso poderia explorar a vulnerabilidade construindo um ficheiro ASP especialmente concebido para o efeito, o que poderia potencialmente permitir a execução remota de código se o Internet Information Services (IIS) processasse um destes ficheiros. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado.

**O que provoca a vulnerabilidade?**
Uma memória intermédia não verificada no IIS.

**O que são as Páginas de Servidor Activas (ASP)?**
As Páginas de Servidor Activas da Microsoft (ASP) são uma tecnologia de execução de scripts baseados no servidor, que pode ser usada para criar aplicações Web dinâmicas e interactivas. Uma página ASP é uma página HTML que contém scripts baseados no servidor que são processados pelo servidor Web antes de serem enviados para o browser do utilizador.

**Como poderia um intruso utilizar a vulnerabilidade?**
Um intruso que conseguisse tirar partido desta vulnerabilidade com êxito poderia controlar totalmente o sistema afectado.

**Quem poderia explorar a vulnerabilidade?**
Seria necessário que um intruso tivesse credenciais de início de sessão válidas no servidor para poder explorar esta vulnerabilidade. Contudo, se um servidor tivesse sido intencionalmente configurado para permitir que os utilizadores, sejam eles anónimos ou autenticados, transferissem conteúdo web, como páginas .ASP, para web sites, o servidor poderia ser atacado com sucesso através da exploração desta vulnerabilidade.

**Como poderia um intruso explorar a vulnerabilidade?**
Um intruso poderia tentar explorar a vulnerabilidade criando uma mensagem ASP especialmente concebida para o efeito e carregando o ficheiro para um sistema afectado. Se o IIS processasse o ficheiro, poderia fazer com que o sistema afectado executasse o código.

**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**
Os sistemas Windows 2000 Service Pack 4 a executar o IIS 5.0 estão mais susceptíveis a esta vulnerabilidade porque o IIS está activado por predefinição. O Windows XP Professional e o Windows Server 2003 também estão em situação de risco se o serviço estiver activo.

**A vulnerabilidade poderia ser explorada a partir da Internet?**
Sim. Um intruso poderia tentar explorar esta vulnerabilidade pela Internet se lhe fossem permitidos acessos de escrita no servidor. Os procedimentos recomendados em relação à utilização de firewalls e as predefinições padrão de uma firewall podem ajudar a proteger os sistemas contra ataques provenientes da Internet. A Microsoft disponibilizou informações sobre como pode ajudar a proteger o seu computador. Os utilizadores finais podem visitar o [Web site Proteja o Seu PC](http://go.microsoft.com/fwlink/?linkid=21169). Os profissionais de informática podem visitar o [Web site do Security Guidance Center](http://go.microsoft.com/fwlink/?linkid=21171).

**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação responsável. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido divulgada publicamente, quando este boletim de segurança foi publicado pela primeira vez.

**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**
Não. A Microsoft não tinha recebido quaisquer informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes e não tinha conhecimento de exemplos publicados de código para prova de conceito quando este boletim de segurança foi criado pela primeira vez.

**Requisito de Reinício**

Esta actualização não requer um reinício. O programa de instalação pára os serviços necessários, aplica a actualização e, em seguida, reinicia os serviços. Contudo, se não for possível interromper os serviços necessários por qualquer motivo, ou se os ficheiros necessários estiverem a ser utilizados, esta actualização vai requerer o reinício do computador. Se for esse o caso, será apresentada uma mensagem aconselhando-o a reiniciar.

Para ajudar a reduzir a hipótese de ser necessário reiniciar o computador, interrompa todos os serviços afectados e feche todas as aplicações que possam usar os ficheiros afectados antes de instalar a actualização de segurança. Para mais informações sobre por que razão pode ser solicitada a sua confirmação para o reinício do seu computador, consulte o [Artigo 887012 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/887012).
