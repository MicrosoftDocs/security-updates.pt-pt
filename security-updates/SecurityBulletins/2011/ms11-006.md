Security Bulletin

Microsoft Security Bulletin MS11-006 - Crítica
==============================================

Vulnerabilidade no Processamento de Gráficos na Shell do Windows Poderia Permitir Execução Remota de Código (2483185)
---------------------------------------------------------------------------------------------------------------------

Data de publicação: 8 de fevereiro de 2011 | Updated: 14 de fevereiro de 2011

**actualizada:** 1.1

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve uma vulnerabilidade no processador de gráficos na Shell do Windows, divulgada publicamente. A vulnerabilidade poderia permitir a execução remota de código se um utilizador visualizasse uma imagem em miniatura especialmente concebida para o efeito. Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia obter os mesmos privilégios que o utilizador com sessão iniciada. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.

Esta actualização de segurança está classificada como Crítica para todas as edições suportadas do Windows XP, Windows Server 2003, Windows Vista e Windows Server 2008. As edições suportadas do Windows 7 e do Windows Server 2008 R2 não são afectadas pela vulnerabilidade. Para mais informações consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve a vulnerabilidade corrigindo a forma como o processador de gráficos na Shell do Windows analisa imagens em miniatura. Para mais informações sobre a vulnerabilidade, consulte a subsecção de Perguntas Mais Frequentes (FAQ) quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as vulnerabilidades**.

Esta actualização de segurança também resolve a vulnerabilidade descrita previamente no [Aviso de Segurança da Microsoft 2490606](http://technet.microsoft.com/security/advisory/2490606).

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem imediatamente a actualização, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões Conhecidas.** Nenhum

Vulnerabilidade de Sobrecarga de Processamento de Gráficos na Shell do Windows - CVE-2010-3970  
----------------------------------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de execução remota de código na forma como o processador de gráficos na Shell do Windows processa imagens em miniatura especialmente concebidas para o efeito. Um intruso que explorasse esta vulnerabilidade com sucesso poderia executar código arbitrário no contexto de segurança do utilizador com sessão iniciada. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970).
  
#### Factores Atenuantes para a Vulnerabilidade de Sobrecarga de Processamento de Gráficos na Shell do Windows - CVE-2010-3970
  
A atenuação refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   A vulnerabilidade não pode ser automaticamente explorada através de correio electrónico.  
-   Um intruso que conseguisse explorar esta vulnerabilidade com êxito poderia obter os mesmos privilégios que o utilizador com sessão iniciada. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
#### Soluções Alternativas para a Vulnerabilidade de Sobrecarga de Processamento de Gráficos na Shell do Windows - CVE-2010-3970
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e estados na discussão acerca de casos em que uma solução alternativa reduz a funcionalidade:
  
-   **Modificar a Lista de Controlo de Acesso (ACL) no ficheiro shimgvw.dll em sistemas Windows XP Windows Server 2003**
  
    **Nota** Consulte o [Artigo 2483185 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/2483185) para utilizar a solução automática de **correcção da Microsoft** para activar ou desactivar esta solução alternativa.
  
    Para modificar a ACL no ficheiro shimgvw.dll de modo a que seja mais restritivo, execute os comandos que se seguem a partir de uma linha de comandos como administrador:
  
    **Para edições de 32 bits do Windows XP e Windows Server 2003:**
  
    `Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:N`
  
    **Para edições de 64 bits do Windows XP e Windows Server 2003:**
  
    `Echo y| cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /P everyone:NEcho y| cacls %WINDIR%\SYSWOW64\shimgvw.dll /E /P everyone:N`
  
    **Impacto da solução alternativa.** Os ficheiros multimédia tipicamente processados pelo processador de gráficos na Shell do Windows não serão correctamente apresentados. Tenha em conta que deve anular esta solução alternativa de sistemas onde a tenha aplicado antes de instalar a actualização de segurança MS11-006.
  
    **Como anular esta solução alternativa.**
  
    Execute os comandos seguintes a partir de uma linha de comandos como administrador:
  
    Para edições de 32 bits do Windows XP e Windows Server 2003:
  
    `cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyone`
  
    Para edições de 64 bits do Windows XP e Windows Server 2003:
  
    `cacls %WINDIR%\SYSTEM32\shimgvw.dll /E /R everyonecacls %WINDIR%\SYSWOW64\shimgvw.dll /E /R everyone`
  
-   **Desactivar a visualização de miniaturas no Explorador do Windows em sistemas Windows Vista e Windows Server 2008**
  
    Para desactivar a visualização de miniaturas no Explorador do Windows no Windows Vista e no Windows Server 2008, execute os seguintes passos:
  
    1.  Abra o **Painel de controlo** e seleccione **Aspecto e Personalização**.  
    2.  Seleccione **Opções de Pastas** no menu.  
    3.  Seleccione o separador **Ver** e certifique-se de que a opção **Mostrar sempre ícones, nunca miniaturas** está seleccionada. Clique em **OK** para guardar as definições.  
    4.  Feche todas as instâncias abertas do Windows Explorer para a alteração ter efeito.
  
    **Impacto da solução alternativa.** O Explorador do Windows não apresentará imagens em miniatura.
  
    **Como anular esta solução alternativa.**
  
    Para activar as miniaturas no Windows Vista e no Windows Server 2008, execute os seguintes passos:
  
    1.  Abra o **Painel de controlo** e seleccione **Aspecto e Personalização**.  
    2.  Seleccione **Opções de Pastas** no menu.  
    3.  Seleccione o separador **Ver** e certifique-se de que a opção **Mostrar sempre ícones, nunca miniaturas** não está seleccionada. Clique em **OK** para guardar as definições.  
    4.  Feche todas as instâncias abertas do Windows Explorer para a alteração ter efeito.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Sobrecarga de Processamento de Gráficos na Shell do Windows - CVE-2010-3970
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de execução remota de código. Um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter o controlo total de um sistema afectado remotamente. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é causada quando o processador de gráficos na Shell do Windows analisa inadequadamente uma imagem em miniatura especialmente concebida para o efeito, resultando numa sobrecarga de memória intermédia.
  
**O que é a Shell do Windows?**    
A interface de utilizador do Windows (UI) fornece aos utilizadores acesso a uma grande variedade de objectos necessários para executar aplicações e gerir o sistema operativo. Os mais numerosos e familiares destes objectos são as pastas e ficheiros que residem nas unidades de disco do computador. Existem igualmente vários objectos virtuais que permitem que o utilizador efectue tarefas, tais como enviar ficheiros para impressoras remotas ou aceder à Reciclagem. A Shell organiza estes objectos num espaço de nome hierárquico e fornece aos utilizadores e aplicações uma forma consistente e eficiente de gerir e aceder a objectos.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Se um utilizador tiver sessão iniciada com direitos de utilizador administrativos, um intruso que conseguisse tirar partido desta vulnerabilidade poderia obter controlo total sobre o sistema afectado. Um intruso poderia então instalar programas; ver, alterar ou eliminar dados; ou ainda criar novas contas com todos os privilégios. Os utilizadores cujas contas estão configuradas com menos direitos de utilização no sistema podem correr menos riscos do que os utilizadores que trabalham com direitos de utilização de administrador.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Esta vulnerabilidade requer que um utilizador visualize uma imagem em miniatura especialmente concebida para o efeito.
  
Num cenário de ataque através da rede, um intruso poderia colocar uma imagem em miniatura especialmente concebida para o efeito, ou um ficheiro contendo uma imagem em miniatura especialmente concebida para o efeito, numa partilha de rede, tal como numa localização UNC ou WebDAV, convencendo em seguida o utilizar a navegar até essa localização através do Explorador do Windows. Quando o utilizador navegar até à localização de partilha, o caminho de controlo afectado é activado através do processador de gráficos na Shell do Windows. A imagem em miniatura especialmente concebida para o efeito poderia então explorar a vulnerabilidade e executar código no contexto de segurança do utilizador com sessão iniciada. Um intruso não conseguiria obrigar os utilizadores a visitarem uma localização de partilha de rede, UNC ou WebDAV. Em vez disso, um intruso teria de convencê-los a visitar a partilha, levando-os normalmente a clicar numa hiperligação presente numa mensagem de correio electrónico ou num programa de mensagens instantâneas que os levasse até à localização da partilha de rede, UNC ou WebDAV.
  
**O que é a Convenção de Nomenclatura Universal (UNC, Uniform Naming Convention)? **  
A Convenção de Nomenclatura Universal (UNC) é um método de identificação de recursos tal como nomes de partilhas ou ficheiros numa rede. Um nome UNC típico começa com duas barras invertidas, seguidas do nome de um servidor, tal como por exemplo \\\\servidor\\partilha\\subdirectório\\ficheiro.
  
**Quais os sistemas mais susceptíveis a esta vulnerabilidade?**    
Os sistemas afectados são principalmente estações de trabalho e servidores de terminais. Os servidores podem estar mais sujeitos a risco se os administradores permitirem que os utilizadores iniciem sessão nos servidores e executem programas. Contudo, os procedimentos recomendados desencorajam essa acção.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade corrigindo a forma como o processador de gráficos na Shell do Windows analisa imagens em miniatura.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Sim. Esta vulnerabilidade foi divulgada publicamente. Foi-lhe atribuído o número de Vulnerabilidade e Exposição Comum [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970). Esta vulnerabilidade foi previamente descrita no [Aviso de Segurança da Microsoft 2490606](http://technet.microsoft.com/security/advisory/2490606).
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido informações sobre a exploração desta vulnerabilidade?**    
Não. A Microsoft tinha visto exemplos divulgados publicamente de código para prova de conceito, mas não tinha recebido quaisquer relatórios directos que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.
  
**A aplicação desta actualização de segurança ajuda a proteger os clientes contra o código que foi divulgado publicamente e que tenta explorar esta vulnerabilidade?**    
Sim. Esta actualização de segurança resolve a vulnerabilidade que poderia ser potencialmente explorada usando o código de prova de conceito publicado. À vulnerabilidade que foi corrigida foi atribuído o número de Vulnerabilidade e Exposição Comum [CVE-2010-3970](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2010-3970).

-   **Verificação das Versões dos Ficheiros**

    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.

    1.  Clique em **Iniciar** e, em seguida, em **Procurar**.
