Security Bulletin

Microsoft Security Bulletin MS11-064 - Importante
=================================================

Vulnerabilidades na Pilha de TCP/IP Poderiam Permitir Negação de Serviço (2563894)
----------------------------------------------------------------------------------

Data de publicação: 9 de agosto de 2011

**actualizada:** 1.0

### Informações Gerais

#### Resumo Executivo

Esta actualização de segurança resolve duas vulnerabilidades no Microsoft Windows, comunicadas de forma privada. As vulnerabilidades poderiam permitir negação de serviço se um intruso enviasse uma sequência de mensagens ICMP (Internet Control Message Protocol) especialmente concebidas para o efeito para um sistema visado, ou se enviasse um pedido URL especialmente concebido para o efeito para um servidor que fornecesse conteúdos Web e que tivesse a funcionalidade QoS (Quality of Service) baseada em URL activada.

Esta actualização de segurança está classificada como Importante para todas as edições suportadas do Windows Server 2008 e Windows Server 2008 R2 e como Moderada para todas as edições suportadas do Windows Vista e Windows 7. Para mais informações, consulte a subsecção **Software Afectado e Software Não Afectado**, nesta secção.

A actualização de segurança resolve as vulnerabilidades modificando a forma como a pilha de TCP/IP do Windows processa mensagens ICMP e processa URLs na memória quando a QoS baseada em URL está activada. Para mais informações sobre as vulnerabilidades, consulte a subsecção de Perguntas Mais Frequentes quanto à vulnerabilidade específica na secção seguinte, **Informações sobre as Vulnerabilidades**.

**Recomendação.** A maioria dos clientes tem a actualização automática activada e não necessitará de efectuar qualquer acção, porque esta actualização de segurança será transferida e instalada automaticamente. Os clientes que não tenham a actualização automática activada necessitam de verificar as actualizações e instalar esta actualização manualmente. Para informações sobre opções específicas de configuração da actualização automática, consulte o [Artigo 294871 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/294871).

Para administradores e instalações empresariais, ou utilizadores finais que pretendam instalar esta actualização de segurança manualmente, a Microsoft recomenda que os clientes apliquem a actualização na primeira oportunidade, utilizando software de gestão de actualizações ou verificando as actualizações através do serviço [Microsoft Update](http://go.microsoft.com/fwlink/?linkid=40747).

Consulte também a secção **Orientações e Ferramentas de Detecção e Implementação**, mais à frente neste boletim.

**Questões conhecidas.** Nenhum

Vulnerabilidade de Negação de Serviço de ICMP - CVE-2011-1871  
-------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de negação de serviço na pilha de TCP/IP do Windows que é provocada quando a pilha de TCP/IP processa inadequadamente uma sequência de mensagens ICMP especialmente concebidas para o efeito. Um intruso que explorasse esta vulnerabilidade com êxito poderia fazer com que o sistema visado deixasse de responder e fosse reiniciado automaticamente.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2011-1871](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1871).
  
#### Factores Atenuantes para a Vulnerabilidade de Negação de Serviço de ICMP - CVE-2011-1871
  
A Microsoft não identificou quaisquer factores atenuantes para esta vulnerabilidade.
  
#### Soluções Alternativas para a Vulnerabilidade de Negação de Serviço de ICMP - CVE-2011-1871
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e indica se a solução alternativa reduz a funcionalidade:
  
-   **Bloquear ICMP na firewall**
  
    Bloquear todas as mensagens ICMP na firewall de limite de rede ou no router ajudará a proteger os sistemas que estão por detrás desta firewall ou router contra tentativas de explorar esta vulnerabilidade. Recomendamos que bloqueie todas as comunicações de entrada não solicitadas da Internet.
  
    **Impacto da solução alternativa.** Esta solução alternativa pode ter um impacto negativo no desempenho, impedindo que o TCP optimize a comunicação de rede. Os pacotes de rede ICMP podem eliminar a fragmentação em routers que ligam redes com MTUs diferentes. A fragmentação reduz o débito do TCP e aumenta o congestionamento da rede.
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Negação de Serviço de ICMP - CVE-2011-1871
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de negação de serviço.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é provocada quando a pilha de TCP/IP do Windows processa inadequadamente uma sequência de mensagens ICMP especialmente concebidas para o efeito.
  
**Qual é o componente afectado pela vulnerabilidade?**    
O Tcpip.sys é o componente afectado pela vulnerabilidade.
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade com êxito poderia fazer com que o sistema visado deixasse de responder e fosse reiniciado automaticamente.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Um intruso remoto não autenticado poderia explorar esta vulnerabilidade criando um programa para enviar uma sequência de mensagens ICMP especialmente concebidas para o efeito para um sistema visado.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Os servidores e estações de trabalho são afectados por esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade modificando a forma como a pilha de TCP/IP do Windows processa mensagens ICMP.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**    
Não. A Microsoft recebeu informações sobre esta vulnerabilidade através de divulgação coordenada de vulnerabilidades.
  
**Quando este boletim de segurança foi publicado, a Microsoft já tinha recebido relatórios indicando que esta vulnerabilidade estava a ser explorada?**    
Não. A Microsoft ainda não tinha recebido informações que indicassem que esta vulnerabilidade tivesse sido utilizada publicamente para atacar clientes, quando este boletim de segurança foi publicado pela primeira vez.
  
Vulnerabilidade de Negação de Serviço de QoS de TCP/IP - CVE-2011-1965  
----------------------------------------------------------------------
  
<span></span>
Existe uma vulnerabilidade de negação de serviço na pilha de TCP/IP do Windows que é provocada quando a pilha de TCP/IP processa inadequadamente URLs na memória quando a QoS (Quality of Service) baseada em URL está activada. Um intruso que explorasse esta vulnerabilidade com êxito poderia fazer com que o sistema visado deixasse de responder e fosse reiniciado automaticamente.
  
Para visualizar esta vulnerabilidade como uma entrada comum na lista de Vulnerabilidades e Exposições Comuns, consulte [CVE-2011-1965](http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=cve-2011-1965).
  
#### Factores Atenuantes para a Vulnerabilidade de Negação de Serviço de QoS de TCP/IP - CVE-2011-1965
  
O factor atenuante refere-se a uma definição, uma configuração comum ou uma prática recomendada geral, que existe num estado predefinido, que poderia reduzir a gravidade da exploração de uma vulnerabilidade. Os seguintes factores atenuantes podem ser úteis para a sua situação:
  
-   Por predefinição, a funcionalidade Quality of Service baseada em URL não está activada em nenhum sistema operativo Windows. Para serem afectados por esta vulnerabilidade, os utilizadores teriam de instalar manualmente esta funcionalidade.
  
#### Soluções Alternativas para a Vulnerabilidade de Negação de Serviço de QoS de TCP/IP - CVE-2011-1965
  
Uma solução alternativa refere-se a uma definição ou alteração de configuração que não corrige a vulnerabilidade subjacente, mas que ajudaria a bloquear vectores de ataque conhecidos antes de aplicar a actualização. A Microsoft testou as seguintes soluções alternativas e indica se a solução alternativa reduz a funcionalidade:
  
-   Desactive a QoS baseada em políticas. Para mais informações, consulte o artigo da Microsoft TechNet sobre [QoS (Quality of Service) baseada em políticas](http://technet.microsoft.com/en-us/library/dd919203(ws.10).aspx).
  
#### Perguntas Mais Frequentes sobre a Vulnerabilidade de Negação de Serviço de QoS de TCP/IP - CVE-2011-1965
  
**Qual é a abrangência desta vulnerabilidade?**    
Esta é uma vulnerabilidade de negação de serviço.
  
**O que provoca a vulnerabilidade?**    
A vulnerabilidade é provocada quando a pilha de TCP/IP processa inadequadamente URLs especialmente concebidos para o efeito na memória. Esta vulnerabilidade apenas é activada quando o sistema visado aloja conteúdo Web através de um serviço como o IIS e tem a QoS baseada em URL activada.
  
**Qual é o componente afectado pela vulnerabilidade?**    
O Tcpip.sys é o componente afectado pela vulnerabilidade.
  
**O que é a QoS (Quality of Service) baseada em URL?**    
A QoS (Quality of Service) para redes é um conjunto de normas e mecanismos que abrange toda a indústria, destinado a garantir um desempenho de alta qualidade em aplicações críticas. Os administradores de rede podem utilizar mecanismos da QoS para dar prioridade e gerir a taxa de envio do tráfego de rede de saída. A utilização destes mecanismos assegura uma utilização eficaz dos recursos, de forma a oferecer o nível de serviço pretendido.
  
O Windows Server 2008 R2 permite que os profissionais de TI dêem prioridade ao tráfego da Web baseado em URLs. Com a QoS baseada em URL, os profissionais de TI podem garantir que o tráfego Web importante é processado antes do tráfego menos importante, melhorando o desempenho nas redes mais ocupadas. Por exemplo, os profissionais de TI podem atribuir maior prioridade ao tráfego de Web sites críticos internos, por oposição aos Web sites externos, melhorando assim o desempenho quando a rede está ocupada. Para mais informações, consulte o artigo da Microsoft TechNet sobre [QoS baseada em URL](http://technet.microsoft.com/en-us/library/dd637810(ws.10).aspx).
  
**O que é um URI?**    
Um URI (Uniform Resource Identifier) é uma cadeia de caracteres utilizada para identificar ou agir sobre recursos da Internet ou de uma rede. Um URL é um exemplo típico de um URI que referencia um recurso, como um Web site. Para mais informações sobre URIs, consultar [RFC-2396](http://www.ietf.org/rfc/rfc2396.txt).
  
**Como poderia um intruso utilizar a vulnerabilidade?**    
Um intruso que explorasse esta vulnerabilidade com êxito poderia fazer com que o sistema visado deixasse de responder e fosse reiniciado automaticamente.
  
**Como poderia um intruso explorar a vulnerabilidade?**    
Num cenário de ataque de ataque remoto, um intruso não autenticado poderia explorar esta vulnerabilidade enviando um pedido de URL especialmente concebido para o efeito para um servidor que fornecesse conteúdo Web e tivesse a QoS baseada em URL activada.
  
**Quais os sistemas mais susceptíveis face a esta vulnerabilidade?**    
Os servidores que alojam conteúdo Web e que têm a QoS baseada em URL activada estão mais susceptíveis. Os servidores que não têm a QoS baseada em URL activada ou que não fornecem conteúdo Web não são afectados. As estações de trabalho, como os sistemas com Windows 7, que podem executar o Internet Information Services (IIS) estão mais susceptíveis se tanto o IIS como a QoS baseada em URL estiverem activados.
  
**Os sistemas com Windows 7 estão mais susceptíveis quando configurados como servidores com o IIS?**    
Sim. O software do servidor, como o IIS, pode ser instalado e configurado nos sistemas com o Windows 7. Embora os sistemas com o Windows 7 sejam considerados estações de trabalho, nesta configuração funcionam como servidores, pelo que estão mais susceptíveis a esta vulnerabilidade.
  
**O que faz a actualização?**    
A actualização resolve a vulnerabilidade modificando a forma como a pilha de TCP/IP processa URLs na memória quando a QoS baseada em URL está activada.
  
**Quando este boletim de segurança foi publicado, esta vulnerabilidade já tinha sido divulgada publicamente?**

#### Informações de Implementação
  
**Instalação da actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de uma correcção da Microsoft.
  
Para mais informações sobre a terminologia que é usada neste boletim, tal como *correcção*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro         | Descrição                                                                                                                                            |  
|-------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help** | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**        | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**    | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte o [Artigo 934307 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/934307).
  
**Verificar se a actualização foi aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e Ferramentas de Detecção e Implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização na caixa **Iniciar Pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.

#### Informações de Implementação
  
**Instalação da actualização**
  
Quando instala esta actualização de segurança, o programa de instalação verifica se um ou mais ficheiros que estão a ser actualizados no seu sistema foram alvo de uma actualização anterior através de uma correcção da Microsoft.
  
Para mais informações sobre a terminologia que é usada neste boletim, tal como *correcção*, consulte o [Artigo 824684 da Base de Dados de Conhecimento da Microsoft](http://support.microsoft.com/kb/824684).
  
Esta actualização de segurança suporta os seguintes parâmetros de configuração.
  
| Parâmetro                            | Descrição                                                                                                                                            |  
|--------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------|  
| **/?, /h, /help**                    | Apresenta ajuda para os parâmetros suportados.                                                                                                       |  
| **/quiet**                           | Elimina a janela de mensagem de estado ou de erro.                                                                                                   |  
| **/norestart**                       | Quando combinado com **/quiet**, o sistema não será reiniciado depois da instalação, mesmo que seja necessário reiniciar para concluir a instalação. |  
| **/warnrestart:&lt;seconds&gt;**     | Quando combinado com **/quiet**, o programa de instalação avisa o utilizador antes de dar lugar ao reinício.                                         |  
| **/promptrestart**                   | Quando combinado com **/quiet**, o programa de instalação solicitará confirmação antes de dar lugar ao reinício.                                     |  
| **/forcerestart**                    | Quando combinado com **/quiet**, o programa de instalação força o encerramento das aplicações e dá lugar ao reinício.                                |  
| **/log:&lt;file name&gt;**           | Activa o registo no ficheiro especificado.                                                                                                           |  
| **/extract:&lt;destination&gt;**     | Extrai o conteúdo do pacote para a pasta de destino.                                                                                                 |  
| **/uninstall /kb:&lt;KB Number&gt;** | Desinstala a actualização de segurança.                                                                                                              |
  
**Nota** Para mais informações sobre o programa de instalação wusa.exe, consulte a informação referente ao Programa de Instalação Independente do Windows Update no artigo TechNet sobre [Alterações Diversas no Windows 7](http://technet.microsoft.com/en-us/library/dd871148(ws.10).aspx).
  
**Verificar se a actualização foi aplicada**
  
-   **Microsoft Baseline Security Analyzer**
  
    Para verificar se uma actualização de segurança foi aplicada num sistema afectado, poderá utilizar a ferramenta Microsoft Baseline Security Analyzer (MBSA). Consulte a secção "**Orientações e Ferramentas de Detecção e Implementação**" mais acima neste boletim para obter mais informações.
  
-   **Verificação das Versões dos Ficheiros**
  
    Dado existirem várias edições do Microsoft Windows, os passos que se seguem podem ser diferentes no seu sistema. Caso efectivamente o sejam, consulte a documentação do produto para executar estes passos.
  
    1.  Clique em **Iniciar** e depois introduza um nome de ficheiro de actualização na caixa **Pesquisa**.  
    2.  Quando o ficheiro aparecer em **Programas**, clique com o botão direito do rato no nome do ficheiro e depois em **Propriedades**.  
    3.  No separador **Geral**, compare o tamanho do ficheiro com as tabelas de informação de ficheiros fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
        **Nota** Dependendo da edição do sistema operativo, ou dos programas instalados no sistema, alguns dos ficheiros incluídos na tabela de informações de ficheiro poderão não estar instalados.  
    4.  Também pode clicar no separador **Detalhes** e comparar informações, como a versão de ficheiro e a data de modificação, com as tabelas de informação fornecidas no artigo da Base de Dados de Conhecimento do boletim.  
        **Nota** Outros atributos que não a versão do ficheiro poderão sofrer alterações durante a instalação. Comparar outros atributos de ficheiro com as informações contidas na tabela de informações de ficheiro não é um método suportado para verificar a aplicação da actualização. Além disso, em determinados casos, o nome dos ficheiros poderá ser alterado durante a instalação. Se as informações de ficheiro ou de versão não estiverem presentes, utilize um dos outros métodos disponíveis para verificar a instalação da actualização.  
    5.  Por fim, também pode clicar no separador **Versões anteriores** e comparar a informação de ficheiros sobre a versão anterior do ficheiro com a informação da versão do ficheiro nova ou actualizada.
